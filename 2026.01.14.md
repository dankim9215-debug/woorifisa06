1.데이터 분석 패키지 판다스와 그 활용법

1-1.판다스 패키지의 이해

- 판다스는 통계 분석 패키지로, 데이터의 경향성을 파악함
    
- 데이터를 분석하기 위해 그래프를 그리고 시각화를 통해 탐색이 가능함
    
- 판다스 패키지에는 데이터를 분석하는 방법과 데이터를 만드는 방법이 포함됨
    
- 판다스를 활용해 데이터를 분석하면 어떤 결과를 얻을 수 있는지 방법을 함께 확인해야 함
    

1-2.판다스 패키지의 불러내기 및 활용

- 판다스 패키지 사용을 위해 필요한 것은 버스 노선을 확인하는 것임
    
- 판다스 패키지에서는 불러온 데이터를 판다스의 데이터 프레임으로 변환함
    
- 판다스는 데이터를 일괄적으로 처리하고, 데이터의 대부분을 문자열로 이루어진 데이터임
    
- 판다스는 낱낱의 차원을 가진 데이터를 다루기 위해 사용됨
    

1-3.판다스 패키지의 데이터 프레임 조작

- 판다스 패키지에서 데이터 프레임 조작을 위해 필요한 것은 CSV 파일을 판다스의 데이터 프레임으로 만드는 것임
    
- 판다스 패키지는 불러온 데이터를 판다스의 데이터 프레임으로 만듦
    
- 판다스 패키지에서 데이터를 분석할 때는 판다스의 데이터 프레임을 활용함
    
- 판다스 패키지는 데이터 분석의 결과를 바탕으로 프로그램을 개발함
    

2.데이터 처리

2-1.데이터 프레임

- 판다스 명령어로 데이터를 가져옴
    
- 판다스 데이터 프레임은 열, 행, 인덱스, 값으로 구성됨
    
- 원본 파일에서 순서를 의미하는 인덱스를 가져와 활용함
    
- 데이터 프레임은 일괄적용이 가능하도록 구성되어 있음
    
- 컬럼과 인덱스를 통한 데이터의 변환과 탐색이 가능함
    

2-2.데이터 확인과 변환

- '정보' 명령어로 데이터의 전반적인 구조를 확인함
    
- '비_없음'을 포함한 자료형의 값이 없을 경우, 원래 자료형으로 변환함
    
- '참고' 명령어로 문자열 데이터의 빈도, 중복 등을 확인함
    
- '대표' 명령어로 통계치를 확인하여 데이터의 특성을 파악함
    
- '인보' 명령어로 등록일자를 시간 자료형으로 변환함
    

2-3.데이터의 자료형 변환

- '인보' 명령어로 등록일자의 자료형을 변경함
    
- 원본 데이터를 '시간' 자료형으로 변환한 후, 메모리에 저장함
    
- '때문에' 명령어로 원본 데이터의 자료형을 유지하면서 변환 가능함
    
- 원본 데이터를 유지하면서 자료형을 변경하는 작업은 대부분이 아니며, 파라미터 공유를 통한 변환이 이루어짐
    

3.파이썬과 판다스를 활용한 데이터 분석

3-1.데이터 분석을 위한 파이썬과 판다스 사용법

- 파이썬에서는 불링 인덱싱을 사용하여 조건에 맞는 값을 찾음
    
- 필요한 값들을 원하는 방식으로 리턴할 수 있음
    
- 개나리반이면서 나이가 25살 미만인 사람이나, 점수가 70점 미만인 사람 등을 찾는 예시를 통해 설명함
    

3-2.조건에 따른 값의 행별, 열별 접근 방법

- 파이썬에서는 '와', '또는' 대신에 '그리고', '또는' 방식을 사용하여 값의 행별, 열별 접근을 함
    
- 행별로 값을 뜯어오는 방식을 이용하여 특정 조건에 따른 값을 행별로 접근함
    
- 판다스에서는 열행 순서로 접근함
    
- 조건에 따라 행별로 값을 뜯어오는 방식을 사용하면, 각각의 행에 대해 행별로 값을 뜯어올 수 있음
    

3-3.조건에 따른 값을 가져오기

- 조건에 따라 값을 가져오는 경우, 특정 조건을 만족하는 값을 별도로 찾아낼 수 있음
    
- 조건을 만족하는 값들을 찾을 때, '그리고'나 '또는'을 사용하여 각 조건에 따른 값을 찾아낼 수 있음
    
- 2개 이상의 조건을 만족하는 값을 찾아내는 예시를 통해 설명함
    
- 조건에 따라 다른 값을 가져오기 위해 '그리고'나 '또는'을 사용하는 방법을 강조함
    

4.파이썬 판다스 활용

4-1.판다스 소개

- 판다스는 파이썬의 빅데이터 커널 툴임
    
- 테스트, 가설 검증, 시각화, 분석을 하나의 명령어로 통합함
    
- 여러 열을 통합해 하나의 열을 만들거나, 특정 열을 선택하여 직접 제어할 수 있음
    
- '구조화된 방식'으로 데이터를 관리하기 위해 '데이터 프레임' 자료형을 사용함
    
- '디프레임'과 '리프레임'의 차이를 명확히 이해해야 함
    

4-2.컬럼 제어

- '컬럼'을 삭제하거나 '행'이나 '열' 단위로 직접 접근하여 제어할 수 있음
    
- '키워드'를 활용하여 원하는 위치의 값을 직접 찾을 수 있음
    
- '라디오' 명령어를 사용하여 열의 위치를 조정할 수 있음
    
- '디프레임'에서 '리프레임'으로 데이터를 레이아웃을 변경할 수 있음
    
- '디프레임'과 '리프레임'을 구분하여 사용해야 함
    

4-3.값 수정

- '값 수정' 명령어를 통해 특정 위치의 값을 수정할 수 있음
    
- '코드'를 통해 해당 위치의 값을 수정하고, '데이터 프레임'에서 새로운 데이터를 추가할 수 있음
    
- '데이터 프레임'의 '리프레임'과 '디프레임'을 명확히 구분하여 사용해야 함
    
- '리프레임'을 활용하여 데이터의 순서를 바꾸고, '디프레임'을 통해 특정 위치의 값을 직접 확인할 수 있음
    
- 값을 수정하는 과정은 '코드'를 통해 원하는 위치의 값을 확인하고, '데이터 프레임'에서 새로운 데이터를 추가하거나 수정할 수 있음
    

5.데이터프레임

5-1.데이터프레임 조작

- 데이터를 원하는 대로 재구조화하여 관리 가능함
    
- 값이 없는 자료를 맨 앞으로 순서로 정렬하거나 맨 위로 정렬해줌
    
- 정렬된 순서 데이터를 다시 그대로 주석 처리하여 분석 용이하게 함
    
- 원본 데이터를 조작하여 다른 데이터 프레임으로 변환 가능함
    
- 조작 과정 중 분열된 데이터를 '중간'을 통해 하나로 합칠 수 있음
    

5-2.집계함수

- 키 기준으로 데이터를 묶어 통계치를 구하는 집계함수 있음
    
- 집계함수의 결과는 통계치로 개선됨
    
- 조건에 맞는 데이터를 뽑아내기 위해 '그룹바이' 방식을 사용함
    
- '프라이스'와 '수량' 컬럼을 조합하여 가격과 수량을 보여줌
    
- 원본 데이터를 조작하여 다른 형태의 데이터 프레임으로 변환 가능함
    

5-3.데이터 변환

- '그룹바이' 방식을 통해 원본 데이터를 변환하여 관리하기 쉬이 만듦
    
- '매트'와 '상품'을 조건에 따라 가격과 수량에 따라 정렬하여 나타냄
    
- 행렬 형태로 표현하여 데이터의 가격과 수량을 쉽게 확인 가능하게 함
    
- 조건에 따라 데이터를 묶어 놓는 '집계함수'를 사용하여 데이터를 재구조화함
    
- 이 방법을 통해 각 마트별 상품 가격과 수량을 쉽게 알 수 있음
    

6.데이터 프레임 기능

6-1.데이터 프레임 기능 소개

- 데이터 프레임, 원본 데이터를 처리해 새로운 데이터 집합을 만드는 작업임
    
- 원본 데이터를 그룹바이를 통해 묶고 새로운 행렬을 만드는 과정을 포함함
    
- 그룹바이 기법은 분류, 카테고리화, 상호작용 분석 등 다양한 분야에서 활용됨
    
- 원본 데이터를 행렬로 변환하는 과정은 필요에 따라 다양한 방식으로 진행될 수 있음
    
- 그룹바이를 통해 카테고리화된 데이터를 표현할 수 있음
    

6-2.그룹바이와 피벗의 차이

- 그룹바이는 데이터를 카테고리로 묶어 새로운 행렬을 만드는 과정임
    
- 피벗은 원래의 데이터를 컬럼으로 재정렬하여 새로운 데이터 집합을 만드는 방법임
    
- 피벗은 회사의 사업 방향을 변경하는 등의 의미로도 사용됨
    
- 피벗을 통해 원래 데이터의 순서를 바꾸고, 새로운 순서에 따라 새로운 데이터 집합을 생성할 수 있음
    
- 데이터 엔지니어는 이러한 기능을 활용하여 데이터를 분석하고, 시각화하여 보고서를 작성함
    

6-3.맵과 어플라이의 사용

- 맵은 원본 데이터에서 필요한 동작을 특정한 순서로 정리한 데이터 프레임임
    
- 어플라이는 원본 데이터를 묶어주는 일괄적인 동작을 가진 데이터 프레임임
    
- 맵과 어플라이는 데이터 프레임의 기능으로, 원본 데이터를 특정 순서로 묶어 새로운 데이터 집합을 만드는 데 사용됨
    
- 맵과 어플라이는 원본 데이터에 스트링 메서드를 적용하거나, 다른 순서로 데이터를 정렬하는 등의 기능을 제공함
    
- 데이터 엔지니어는 이러한 기능을 활용하여 데이터를 분석하고, 보고서를 작성함
    

7.데이터 프레임 함수

7-1.데이터 프레임을 통한 값 출력

- 데이터 프레임의 각 열에 따라 함수를 만들어서 값들을 출력함
    
- 열의 번호와 해당 열에 값을 대입한 후, 해당 열에 따라 값을 출력하는 함수를 만듦
    
- 예를 들어, x라는 문자열을 입력받아 그에 따라 결과값을 출력하는 함수를 만들 수 있음
    
- 이렇게 생성한 함수를 활용하여 데이터 프레임을 입력받아 각 열에 따른 값을 출력함
    
- 데이터 프레임의 특정 열에만 값을 대입하는 등, 필요한 부분만 선택하여 동작하도록 함수를 조정할 수 있음
    

7-2.시리즈 단위로의 함수 적용

- 데이터 프레임을 시리즈 단위로 입력받아 각 원소별, 시리즈별로 동작하는 함수를 만들 수 있음
    
- 시리즈별로 동작하는 함수를 만들기 위해, 각 시리즈를 통째로 입력받아 그 안의 값을 하나씩 차례대로 가져오는 방식을 활용함
    
- 첫 번째 데이터 프레임을 통째로 입력받아 그 안의 모든 값을 가져다가 각 시리즈별로 동작하도록 함수를 만듦
    
- 두 번째 데이터 프레임을 입력받아 각 시리즈별로 원소별, 개별/장미 등 각 열에 따라 다른 값을 가져다 줌
    
- 이렇게 시리즈 단위로의 함수 적용을 통해, 각 시리즈별로 원소별로 다른 값을 가져다 주는 방식을 확인할 수 있음
    

7-3.함수의 재사용과 한계

- 직접 함수를 만들어서 사용할 때, 함수의 작동 구조와 입력 파라미터를 확인할 수 있도록 주의해야 함
    
- 함수를 재사용하기 위해서는 입력 파라미터를 하나씩 확인하여 수정하는 과정이 필요함
    
- 이렇게 추가한 함수를 활용하여, 각 시리즈별로 원소별, 개별/장미 등 각 열에 따라 다른 값을 가져다 주는 방식을 확인할 수 있음
    
- 데이터 프레임을 일괄적으로 동작시키고 싶다면, 행 번호를 기준으로 동작하도록 함수를 만들고, '열' 기준으로 동작하도록 설정함
    

8.데이터 전처리 함수

8-1.함수 작성과 시리즈 단위

- 일괄처리 방식은 데이터 프레임의 열에만 적용하려 할 때 사용함
    
- 데이터 프레임의 시리즈 단위로 값을 적용하는 함수는 복잡한 데이터 전처리 시 사용함
    
- 열 단위로 적용하려면 엑시스 파라미터를 사용하여 특정 열에만 처리함
    
- 복잡한 데이터 전처리 시 데이터 프레임 단위로 일괄처리하는 함수를 만듦
    
- 데이터 전처리 함수를 일괄 적용하려면 데이터 프레임 단위로 적용해야 함
    

8-2.실제 사례

- 스트링 데이터를 전처리할 때, 데이터 프레임의 열에 따라 일괄처리하는 함수를 만들 수 있음
    
- 예를 들어, 서울특별시, 서울시, SEOL로 데이터를 일괄적으로 변환할 수 있음
    
- 데이터 전처리 함수를 직접 만들어 모든 열에 일괄적으로 적용함
    
- 일괄처리 함수는 데이터 프레임 단위로 적용하여 복잡한 데이터 전처리를 수행함
    
- 예제로, 강동구와 광명시의 위치를 변경하고 분점이라는 컬럼을 추가하는 함수를 작성함
    

8-3.일괄처리의 활용

- 일괄처리 함수를 사용하면 메모리 공간에 결과 값을 일괄적으로 적용함
    
- 결과 값의 수정이 없으면 일괄처리 후 원하는 방식으로 데이터를 저장함
    
- 수업 시간이 오래 걸리니, 수업 후 바로 온라인으로 확인해 보기를 권장함
    
- 일괄처리 함수는 현업에서 실시간으로 데이터를 처리할 때 중요한 기능임
    

9.데이터 분석을 위한 파이썬 함수 개발

9-1.데이터 처리를 위한 파이썬 함수의 필요성

- 데이터 분석 작업을 파이썬 함수를 활용하면 편리하고 효율적임
    
- 복잡한 데이터 처리를 위한 다양한 내장 함수들 활용 필요성 인식
    
- 단순한 데이터 처리 이외 복잡한 작업 처리를 위한 함수 개발 필요성 제시
    
- 데이터 프레임 단위로의 작업이 가능하도록 파이썬 함수 작성 필요성 제시
    

9-2.파이썬 함수를 통한 데이터 반의 동작

- 데이터의 반을 어떤 메서드로 변경하느냐에 따라 결과 값이 달라짐
    
- 키워드에 따라 동작하는 파이썬 함수를 만들어 원하는 값에 반영 가능
    
- 특정 값이 없는 경우를 대비해 미리 정의하여 사용하는 것이 효과적임
    
- 데이터 프레임 단위로 적용 시, 행 단위로 적용 시에 동작 방식의 차이점을 인지해야 함
    

9-3.파이썬 함수 개발 시 주의점

- 파이썬 함수 개발 시 키워드에 따른 동작 확인이 필수적임
    
- 불필요한 작업을 최소화하고, 원하는 동작만 선택하여 실행할 수 있도록 관리 필요
    
- 동작 결과를 원하는 방향으로 변경하려면, 각각의 동작을 확인하고 필요한 값을 할당하는 것이 중요
    
- 데이터 프레임 단위로 적용 시, 행 단위로 적용 시에는 동작 방식의 차이점을 인지해야 함
    

10.데이터 탐색과 함수

10-1.데이터 탐색 기초

- 데이터 프레임은 컬럼, 인덱스, 밸류로 구성됨
    
- 컬럼과 인덱스는 특정 데이터 값에 접근하기 위한 기준이 됨
    
- 값 자체에서 특정 값을 찾는 명령어와 컬럼에서 특정 값을 찾는 명령어는 다름
    
- 필터, 블링 인덱싱, 로케이션, 아이로케이션 명령어를 사용해 데이터 탐색 가능
    
- 아이템이라는 키워드를 사용해 특정 컬럼의 값을 찾을 수 있음
    

10-2.원하는 문자열 찾기

- 필터 명령어를 사용해 원하는 문자열을 찾아줌
    
- 정규식으로 원하는 문자열을 찾아주고, 열 기준으로 원하는 값을 찾을 수 있음
    
- 블링 인덱싱과 로케이션, 아이로케이션 명령어도 사용 가능
    
- 아이템이라는 키워드를 사용해 특정 컬럼의 값을 찾을 수 있음
    
- 값 자체에서 원하는 문자열을 찾을 수도 있음
    

10-3.데이터 저장과 함수

- 데이터를 저장하기 위해선 결과를 메모리에 달아야 함
    
- 이프문을 통해 최종적으로 원하는 결과를 반환해 메모리에 남김
    
- 함수를 만들 때, 반환 메모리에 결과를 저장해야 메모리에 남게 됨
    
- 함수를 통해 결과를 메모리에 저장하고, 다시 원하는 결과를 반환하는 과정이 중요함
    
- 내장 함수를 활용하거나, 직접 함수를 만들어 데이터 저장 및 반환을 처리할 수 있음
    

11.컬럼 검색

11-1.컬럼 검색의 기본

- 기본값을 기준으로 특정 값을 찾을 수 있음
    
- 필터 명령어를 사용하면 열 또는 행 기준으로 특정 값을 찾을 수 있음
    
- 이징 명령어를 통해 기본값과 동일한 값을 찾을 수 있음
    
- 나머지 값을 검색할 때는 검색 조건을 넣어줘야 함
    
- 나머지 값이 모두 포함돼야 함
    

11-2.조건부 검색

- 완전 일치하는 값을 찾을 수도 있고, 일부 일치하는 값을 찾을 수도 있음
    
- 조건부 검색을 위해서는 포함되는 문자열을 정하고, 포함되는 값을 찾아야 함
    
- 예를 들어, 문자열에서 완전 일치하는 단어를 찾고, 그 단어가 포함된 모든 컬럼을 찾을 수 있음
    
- 컨테인즈 명령어를 사용하면 특정 문자열이 포함된 열을 찾을 수 있음
    
- 이징 명령어를 사용하면 완전 일치하는 문자열이 포함된 열을 찾을 수 있음
    

11-3.패턴 기반 검색

- 일부 패턴을 갖는 값을 찾을 때는 점(=)을 사용해 기준을 정할 수 있음
    
- 패턴의 시작이나 끝에 특정 패턴이 포함된 값을 찾을 수 있음
    
- 이징 명령어를 사용하면 특정 패턴이 포함된 열을 찾을 수 있음
    
- 패턴에 따라 다른 값을 찾을 수 있으므로, 검색 시 유의해야 함
    

12.데이터 프레임과 데이터 샘플의 이해

12-1.데이터 프레임의 개념과 필요성

- 데이터 프레임은 하나하나의 샘플들로 구성된 데이터의 구조
    
- 각각의 샘플이 데이터의 특정한 특성을 갖는 데이터 항목들로 구성
    
- 각각의 샘플을 바탕으로 전체 데이터를 이해하고 해석
    
- 각각의 샘플을 하나씩 더해 나가며 전체 데이터를 구성
    
- 다양한 도메인에서 사용되는 데이터 프레임의 예로는 시리즈, 행, 열, 원 등이 있음
    

12-2.데이터 프레임의 구조와 작성

- 데이터 프레임은 특정 패턴에 맞는 값을 찾아내서, 이를 바탕으로 데이터를 처리하거나 변경 가능
    
- 데이터 프레임 작성 시에는 실제 데이터를 토대로 정확하게 구조화
    
- 데이터 프레임의 각각의 값들은 서로 연관되어 있으며, 이를 통해 전체 데이터를 이해
    
- 데이터 프레임 작성 시에는 필요한 조건에 맞는 값을 찾아내는 작업이 포함
    
- 각 데이터 프레임을 합칠 때에는 메타데이터가 포함되어야 함
    

12-3.데이터 프레임 작성 연습과 점수 관리

- 실제 데이터를 바탕으로 데이터 프레임 작성 연습이 필요
    
- 각각의 데이터를 성공적으로 작성하고, 작업을 완수할 경우 점수를 증가
    
- 점수는 강의 복습과 문제 풀이를 통해 쌓아가며, 성공적인 시험을 위한 핵심 요소
    
- 강의에서 진행하는 문제는 중복 학습을 방지하기 위해 고유한 문제를 사용
    
- 강의 복습과 문제 풀이를 통해 점수를 올리는 것이 목표
    

13.데이터 프레임

13-1.데이터 프레임과 행, 열

- 데이터 프레임은 현실의 데이터를 컴퓨터에 저장하기 위한 자료형임
    
- 데이터 프레임의 개수를 '구조체의 단위'라고 부름
    
- 구조체의 단위는 '행'이라고 불리기도 하고, '열'이라고도 불리기도 함
    
- 데이터 프레임에서 '행'은 인스턴스, '열'은 '피처'라고도 부름
    
- 행과 열의 개수를 고려하여 데이터 프레임을 재구성해야 함
    

13-2.파생변수

- 데이터 프레임의 개수를 일괄적으로 나눔으로써 새로운 파생변수를 만들 수 있음
    
- 파생변수는 기존 데이터에서 새로운 속성을 추가한 것임
    
- 새로운 파생변수는 기존 행의 속성에 새로운 값(밀도)을 대입하여 생성함
    
- 데이터의 단위가 맞지 않으면 별도의 처리가 필요함
    
- 라운드하여 값(밀도)을 처리하여 올바른 단위로 삽입함
    

13-3.수도와 인구 출력

- 각 국가의 수도와 인구를 출력하려고 함
    
- 먼저, 행을 '트루 폴스'로 찾고, 열(피처)을 '리스트'로 출력함
    
- 이후, 'D크로스' 함수를 통해 국가의 평균 인구를 구할 수 있음
    
- 기존에는 행 안의 인구만 대입하여 계산했지만, 이제는 전체 인구를 대입하여 계산함
    
- 출력 결과를 수정하여, 각 국가의 인구밀도를 표시함
    

14.파이썬 명령어와 데이터 분석

14-1.데이터 처리와 명령어 사용

- 파이썬에서 데이터를 처리하는 여러 가지 명령어들을 이해함
    
- 컨트리스, 캐피털 등의 데이터 요소를 활용해 데이터를 찾음
    
- 값의 변화를 주기 위해선 행렬 순서대로 인덱싱을 하는 것이 중요함
    
- 명령어를 사용해 원하는 데이터를 찾을 수 있음
    

14-2.데이터 조건에 따른 명령어 사용

- 데이터의 조건에 따라 명령어를 다르게 사용해야 함
    
- 인구밀도가 100 초과 및 300 미만인 국가들을 찾을 때, 인구밀도와 300미만 조건을 모두 만족하는 국가들만 찾을 수 있음
    
- 데이터의 특성에 따라 스트링 명령어 또는 컨테인즈 명령어를 사용해야 함
    

14-3.데이터 결과 해석과 시각화

- 데이터의 결과를 해석하고 시각화하는 방법을 배움
    
- 데이터의 결과가 원본으로 바뀌지 않으며, 원본은 변하지 않음
    
- 시험을 위해 데이터를 정리하고, 필요한 정보만 추출하는 방법을 학습함
    
- 데이터의 특성에 따라 적절한 명령어를 선택하는 것이 중요함

1.데이터 탐색

1-1.데이터 탐색의 중요성

- 데이터 탐색 과정을 통해 데이터를 전처리하고 분석을 위한 준비를 함
    
- 데이터 프레임을 탐색하며 데이터의 구조와 특성을 파악하는 것이 중요함
    
- 탐색 과정을 통해 데이터의 개수, 평균, 최소, 최대, 분산, 표준편차, 중앙값 등을 확인할 수 있음
    
- 시각화를 통해 데이터의 분포를 확인하고, 분석을 위한 파생변수를 만들 수 있음
    
- 데이터 탐색 시, 여러 가지 시각화를 통해 복잡한 데이터를 한눈에 파악할 수 있음
    

1-2.통계적 방법론과 용어

- 통계적 방법론을 통해 데이터를 분석하고, 결과를 도출함
    
- 데이터의 특성에 따라 다양한 분류 체계(클러스터)를 만들 수 있음
    
- 분류 체계는 설문조사나 관찰을 통해 얻은 데이터를 기반으로 함
    
- 데이터의 특성에 따라 적절한 분류 체계를 선택해야 함
    
- 통계적 방법론은 대표성을 갖춘 데이터를 선택하고 추출하는 데 사용됨
    

1-3.통계적 방법론의 활용

- 데이터의 양이 방대할 경우, 통계적 방법론을 통해 효율적으로 분석할 수 있음
    
- 설문조사나 관찰을 통해 데이터를 수집하고, 이를 통계적으로 분석하여 의미 있는 결과를 도출함
    
- 통계적 방법론은 다양한 분야에서 활용될 수 있음
    
- 통계적 분석을 통해, 데이터의 특성을 파악하고 인사이트를 도출할 수 있음
    
- 데이터의 대표성을 갖춘 부분에서 통계적 방법론을 적용하여 분석할 수 있음
    

2.모집단과 표본집단

2-1.모집단과 표본집단

- 통계조사의 목표는 모집단에 대한 정보를 얻는 것임
    
- 모집단의 일부 대표성 있는 표본을 수집하여 모집단을 대표하는 집단을 만듦
    
- 데이터를 수집할 때, 숫자와 변수의 종류에 따라 데이터 정제가 필요함
    
- 수집한 데이터를 코딩하여 리코드로 저장함
    
- 특정 시점을 기준으로 모집단을 분석하는 종단적 연구와 시간을 흐르는 횡단적 연구가 있음
    

2-2.표본집단과 대표성

- 설문지를 통해 모집단을 대표하는 표본 집단을 찾음
    
- 대표성을 위해 설문지를 구간별로 나누어 수집함
    
- 예를 들어, 20대 초반에 1만 장의 설문지를 나누고 20대 중반부터 20대 후반까지 800여 장씩 나눔
    
- 데이터의 대표성을 확보하기 위해 특정 시대와 지역에 집중함
    
- 표본집단의 대표성을 확보하기 위한 고민과 탐색이 중요함
    

2-3.데이터 탐색과 통계학

- 통계학은 가설을 세우고 검증하는 방식으로 연구를 진행함
    
- 머신러닝은 새로 들어온 데이터의 경향성을 찾는 인추적 학습을 목표로 함
    
- 인추적 학습을 통해 새로 들어온 데이터가 잘 맞출 수 있도록 함
    
- 통계학과 머신러닝의 EDA는 데이터의 정제와 목표가 다름
    
- 통계학은 과거 데이터를 바탕으로 경향성을 설명하는 것, 머신러닝은 새로운 데이터의 경향성을 찾는 것임
    

3.머신러닝과 통계학의 상호관계

3-1.머신러닝의 목표와 통계학의 목표

- 통계학의 목표는 데이터를 바탕으로 가설을 검증하고 추정하는 것임
    
- 머신러닝의 목표는 가지고 있는 데이터를 바탕으로 새로운 패턴을 파악하거나 통계 모형을 만드는 것임
    
- 통계학은 데이터의 패턴을 분석하고, 머신러닝은 이 패턴을 기반으로 새로운 통계 모형을 만듬
    
- 머신러닝은 과거 데이터를 바탕으로 미래 데이터를 예측하고, 통계학은 과거 데이터를 분석하여 새로운 패턴을 찾음
    
- 머신러닝에서 데이터 분석은 확장적인 추론 통계학을 사용함
    

3-2.머신러닝과 통계학의 상호관계

- 통계학에서 머신러닝은 데이터의 패턴을 찾기 위해 확장적인 추론 통계학을 사용함
    
- 머신러닝은 복잡한 패턴을 파악하기 위해 데이터를 다양한 관점에서 분석함
    
- 통계학은 데이터의 패턴을 분석하여 가설을 검증하고, 머신러닝은 이 가설을 기반으로 새로운 패턴을 찾아냄
    
- 이 과정에서 통계학과 머신러닝은 서로 상호작용하며 발전하고 있음
    
- 머신러닝과 통계학은 서로를 보완하며 사용되며, 각각의 장점과 단점을 가지고 있음
    

3-3.머신러닝과 통계학의 응용

- 머신러닝과 통계학의 상호작용은 빅데이터 분석에 이르기까지 응용되고 있음
    
- 빅데이터 분석은 AI와 빅데이터의 등장에 따라 더욱 중요해짐
    
- 머신러닝은 데이터를 기반으로 새로운 패턴을 찾아내고, 통계학은 이를 검증하며 가설을 검증함
    
- 이런 방식은 과거 데이터를 바탕으로 미래를 예측하고, 이를 통해 통계학은 새로운 패턴을 찾는 것을 목표로 함
    
- 이러한 접근법은 빅데이터 분석의 중요성을 더욱 강조하고, 이를 통해 더 나은 의사결정을 가능하게 함

- 데이터의 패턴을 발견하고 의미 있는 정보를 도출하는 것이 중요함
    

5.데이터분석 방법론

5-1.데이터의 탐색과 기준 설정

- 다양한 차원에서 데이터를 탐색하고 기준을 설정해야 함
    
- 카드 소비 패턴 분석 시, 사용자의 성별, 연령대 등 기준으로 데이터를 쪼개볼 수 있음
    
- 남성, 여성별 소비 패턴 차이를 20대, 30대, 40대 등 연령대별로 나누어 봄
    
- 전체 데이터를 무작위로 선택해 조합하여 새로운 패턴을 발견할 수 있음
    
- 가설을 설정할 때, 현실 세계의 데이터는 시간과 공간의 연결고리를 가짐
    

5-2.데이터 분석의 주의점

- 데이터 분석 시, 상관관계와 인과관계를 고려해야 함
    
- 상관관계는 패턴 간의 영향을, 인과관계는 원인과 결과의 관계를 의미함
    
- 가설을 세울 때, 데이터가 보여주는 패턴을 분석하여 새로운 인과관계를 발견할 수도 있음
    
- 다양한 컬럼을 조합하여 원하는 목표를 달성하는 것이 중요함
    
- 이상치는 분석의 결과에 큰 영향을 줄 수 있으므로 주의해야 함
    

5-3.시각화 도구의 사용과 중요성

- 3차원 그래프는 데이터를 직관적으로 전달할 수 있지만, 문화적 차이를 고려해야 함
    
- 중요한 데이터를 먼저 보고하고, 그다음 중요한 순서로 늘어놓는 것이 효율적임
    
- 시각화 도구를 사용할 때, 데이터의 중요도에 따라 차순으로 놓는 것이 좋음
    
- EDA의 목표는 남들의 분석 시간을 단축시키는 것임
    

6.에듀 및 시각화

6-1.시각화 과정

- 중요도에 따라 우선순위에 시각화 시 고려함
    
- EDA 시 누군가에게 보고할 때, 중요도가 낮은 항목부터 보고하고, 중요도가 높은 항목부터 사라지게 함
    
- 다양한 시각화 툴을 사용해 데이터를 시각화하고, 그중 일부만 눈에 들어옴
    
- 데이터를 수집하는 과정은 매우 중요하며, 여러분의 역할은 데이터를 수집하는 것임
    
- 여러분은 수집된 데이터를 시각화할 수 있는 툴을 사용해야 함
    

6-2.대시보드와 시각화

- 대시보드는 다양한 데이터를 한 화면에 비교하고, 주 시계열 그래프를 통해 인사이트를 제공함
    
- 대시보드 화면에는 주로 서비스 성과를 측정하는 지표들을 표시함
    
- 각 서비스마다, 혹은 부서마다 중요한 지표들이 다름
    
- 같은 카카오톡이라도 부서에 따라 접속자 수, CPU 사용량 등 다른 지표가 중요할 수 있음
    
- 수집된 데이터를 다양한 용어로 부름
    

6-3.질적 변수와 명목형 변수

- 1과 2를 더해서 나누기 2로 평균을 구하면 1.5가 됨
    
- 평균을 구할 때, 값이 아니라 값을 대변하는 실제 값이 의미를 가짐
    
- 1, 2, 3을 더해서 나누기 3로 평균을 구하면 2.5가 됨
    
- 반대로, '교육 수준'은 값 자체는 의미가 없지만, 순서는 의미가 있음
    
- 따라서, '고졸', '중졸', '대졸 이상' 등으로 나누어 '교육 수준'의 평균을 구할 수 있음
    

7.명목/순서/양적 변수

7-1.명목/순서형 변수

- 어떤 크기나 경향성은 없지만, 순서성을 가지고 있다는 의미의 변수는 순서형 명목형 변수라 부름
    
- 사람마다 만족도 평가를 예로 들어, "괜찮다", "나쁨"과 같은 주관적 표현은 순서형 명목형 변수에 해당함
    
- 숫자로 표현할 수 없는 순서성을 가진 변수는 순서형 양적 변수라 부름
    
- 평균을 포함한 대표값들은 순서형 양적 변수라 볼 수 있음
    
- 전체 중에서 몇%가 그러한 생각을 했는지 판단할 때는 순서형 변수를 사용함
    

7-2.양적 변수

- 숫자 자체에 의미가 있는 양적 변수와 그렇지 않은 양적 변수를 구분함
    
- 연속형 변수를 임의로 이산형 구간으로 나누어 사용하기도 함
    
- 신발 사이즈와 시간 등의 연속형 변수를 1분 단위 또는 1초 단위로 구분하여 이산형 양적 변수로 사용함
    
- 양적 변수는 덧셈, 뺄셈과 같은 연산을 통해 값이 의미를 가짐
    
- 질적 변수와 달리 양적 변수는 연속형이면서 구간이나 이산형으로 나누어 사용됨
    

7-3.데이터 시각화

- 시각화 툴인 네트플러이브를 이용해 데이터를 시각화함
    
- MATLAB(맥플러리)을 플롯 기능을 갖춘 라이브러리로, 파이썬 언어를 이용한 데이터 분석 프로그램임
    
- 태어나기 전부터 다양한 그래프를 접했기 때문에 직관적으로 데이터 시각화를 하기로 함
    
- 시각화를 할 때, 의도하는 바를 명확히 표현하기 위해 네트플러이브를 사용함
    

8.데이터 시각화

8-1.데이터 시각화 도구의 이해

- 시각화 도구를 활용해 데이터를 시각화함
    
- MATLAB에서 불러온 패키지는 한국어 폰트가 깔려 있으면 사용 가능함
    
- 패키지를 불러올 때 한국어 폰트가 안 깔려 있는 경우 홈트 매니저 패키지를 같이 불러옴
    
- 시각화 도구로써 업그레이드된 산업망, 산업망, 산업망2, 산업망3 등을 사용함
    
- 데이터 시각화는 객체 지향 프로그래밍으로, 먼저 도형을 그리고 실제 좌표를 넣어 그림을 그림
    

8-2.파이썬 패키지 활용

- 시각화 도구로서 파이썬 패키지의 장점을 살려 사용함
    
- 논문에서 많이 사용되는 패키지인 '피플롯'을 포함함
    
- '피플롯' 패키지는 기본 그래프 도형에서 좌표 입력을 통해 데이터를 시각화함
    
- 데이터를 입력하면, 결과만 그래프로 보여주는 간단한 시각화 방법을 제공함
    
- '도화지' 객체에 'X축', 'Y축' 좌표를 입력하면, 해당 데이터의 시각화된 도화지가 생성됨
    

8-3.데이터 시각화

- 데이터 시각화에서 '도화지' 객체에 '좌표 입력' 기능을 통해 데이터를 입력함
    
- 좌표 입력 기능을 통해 원하는 점들을 생성할 수 있음
    
- '도화지' 객체에 '마커' 파라미터를 입력하면, 좌표의 값을 수정할 수 있음
    
- 데이터 시각화에서 '도화지' 객체의 '시각화', '모양', '버호', '색상'을 수정할 수 있음
    
- 데이터 시각화에서 데이터를 입력하면, 결과만 그래프로 보여줌
    

9.그래프 만들기

9-1.그래프 그리기 개요

- 그래프 만들기 위해선 좌표와 마커, 선 등이 필요함
    
- 마커는 그래프의 각 점에 표시되어 좌표와 선에 대한 정보를 제공함
    
- 선 스타일, 라인 스타일 등의 속성으로 점선이나 마커를 디자인할 수 있음
    
- 여러 개의 좌표를 한꺼번에 찍을 때는 색깔을 바꾸거나 선을 지정하는 등의 세부 사항을 설정할 수 있음
    
- 랜덤한 값을 생성해 다양한 좌표를 설정하고 여러 개를 찍어 그래프에 그려줄 수 있음
    

9-2.컬러 사용법

- 컬러는 레드, 초록, 파란색 등의 색상을 16진수로 변환한 것임
    
- RGB 색상은 0에서 255 사이의 명암으로 표현되며, 이는 빛의 삼원색으로도 표현 가능함
    
- 매틀랩에서 제공하는 다양한 색깔 이름과 코드를 활용하여 컬러를 지정할 수 있음
    
- 연구에서는 색깔이 중요하므로 컬러의 순서를 지정하는 것이 중요함
    
- 매틀랩에서 제공하는 색깔 지정 방법은 라인 스타일, 마커, 컬러 순서로 작성함
    

9-3.그래프 제목과 외부 자료

- 그래프에 제목과 외부 자료를 추가할 수 있음
    
- 그래프의 제목은 X축과 Y축의 제목을 포함해 다양한 방법으로 지정할 수 있음
    
- 그래프의 모서리나 선에 상징을 그어 설명하는 것도 가능함
    

10.파이썬을 활용한 그래프 작성과 수정

10-1.그래프 작성과 레이블

- 파이썬을 활용하여 그래프를 작성할 수 있음
    
- 그래프의 축에 제목을 붙이는 '최적화' 기능을 활용함
    
- 각 그래프에 대해 설명을 제공하기 위해 '표시' 함수 사용
    
- 그래프의 '제목'에 대한 확인과 수정이 가능함
    
- 그래프의 축을 설정하고 각 축에 맞는 레이블을 지정함
    

10-2.그래프 수정과 레이블 지정

- 원하는 바에 따라 그래프를 수정하고 레이블을 지정할 수 있음
    
- 축의 최솟값과 최댓값을 사용하여 그래프의 편리성을 높일 수 있음
    
- 그래프의 레이블은 '최종 결과'에 대한 표시를 담아야 함
    
- 그래프의 제목은 각각의 값들을 의미하도록 설정해야 함
    
- 본문에서 설명한 그래프 작성과 수정 방법을 활용하여 자신의 그래프를 만들 수 있음
    

10-3.그래프 분리 및 화면 구성

- 여러 개의 그래프를 만들고 싶을 때 '보여주기' 명령어를 활용함
    
- 여러 개의 그래프를 하나의 그래프로 만들 수 있으며, 이는 '분리' 명령어로 가능함
    
- 분리 명령어를 활용하면 각 그래프는 독립적으로 실행할 수 있음
    
- 파이썬을 이용한 그래프 생성과 수정은 본문에서 배운 내용을 기반으로 진행할 수 있음
    
- 시각화 프로그램은 특정 명령어를 통해 그래프를 생성하고, 이를 통해 정보를 시각화할 수 있음
    

11.시각화 패키지 활용과 폰트 설정

11-1.시각화 패키지의 활용과 오타 해결

- 시각화 패키지의 다양한 옵션을 활용하여 사용자가 의도하는 방식으로 데이터를 시각화할 수 있음
    
- 오타 메시지를 통해 사용자의 시도에 대한 시스템의 반응을 이해할 수 있음
    
- 예상치 못한 오류가 발생했을 때, 시스템이 어떻게 판단하고 대응하는지 파악하는 것이 중요함
    
- 사용자가 입력한 옵션에 대한 정확한 이해와 사용이 요구됨
    

11-2.시각화 패키지의 통계적 지표 생성과 활용

- 시본이라는 통계 함수를 내장하고 있는 시각화 패키지로 데이터를 시각화할 수 있음
    
- 데이터를 보여주는 방식은 통계적 지표를 활용하여 파악하는데, 이를 활용한 시각화 패키지 사용의 장점 파악 가능
    
- 시각화 패키지는 한 줄의 시각 정보를 제공하는 데 있어 핵심적인 역할을 함
    
- 시각화 패키지의 사용을 통해 데이터를 한눈에 파악하고 이해하는 데 도움을 줄 수 있음
    

11-3.폰트 설정과 그 활용

- 한글이 깨져 있는 경우, 폰트 설정을 수정하거나 설치할 수 있음
    
- 폰트 설정에 필요한 파라미터들을 확인하고 수정하며, 사용자가 원하는 폰트를 선택할 수 있음
    
- 한글을 지원하지 않는 폰트는 사용자의 입력에 따라 설치될 수 있음
    
- 사용자들이 원하는 폰트를 선택하고 이를 시각화 패키지에 적용할 수 있음
    
- 이 과정을 통해 한글이 깨진 컴퓨터에서도 시각화 패키지를 효과적으로 사용할 수 있음
    

12.그래프 작성법

12-1.시각화 파라미터

- 프레임, 테두리, 위치, 스타일 등의 파라미터를 사용해 그래프를 만듦
    
- 위치 파라미터를 지정하면, 그래프의 중심에 위치시킴
    
- 스타일 파라미터를 지정하면, 그래프의 모양을 결정함
    
- 여러 개의 범주형 변수나 질적 변수를 시각화할 때 막대그래프를 사용함
    
- 코랩에서 제공하는 막대그래프 툴을 사용해 그래프를 만듦
    

12-2.그래프 그리기

- 먼저 범주형 변수나 질적 변수의 개수를 정하고, 그래프의 위치와 자리를 지정함
    
- 개수를 막대그래프의 개수 레이아웃에 따라 넣어주고, 라벨에 색깔과 번호를 매겨줌
    
- 순서를 지정하면, 그래프의 각 항목을 그 순서대로 표시할 수 있음
    
- X축과 Y축의 제목을 지정하면, 각 축의 이름을 표시할 수 있음
    
- 라벨의 개수에 따라 라벨의 색깔과 번호를 지정함
    

12-3.그래프 종류

- 시각화 함수에 따라 그래프의 종류가 달라짐
    
- 시각화 함수에서 제공하는 막대그래프 툴을 사용하면, 범주형 변수나 질적 변수를 시각화할 수 있음
    
- 막대그래프를 사용하면, 개수를 세어 전체 개수를 나타내는 그래프를 만듦
    
- 코랩에서 제공하는 막대그래프 툴을 사용하면, 시각화 함수에 따라 다양한 그래프를 만들 수 있음
    

13.데이터 시각화 방법과 그래프 이해

13-1.데이터 시각화의 중요성과 그래프의 위치

- 데이터를 시각화하는 것은 복잡한 정보를 쉽게 이해하고 기억하게 함
    
- 그래프는 데이터를 시각화하는 데 필요한 도구로, 정확한 위치와 값의 표시가 필요
    
- 그래프의 위치를 잘 살펴보는 것이 중요하며, 각 데이터의 위치와 값이 시각화된 그래프에 정확하게 표시되어야 함
    
- 그래프의 위치를 조정하면, 데이터의 순서와 상관관계를 이해하는 데 도움이 됨
    
- 주의해서 볼 필요가 있는 부분은 4번 위치와 6번 위치로, 두 개의 바에 덮인 데이터의 위치를 확인하는 것
    

13-2.그래프의 시각화와 데이터 손실

- 그래프는 여러 개의 데이터를 한 번에 표현할 수 있음
    
- 순서대로 그래프를 그려놓으면, 뒤에 있는 값이 앞에 있는 값보다 앞서 오버랩 되는 현상이 발생할 수 있음
    
- 이는 시각화 과정에서 발생하는 현상으로, 좌표의 대인척을 확인하면 해결 가능
    
- 데이터의 손실이 발생할 수 있으므로, 그래프를 그릴 때 주의해야 함
    

13-3.데이터 분포를 시각화하는 방법 - 국, 도수, 사과

- 특정 구간으로 연속형 변수들을 나눠 데이터의 빈도를 막대그래프로 표시할 수 있음
    
- 누적합을 이용하여, 천 개의 데이터가 특정 구간에 몇 개 있는지 볼 수 있음
    
- 시각화 도구를 이용하면, 천 개까지 도달하는 동안 특정 구간에 몇 개가 있었는지 확인 가능
    
- 숫자가 나열된 도표나 데이터를 살펴보는 것이 중요함
    

14.스태플롯

14-1.스태플롯 이해

- 키와 몸무게 등 양적 변수로 좌표 평면에 늘어놓으면 분포 형태 파악 가능함
    
- 몸무게가 키보다 큰 사람들은 대부분 해당 경향 보임
    
- 이상치 제거 시 평균값 사용, 분포와 함께 이상치 분포도 함께 보여줌
    
- 스택플롯은 같은 값 공유 플롯에서 오버랩 해소하고 색깔 달리해 2차원 데이터 볼 수 있게 해줌
    
- 파이플롯과 달리 캐나다 값이 더 많으면 별로 의미 없음
    

14-2.스태플롯 예시

- 16대, 17대, 18대 투표율 비교를 스태플롯으로 설명 가능
    
- 각 진영별 투표율과 전체 투표율 비교 가능
    
- 특정 시점의 총합과 평균 투표율 등도 파악 가능
    
- 파이플롯은 비율을 알 수 없으므로 시각화에 한계 있음
    

14-3.기타 분석 방법

- 양적 변수와 질적 변수의 비율을 파악할 때는 파이플롯 사용
    
- 분석하려는 비율이 많으면 의미 없으므로 간결한 시각화가 중요
    
- 비슷한 값들이 많으면 표로 그리는 게 훨씬 좋음
    
- 오행으로 자리 배정, 팀 명칭과 각 팀원 이름으로 자리 만들기 진행
