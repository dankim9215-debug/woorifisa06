1.문자열 다루기

1-1.문자열을 이용한 데이터 처리

- 데이터 처리에 있어 문자열을 이용한 방법이 중요함
    
- 모델 개발 시 수집된 데이터를 활용하여 모델 학습 및 데이터베이스 생성
    
- 문자열을 자료형으로 변경하고, 여러 내장 메서드를 활용하여 처리함
    
- 문자열을 특정 문자 또는 문자열 기준으로 잘라주거나 순서를 바꿔주는 등 다양한 기능이 있음
    
- 문자열을 리플레이스하여 원하는 값으로 대체하는 등 문자열을 조작하는 여러 방법이 있음
    

1-2.문자열을 활용한 전처리

- 특정 문자열에서 중복 단어를 찾아 빈도수를 세고, '해피'라는 노래 가사가 몇 번 나왔는지 검색해 봄
    
- 대문자와 소문자, 영어권 조를 혼동하지 않도록 주의함
    
- 특수문자(=, ~ 등)도 검색할 때 소문자로 처리하여 포함 여부를 판단함
    
- 중괄호, 콤마 등의 기호가 포함된 문자열은 소문자로 인식하여 처리함
    
- 데이터 처리 시, '고객'이라는 노래 가사가 몇 번 나왔는지 등 특정 문자열을 찾는 전처리 과정이 필요함
    

1-3.정규식을 이용한 데이터 탐색

- 데이터의 패턴을 찾을 때, 정규식을 사용하여 유사한 패턴을 치환하여 탐색하는 방법이 있음

- '정규식'이라는 단어를 알면 정규식 패턴을 이용한 문제 해결이 더 쉬워짐
    
- 파이썬에서 정규식 패턴을 찾기 위해 '레덱스' 패키지를 사용함

2.캡처 패턴

2-1.기본 패턴 구성

- 대괄호 안에 소문자로 시작하는 문자열을 포함해 조건문 작성 가능함
    
- 특정 문자열을 반복하여 포함하는 패턴을 캡처할 때는 '뒤에 있는 값을 대괄호로 포함해 캡처'를 사용함
    
- 패턴의 시작 문자열을 포함한 모든 문자열을 포함하는 방식도 가능함
    
- '뒤에 있는 값'을 포함해 캡처할 때는 '뒤에 있는 문자열을 대괄호로 포함해 캡처'를 사용함
    
- 캡처할 문자열이 하나라도 포함되어 있어야 함
    

2-2.문자열에 포함된 패턴 캡처

- 숫자는 0부터 9까지의 모든 문자열을 포함함

- 대괄호 안에 포함된 패턴이 하나라도 포함되어 있어야 함
    

2-3.숫자와 문자열에 대한 캡처

- 숫자를 포함한 패턴을 캡처할 때는 '숫자의 개수만큼 패턴을 캡처'를 사용함

4.정규식 찾기

4-1.정규식 검색

- 데이터에서 특정 패턴을 찾을 때, DIR 모듈의 재스트링을 사용함
    
- 문자열에서 정규식 패턴과 일치하는지 검색하는 매치 함수를 찾을 수 있음
    
- 전체 문자열을 검색해 정규식과 일치하는지 검색하는 함수도 있음
    
- 매치, 검색, 찾기, 찾어질, 서브스트림을 이용한 정규식 검색 방법을 이해해야 함
    
- 컴파일을 통해 패턴을 작성하고, 재스트링을 통해 원하는 문자열을 찾을 수 있음
    

4-2.정규식 매치

- 정규식 패턴을 컴파일한 후, '매치' 명령어로 값을 찾아줌
    
- '매치' 명령어에 따라 패턴이 일치하는지 확인함
    
- A에서 Z, 혹은 M문자 대문자 A에서 Z, 일치하지 않는 문자열을 찾음
    
- 패턴이 처음부터 일치하지 않으면, 아무것도 리턴하지 않음
    
- 대소문자 안 가리고 찾는 옵션도 있음
    

4-3.정규식 검색 방법

- '검색' 명령어는 패턴과 일치하는 첫 번째 패턴을 찾음
    
- '찾기' 명령어는 패턴을 찾은 후, 그 위치를 저장함
    
- '서브스트림'은 패턴을 바꾸어 찾는 것과 같음
    
- 재스트링은 순서대로 반복 가능한 객체를 만듦
    
- 순서, 위치, 요소, 메모리에 저장된 경우를 이해해야 함

- 패키지는 여러 개의 모듈이 모여 있는 것을 의미하며, 이를 한꺼번에 불러오는 방법을 '패키지'라 함

- 모듈의 모든 기능을 가져올 수 있지만, 적절한 방법을 고려해야 함
    
- 모듈명을 사용할 때는 덮어쓰기를 피해야 함
    
- 모듈의 위치를 지정해 두고 항상 그 위치에서 가져오도록 함
    

13-2.모듈 불러오기의 구체적 방법

- 구글 드라이브에 맞춰 구글 드라이브를 마운트함
    
- 구글 드라이브에 저장된 파일을 다른 컴퓨터로 옮겨도 남아 있는 파일을 다시 마운트할 수 있음
    
- 모듈이 컴퓨터에 연결되어 있는지 확인해야 함
    
- 모듈이 연결되어 있는지 확인하려면 구글 드라이브에 접속해서 " mount"을 하면 됨
    
- 모듈의 위치를 지정해 두고 항상 그 위치에서 가져오도록 함
    

13-3.모듈 불러오기의 실행

- 구글 드라이브에 있는 모듈 파일을 직접 불러오기 전에 컨텐츠 드라이브에 저장된 모듈 파일을 먼저 불러옴
    
- 직접 불러오지 않고 다른 저장된 모듈 파일을 불러오면, 해당 모듈이 저장된 디렉토리에 있는 파일을 그대로 사용하게 됨
    
- 모듈의 함수를 불러오는 방법은 직접 불러오기보다는 모듈명을 사용하여 불러오는 방법이 더 적절함
    
- 모듈명을 사용하여 불러오면, 모듈이 저장된 디렉토리에 있는 파일을 그대로 사용하게 되므로 모듈의 함수를 직접 불러올 때는 모듈명을 사용해야 함

1.예외 처리

1-1.예외와 그 종류

- 프로그램이 의도한 대로 동작하지 않으면 예외 발생함

- 예외 발생 시 처리할 수 있는 방법은 't라이(시도하다)' 예외 처리와 '트라이(트라이하다)' 예외 처리라 함
    
- 문법 오류는 't라이(시도하다)' 예외 처리로, 개발자가 수정할 수 있음
    
- 신텍스 오류는 입구까지 가는 과정에서 발생하므로 예외 처리 불가함
    

1-2.예외 처리 방법

- 예외 발생 시, 에러가 발생한 경우를 확인한 후 '처리할 수 있음'을 알려줌
    
- 간단한 에러는 예외 처리하지 않고 잘 넘기도록 함
    
- 복잡한 코드의 경우, 예외를 다루기 전에 상황에 대한 대처 방안을 적어둠
    
- 현실적으로 모든 경우를 고려하기 어려우므로, 예외를 피하는 것이 중요함
    
- 파이썬에서는 예외를 최소화하고, 인터프리터 언어의 장점을 활용하여 예외 처리를 권장함
    

1-3.예외 처리의 구현

- 간단한 예제에서, 사용자로부터 값 입력받고 조건 만족 여부를 판단함
    
- 입력받은 값이 0 이상의 정수여야 하므로, 조건에 따라 문자열로 출력함

- 예외 처리를 통해 프로그램의 성공 여부를 판단하고, 이를 바탕으로 다음 단계를 결정함
    
- 예외 처리 시에는 모니터링과 관리가 필요하며, 이를 통해 프로그램의 문제점을 파악하고 개선할 수 있음
    

6-3.예외 관리의 중요성

- 예외 관리는 프로그램의 안정성과 신뢰성을 보장하는 중요한 과정임
    
- 예외 처리를 통해 프로그램이 오류를 자동으로 알려주고 대응하도록 함
    
- 예외 관리가 잘 되면, 프로그램의 성공률이 높아지고, 안정성이 향상됨
    
- 예외 관리는 디버깅과 로그 작성에도 필수적임
    
- 예외 처리를 통해 프로그램의 문제점을 파악하고 개선할 수 있음
    

7.프로그래밍 에러 처리와 예외 관리

7-1.프로그래밍 에러의 발생과 대응

- 프로그래밍 에러는 코드의 동작이 잘못된 경우 발생함
    
- 에러 발생 시 가독성 측면에서 문제를 찾아내는 것이 중요함
    
- 프로그래밍 에러는 종류별로 다른 대응 방식이 있음
    
- 예외 처리를 통해 에러 발생 시 적절한 행동을 취할 수 있음
    
- 각각의 예외에 대한 대응은 프로그래밍 언어마다 다름
    

7-2.예외 처리의 필요성과 방법

- 프로그램은 에러 없이 잘 실행되면 예외 처리 없이도 작동함
    
- 예외 처리를 통해 프로그램이 에러를 발생시키는 경우를 미리 대비할 수 있음
    
- 예외 처리를 통해 프로그램의 성능과 안정성을 향상시킬 수 있음
    
- 모든 프로그래밍 언어는 예외 처리를 포함하고 있음
    
- 각 언어마다 예외 처리를 위한 메소드나 기능이 다름
    

7-3.복잡한 문제를 해결하는 예외 처리

- 복잡한 문제를 해결하는 데도 예외 처리가 필요함
    
- 예외 처리를 통해 프로그램이 안정적으로 작동하도록 도와줌
    
- 파이썬처럼 불완전한 기능도 예외 처리를 통해 보완 가능함
    
- 프로그래밍 언어마다 특성에 맞는 예외 처리를 선택해야 함
    
- 예외 처리는 프로그래밍의 필수적인 부분임
    

8.예외처리와 객체지향 프로그래밍

8-1.예외처리와 레이즈 이해

- 예외처리를 통해 코드의 복잡성을 줄이고, 오류 발생 시에 원활하게 대응할 수 있음
    
- 레이즈라는 키워드를 사용해 강제로 예외를 발생시킬 수 있음
    
- 입력값이 스트링일 경우 타입 에러를 일으켜줌
    
- 입력값이 리스트일 경우, 타입 에러를 일으킨 후 예외를 생성하여 출력함
    

8-2.예외 처리를 통한 코드 간결성

- 예외처리를 통해 코드를 간결하게 만들 수 있음
    
- 원하지 않는 오류 발생을 줄여 코드의 신뢰성을 높임
    
- 리스트의 데이터에 대한 인덱스를 지키기 위해 코드 변경이 필요함
    
- 개선된 코드로 기존의 코드보다 많은 양의 코드를 줄일 수 있음
    

8-3.객체지향 프로그래밍과 개발 장점

- 객체지향 프로그래밍은 개발자의 개입을 최소화하고, 코드의 재사용성을 높임
    
- 객체지향 프로그래밍 언어는 개발자가 코드를 조합하고 변형하는 역할을 함
    
- 추상화를 통해 기존의 복잡한 코드를 단순화하고, 개발 과정을 개선함
    
- 객체지향 프로그래밍 언어는 유지 보수가 용이하고, 코드의 재사용성이 높음
    

9.객체지향 프로그래밍의 특징과 장점

9-1.객체지향 프로그래밍의 개요

- 객체지향 프로그래밍은 실 세계 문제를 모형화하고 표현함
    
- 구체적 객체를 모형화하여 각 객체가 가진 특징을 분석하고 추출함
    
- 개발자의 역할이 줄어들어 코드 작성에 집중할 수 있게 됨
    
- 객체 지향 프로그래밍은 강의나 사례 연구를 통한 실습이 필요함
    
- 프로그래밍 언어인 파이썬을 통해 실제 문제를 해결해보는 경험이 중요함
    

9-2.객체 지향 프로그래밍의 장점

- 객체 지향 프로그래밍은 공통적으로 관리할 수 있는 객체를 만듦
    
- 서로 다른 객체들을 공통적으로 다룰 수 있어 코드 재사용이 용이함
    
- 한 개의 클래스에서 여러 개의 자식 클래스를 생성하여 복잡한 문제를 해결함
    
- 개발자의 역할이 줄어들어 효율성이 높아짐
    
- 코드의 재사용성으로 인해 유지 보수가 용이함
    

9-3.객체 지향 프로그래밍의 특징

- 객체 지향 프로그래밍은 다형성, 캡슐화, 은닉성 등 특징을 가짐
    
- 다형성은 객체의 속성이나 기능이 맥락에 따라 다른 역할을 수행하게 함
    
- 캡슐화는 객체의 내부를 공개하지 않아 외부에서의 정보 유출을 방지함
    
- 은닉성은 내부를 숨기고 외부에서 보호받는 특징을 가지고 있음
    
- 다형성, 캡슐화, 은닉성 등의 특징은 객체 지향 프로그래밍의 핵심적인 요소임
    

10.객체지향 프로그래밍과 클래스 이해

10-1.객체지향 프로그래밍의 이해

- 객체지향 프로그래밍은 개발자가 클래스 안에 어떤 속성, 동작을 만들어 놓는지에 관계없이 클래스에 리스트를 만들 수 있음
    
- 객체가 개발자가 만든 함수들을 이용해 속성들을 그대로 가져다 쓸 수 있음
    
- 클래스 안에 있는 함수들을 통해 외부에서 그 동작 원리를 이해할 필요 없이도 그대로 사용 가능
    
- 사용자 인스턴스의 비밀번호를 관리하는 예시를 통해 설명함
    

10-2.캡슐화와 객체의 역할

- 캡슐화는 객체 지향 프로그래밍에서 사용되며, 사용자에게 보여주고 싶지 않은 변수들에 접근하지 못하게 함
    
- 개발에서 클래스를 만드는 이유는 같은 동작들을 수행해야 할 때, 클래스를 통해 특정 자료들을 관리하기 위함임
    
- 파이썬에서 클래스는 기본적으로 객체로 상위 클래스를 만들어 내고, 여기에 필요한 기능들을 추가해 생성
    
- 클래스는 붕어빵 틀과 같은 틀로 비슷한 형식을 가지고 같은 동작들을 수행하며 구체적인 내용은 다름
    

10-3.클래스와 객체의 관계

- 클래스를 만들 때는 기본적으로 클래스를 상속 받아 기존 클래스에서 필요한 기능들을 추가함
    
- 클래스를 통해 하나하나의 구체적인 내용들을 다르게 작성하며, 이를 통해 각 객체가 다른 속성과 동작을 가짐
    
- 개발자들이 클래스를 만들 때, 클래스는 기본적으로 객체로 상위 클래스를 가지고, 그 상위 클래스에서 필요한 기능들을 추가해 생성
    
- 각 객체는 그 상위 클래스에서 물려받은 기능들을 가지고, 그 기능들을 통해 특정 동작을 수행함
    

11.클래스와 객체 관리

11-1.클래스와 객체 이해

- 클래스는 기호나 명사로, 객체는 클래스를 통해 생성된 인스턴스를 의미함
    
- 클래스와 객체를 다루는 인터페이스 역할을 하는 생성자 함수를 포함함
    
- 클래스 내에 변수(필드)와 함수(메소드)를 설정할 수 있음
    
- 클래스의 인스턴스를 생성할 때, 인스턴스 변수를 통해 인스턴스의 정보를 관리함
    
- 파이썬의 모듈과 같이, 클래스는 코드의 틀을 제공함
    

11-2.클래스 생성과 인스턴스 관리

- 클래스의 생성을 통해 중고차 회사의 판매 데이터를 관리할 수 있음
    
- 클래스 변수로 회사 이름과 인스턴스 번호를 설정할 수 있음
    
- 인스턴스 함수로 판매 신청서의 인스턴스 번호를 산출함
    
- 파이썬의 생성자 함수와 유사하지만, 인스턴스 생성 시 한 번만 실행되는 특수한 함수임
    
- 생성자 함수를 통해 인스턴스를 생성할 때, 셀프 키워드를 넣지 않아도 됨
    

11-3.데이터 관리와 활용

- 클래스는 데이터를 일관된 방식으로 관리하고, 이를 통해 동작을 제어함
    
- 클래스 내에 데이터를 저장하고, 이를 통해 중고차 회사의 판매 데이터를 조회할 수 있음
    
- 가입 신청서와 판매 신청서의 속성(명사, 명사, 변수)을 통해 데이터를 관리함
    
- 예를 들어, 가입 신청서의 속성(제품명, 판매가, 판매량)을 통해 판매 데이터를 파악할 수 있음
    
- 클래스와 인스턴스를 통해 데이터를 효과적으로 관리하고 활용할 수 있음
    

12.객체지향 프로그래밍과 파이썬의 특징

12-1.객체지향 프로그래밍의 필요성과 파이썬의 특징

- 객체지향 프로그래밍의 필요성과 파이썬의 특징에 대해 설명함
    
- 파이썬은 객체 지향 방식을 강조하면서 프로그래밍 언어로서의 역할에 집중함
    
- 파이썬은 완벽한 객체 지향 프로그래밍이 불가능하다고 주장함
    
- 파이썬은 프로그래밍 언어로서의 역할에 집중하도록 권장함
    

12-2.객체 변수와 클래스 함수의 이해

- 객체 지향 프로그래밍에서 클래스 변수와 클래스 함수의 중요성에 대해 설명함
    
- 클래스 변수는 객체의 일반적인 속성값을 저장하며, 클래스 함수는 해당 변수를 이용하여 정의된 동작을 수행함
    
- 파이썬에서 클래스 함수는 클래스 안에서만 접근 가능하던 이전과 달리 클래스 외부에서도 접근 가능하게끔 개선함
    
- 클래스의 네임이라는 파라미터를 활용하여 인스턴스의 메모리 주소를 지정하도록 설정함
    

12-3.파이썬의 객체지향 프로그래밍과 인스턴스의 문제점

- 파이썬에서 인스턴스 변수의 네임을 지정하는 방식에 대해 설명함
    
- 파이썬에서 인스턴스의 네임이 클래스 변수의 네임과 같으면, 오류 발생 가능성을 지적함
    
- 인스턴스 변수의 네임을 지정할 때, 클래스의 네임과 인스턴스의 네임 중 함수도 마찬가지로 이뤄짐
    
- 파이썬에서는 이를 해결하기 위해 네임을 제대로 설정하도록 권장함
    

13.객체지향 프로그래밍과 클래스 변수

13-1.객체지향 프로그래밍과 클래스 변수의 기본 이해

- 객체지향 프로그래밍은 실제 세계를 모델링하고자 함
    
- 클래스 변수는 객체의 인스턴스를 통한 공유 변수로 사용됨
    
- 클래스 변수의 인스턴스 접근은 클래스의 메소드를 통해 이뤄짐
    
- 인스턴스는 클래스의 설계도로 간주되며, 객체는 인스턴스를 통해 그 설계도로 지어진 집에 비유됨
    

13-2.클래스 변수의 다양한 활용 방식

- 클래스 변수는 인스턴스의 공유 변수와 메소드로 활용됨
    
- 인스턴스 변수로써의 기능과 클래스 메서드로의 기능이 다름
    
- 클래스 메서드의 값은 공유 메모리 주소를 통해 동일하게 사용됨
    
- 따라서 클래스 변수의 변경은 클래스 메서드의 값에 영향을 주지 않음
    

13-3.클래스 외 변수와 메소드의 제한성

- 클래스 외의 변수와 메소드는 클래스의 인스턴스에 접근하지 못하도록 설계되어 있음
    
- 클래스의 설계도를 바꾸어도 인스턴스의 값과 메소드는 영향을 받지 않음
    
- 클래스의 인스턴스는 서로 다른 객체로 간주되어 각각 별도의 메모리에 저장됨
    
- 따라서 클래스 외 변수와 메소드의 사용은 제한됨
