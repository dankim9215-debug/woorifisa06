1.시각화 기술과 데이터 분석

1-1.시각화 기술과 데이터 수집

- 시각화 기술을 통해 데이터를 시각화하여 분석하고 이해함
    
- 한국사람이 한국어를 쓸 수 있게 만들어 놓은 패키지를 사용하여 시각화를 진행함
    
- 인터넷에 놓인 시각화 패키지에서 파이썬으로 옮겨 사용함
    
- 시각화를 진행할 때, 원하는 대로 데이터를 시각화할 수 있음
    
- 시각화 패키지는 기존의 파이썬 패키지 기능과 사람들의 기능을 그대로 넣어놓음
    

1-2.시각화에서 고려해야 할 데이터 분류

- 데이터는 숫자로서의 의미를 가진 지적 변수와 크기로서의 의미가 없는 양적 변수로 나뉨
    
- 지적 변수는 전체 개수 중에서 몇 프로 등의 비율이 의미를 가짐
    
- 양적 변수는 숫자로서의 의미가 없지만, 크기로서의 의미가 있는 변수들을 의미함
    
- 양적 변수의 예로는 건강상태, 소득 등이 있음
    
- 양적 변수의 수집시에는 순서를 부여하지 않음
    

1-3.양적 변수와 질적 변수의 특성

- 양적 변수는 덧셈이나 뺄셈을 하지 않고 연산의 개념을 적용하지 않음
    
- 질적 변수는 크기로서의 의미가 없지만, 의미를 가지는 범주나 순서를 가짐
    
- 질적 변수의 수집시에는 데이터의 의미를 찾는 작업이 필요함
    
- 질적 변수의 예로는 개인의 성별, 지역, 가족구성원 수 등을 있음
    
- 양적 변수와 질적 변수 모두 데이터를 수집하고 이해하는데 필요한 기술임
    

2.데이터 시각화

2-1.연속형 변수와 시계열 데이터

- 시간에 따라 변화하는 습도를 수집하여 분리할 수 있는 변수를 이산형 변수라 부름
    
- 신발 사이즈를 5mm 단위로 측정하여 구간으로 나눔
    
- 연속형 변수를 임의로 구간으로 나누어 측정하기도 함
    
- 측정된 데이터를 시간의 흐름에 따라 시계열 데이터로 표현함
    
- 백화점의 실적 추이를 1년, 1분기, 1분기 단위로 누적 그래프로 표현함
    

2-2.질적 변수와 시각화 기법

- 질적 변수는 개수가 의미가 있고, 3, 1, 2 등의 순서를 통해 퍼센트를 표현함
    
- 비율을 표현할 때 전체를 보도록 하되, 비율이 높은 변수를 함께 보여주는 것을 권장함
    
- 변수를 하나의 차원이 아닌 여러 차원으로 표현할 때, 트리 맵 또는 차원 차례로 표현함
    
- 인구수를 기준으로 전체 인구를 1이라고 했을 때, 그 값을 가진 국가의 인구를 표현할 때는 트리 맵을 사용함
    

2-3.시각화 기법과 데이터 표현

- 시계열 데이터에서 트렌드를 보기 위해 시각화 기법을 사용함
    
- 시간의 흐름에 따른 데이터의 연속성을 표현하기 위해 선을 그어 나타냄
    
- 연속형 변수의 경우, 시간의 흐름을 따르는 그래프를 시각화 기법으로 표현함
    
- 질적 변수에서는 퍼센트를 표현할 때 비중을 고려하여 색으로 표현하는 것을 권장함
    
- 시각화 기법을 통해 다양한 데이터를 시각화하고, 그 패턴을 파악할 수 있음
    

3.그래프와 시각화

3-1.다양한 그래프와 시각화

- 데이터를 시각화할 때 망랑형으로 나타낸 후 위계 분포를 생각해야 함
    
- 하나의 점에 여러 개의 선을 그려 스테이킹 그래프를 만들 수 있음
    
- 스테이킹 그래프는 X축과 Y축으로 한정된 정보를 세로로 표현함
    
- 공간을 세워 점의 변화와 경향을 보여주는 시각화 방법도 있음
    
- 양의 상관관계와 음의 상관관계를 구분하는 지표로 스케터플롯이 있음
    

3-2.양의, 음의 상관관계

- 몸무게와 한 끼의 식사량 간의 상관관계는 음의 상관관계임
    
- 몸무게가 무거운 사람은 칼로리 섭취량이 많음
    
- 몸무게와 식사량이 비슷한 경우는 음의 상관관계가 나타남
    
- 양의, 음의 상관관계가 없는 경우는 절대적인 상관관계가 없음
    
- 상관관계를 파악할 때는 데이터를 정렬해서 표를 만들거나 스케터플롯을 활용함
    

3-3.데이터 모형

- 수집한 데이터를 모형화하여 표현하는 경우가 많음
    
- 모형을 만들 때는 수집한 데이터를 바탕으로 새로운 데이터를 추려낼 수 있음
    
- 데이터의 특성을 분석하여 모형을 만드는 것이 중요함
    
- 모형을 만들 때는 모형의 기능을 고려해야 함
    
- 데이터의 변환과 선택은 모형을 만드는 과정에 영향을 미침
    

4.데이터 시각화

4-1.상관관계와 표준편차

- 데이터 간 상관관계가 높으면 모델의 유의성이 낮아짐
    
- 모델 학습 시 너무 뻔한 정답 질문으로 학습하면 애매한 상황에 맞추지 못함
    
- 통계학에서는 상관관계가 0.5%를 넘으면 있다고 언급함
    
- 표준편차가 작은 데이터일수록 데이터의 특성이 강조되어 버블차트나 가상관계도로 시각화 가능
    
- 데이터가 많지 않을 때는 막대 그래프나 가감형 그래프가 효과적임
    

4-2.산점도와 버블차트

- 데이터의 특성을 이해하기 위해 산점도를 그릴 때 포인트가 많으면 경향성이 강조되어 나타남
    
- 적은 데이터량에서는 버블차트가 효과적임
    
- 버블차트에서는 차원의 크기와 비중을 조합하여 3차원 이상의 요소 상관관계를 표현할 수 있음
    
- 데이터의 특성에 따라 가감형 그래프나 볼록한 그래프가 적합할 수 있음
    
- 3차원 공간에서 차원의 조합을 통해 시각화 가능
    

4-3.시각화 기법

- 잠재적인 변수를 분류하고, 시각화 기법을 사용하여 직관적으로 데이터를 파악할 수 있음
    
- 히스토그램은 평균과 중앙값을 기준으로 데이터 분포를 시각화함
    
- 히스토그램을 통해 중앙값보다 극단적인 값에 영향을 받지 않는 중앙값을 찾을 수 있음
    
- 히트맵은 데이터의 변화를 시간축을 기준으로 나눠 보여줌
    
- 다양한 데이터를 다른 차원으로 시각화하여 효과적으로 이해할 수 있음
    

5.시각화

5-1.시각화의 필요성과 종류

- 시각화는 데이터를 직관적으로 이해하고 기억하기 위해 사용됨
    
- 도화지의 2차원 공간에 많은 정보를 표현해야 함
    
- 직관적 시각화에는 사람의 얼굴 모양, 표정 등이 있음
    
- 데이터를 시각화하여 한눈에 파악할 수 있는 방법들을 고려해야 함
    
- 시각화 방법은 용어, 기술, 예시 등을 사용하여 설명해야 함
    

5-2.시각화의 고려 사항

- 시각화는 데이터를 효과적으로 전달하고 처리하기 위한 파이프라인임
    
- 시각화의 주요 고려 사항은 데이터의 수집, 처리, 표현, 전달, 시각화에 있음
    
- 수집한 데이터를 바탕으로 시각화 파이프라인을 설계해야 함
    
- 시각화는 가독성, 효율성, 정확성, 설득력, 표현력 등을 고려해야 함
    
- 시각화는 데이터의 특성과 목표에 따라 적절한 형태와 표현 방법을 선택해야 함
    

5-3.시각화 도구와 기술

- 시각화 도구는 다양한 프로그램에서 제공되며, 사용 방법은 프로그램에 따라 다름
    
- 프론트엔드에서 제공하는 시각화 도구들을 활용하여 시각화 코드를 작성할 수 있음
    
- 시각화 코드는 샘플 코드를 통해 작성하고, 필요한 부분을 수정하여 사용함
    
- 시각화 패키지에는 서브플롯과 같은 시각화 도구들이 제공됨
    
- 서브플롯은 화면을 나누어 사용하는 개념으로, 여러 개의 플롯을 보여줄 수 있음
    

6.프론트엔드와 그래프

6-1.프론트엔드와 그리드 이해

- 프론트엔드에서 스피드라는 방법을 사용함
    
- 그리드는 화면을 X-Y 체인으로 나누어 사용하는 방식임
    
- X-Y 각 축을 스탠으로 나누어 사용하는 것이 원칙임
    
- 여러 개의 그리드를 사용해 전체 화면을 나타낼 수 있음
    
- 각 그리드에는 좌표 시작점과 그리드의 배열이 포함됨
    

6-2.그리드의 배열과 화면 구성

- 그리드는 하나의 로우를 두 개, 이로써 여러 개의 블루와 2개의 컬럼을 구성함
    
- 이런 방식으로 그리드를 통해 화면을 5등분하고, 각 축에 대해 2등분할 수 있음
    
- 전체 화면을 3개의 등분과 2개의 광고 등분으로 나누어 표시함
    
- 이러한 화면 구성은 그리드의 기본 원칙 중 하나임
    
- 그리드를 통해 여러 개의 로우와 컬럼을 사용해 화면을 구성할 수 있음
    

6-3.플롯과 그래프의 중요성

- 플롯은 데이터를 2D로 나타내기 때문에, 좌표축을 3D로 표현할 필요가 없음
    
- 플롯은 깊이 있는 데이터를 표현하기 보다는 직관적이고 이해하기 쉬운 형태로 나타냄
    
- 웹플러라이브는 플롯보다 자세한 3D 그래프를 제공함
    
- 피본, 넘파이, 판다스 등의 패키지들은 플롯보다 별도로 사용됨
    
- 통계에서 자주 사용되는 로드 데이터셋을 임포트하여 사용함
    

7.데이터 시각화

7-1.데이터 전처리

- 데이터의 분포와 특징을 파악하기 위해 데이터 탐색 및 시각화 활용함
    
- 시각화에는 모델 테스트에도 사용되는 정답과 시각화 목적에 따른 데이터 전처리 과정이 포함됨
    
- 데이터의 성격을 파악하기 위해 유니크한 값 확인, 개수 세기 등의 연습이 필요함
    
- 데이터 분석 시 색깔 지정, 데이터 출력 화면 구성 등 시각화 패키지의 세팅 기능을 활용함
    
- 데이터의 목록형 변수를 시각화 패키지의 시각화 패키지로 표현 가능함
    

7-2.시각화 패키지 활용

- 시각화 패키지에는 데이터를 다루는 여러 기능이 내장되어 있음
    
- 시각화 패키지의 시각화 패키지로 데이터를 시각화하고 파라미터를 설정하여 데이터를 활용함
    
- 데이터의 개수를 세는 등 기본 연산에 대한 내장 함수를 활용하여 간단한 계산이 가능함
    
- 파라미터를 설정하여 원하는 시각화 스타일을 선택하고, 데이터를 불러와 시각화할 수 있음
    
- 파라미터를 조정하여 데이터의 분포와 특징을 확인하고, 분석 결과를 보여줄 수 있음
    

7-3.데이터 시각화의 실습

- 데이터 시각화 패키지의 다양한 기능을 활용하여 데이터를 시각화하고 분석할 수 있음
    
- 시각화 패키지에는 색깔 지정, 데이터 출력 화면 구성 등 다양한 기능이 있음
    
- 데이터를 시각화하고 파라미터를 설정하여 원하는 시각화 스타일을 선택할 수 있음
    
- 예를 들어, 데이터의 개수를 세거나, 데이터의 평균을 계산하는 등의 연산을 활용하여 시각화할 수 있음
    
- 데이터 시각화는 데이터 분석에서 중요한 기능이므로, 이를 효과적으로 활용할 수 있는 능력이 필요함
    

8.그래프의 표현과 회귀

8-1.그래프의 구조와 기능

- 데이터 프레임의 X, Y 컬럼을 기준으로 그래프를 그려줌
    
- 도표에 있던 기준 컬럼을 X, Y의 자리에 넣어 데이터를 시각화함
    
- 데이터를 기준으로 1, 2, 3, 4 등 숫자를 척도에 기준으로 시각화함
    
- 컬러라는 뜻을 지닌 척도를 사용하여 데이터의 색깔을 구분함
    
- 팔레트라는 디자인 컨셉으로 색깔을 일관되게 적용함
    

8-2.데이터 표현과 신뢰구간

- 시본 데이터를 자연 발생한 데이터로 가정하고 평균과 중앙값이 일치하는 정규분포를 따름
    
- 수시그마 안에 7% 정도의 데이터가 포함된 신뢰구간을 그림으로 표현함
    
- 통계학자들이 분석한 공통적인 특징을 통해 데이터를 표현하는 방정식을 만듦
    
- 데이터의 경향성을 설명하는 회귀선을 그려 데이터의 회귀를 보여줌
    
- 데이터의 경향성을 수렴하는 회귀방정식을 통해 데이터 포함 확률을 표시함
    

8-3.파라미터 설정과 시각화

- 데이터 프레임에서 컬럼을 추가하여 시각화하기 위한 파라미터를 설정함
    
- 그래프의 높이 지정, 점의 크기와 투명도, 그래프의 표시 방식 등을 설정함
    
- C언어와 라이브러리를 사용하여 그래프를 만드는 모형을 저장함
    
- C언어의 파라미터들은 저장된 파일에서 불러와 사용함
    
- 데이터를 그래프에 표현하고, 회귀와 신뢰구간을 표시하여 데이터를 시각화함
    

9.넘파이 프로그램 활용하기

9-1.넘파이 프로그램 소개

- 넘파이는 프로그램을 이용해 데이터를 시각화함
    
- 프로그램을 이용한 시각화는 그림이나 도표와 같은 매체가 아님
    
- 데이터를 여러 개의 파라미터로 설정하여 그림의 각 축을 제어함
    
- 넘파이 사용 시 데이터의 특성에 맞게 파라미터를 설정할 수 있음
    
- 프로그램의 기본 기능은 그래프의 형태와 색깔, 데이터의 추이 등을 관리함
    

9-2.그래프 만들기

- 그래프 만들기 위해서는 축의 설정, 그래프의 기본 스타일, 마크 업데이트 등을 설정함
    
- 각 축에 대한 독립적인 설정이 가능하며, 데이터의 추이에 따라 파라미터를 조정할 수 있음
    
- 그래프의 형태는 원하는 그래프의 형태에 따라 설정할 수 있음
    
- 그래프 내 마크 업데이트는 데이터에 대한 추가 정보를 제공함
    
- 데이터에 따라 마크 업데이트의 디자인을 다르게 설정할 수 있음
    

9-3.그래프 이해

- 그래프는 데이터를 시각화하는 도구로, 데이터에 대한 이해를 돕는 도구임
    
- 시각화 도구로서, 데이터에 대한 설명을 가장 잘 할 수 있는 방법을 제공함
    
- 시각화 도구로써의 기능은 데이터의 특성에 따라 다양하게 활용될 수 있음
    
- 특히, 각 축에 대한 설정이 데이터 시각화의 핵심이 될 수 있음
    
- 시각화 도구를 통해 데이터를 효과적으로 전달하고 이해할 수 있음
    

10.데이터 시각화

10-1.그래프 작성법 이해

- 그래프에 다양한 데이터를 시각화할 수 있음
    
- 그래프의 흐름을 유도하는 '서브플롯'을 사용하여 복잡한 데이터를 단순화할 수 있음
    
- '서브플롯 맥'을 통해 그래프를 세 개로 나누고 각각 다른 데이터를 그림
    
- '서브플롯 맥'에 데이터를 입력하면, 그 데이터가 그래프를 통해 시각화됨
    

10-2.그래프의 다양한 응용

- '서브플롯 맥'의 ' X축 공유' 기능을 통해 데이터가 중복되는 부분 없이 눈에 띄도록 할 수 있음
    
- '서브플롯 맥'의 ' H라인' 기능을 통해 데이터를 시각화할 때 특정 축을 선택하도록 함
    
- '서브플롯 맥'을 통해 여러 개의 그래프를 하나의 그림 안에 그릴 수 있음
    
- '라이플롯'은 TIPS 데이터를 사용하여 여러 개의 그래프를 그릴 수 있는 함수임
    
- TIPS 데이터는 카테고리, 인구 통계학적 변수, 시간적 변수 등을 포함함
    

10-3.시각화 도구 소개

- HIPS 데이터를 통해 다양한 인사이트를 얻을 수 있음
    
- HIPS 데이터는 숫자, 문자열, 논리수, 논리변수 등을 포함함
    
- HIPS 데이터의 예시로 '패키지', '소비자 성별', '소비 팁', '소비 시간' 등을 들어볼 수 있음
    
- HIPS 데이터에서 '패키지'는 판매된 제품의 종류를 의미함
    
- '소비자 성별'은 소비자의 남성 여부를 의미하며, '소비 팁'은 매 레벨에 따른 소비 팁의 양을 의미함
    

11.카테고리 데이터 처리와 분석

11-1.카테고리 데이터의 특성과 필요성

- 카테고리라는 데이터형은 0과 1로 이루어져 있으며, 이를 통해 성별 등의 정보를 판별함
    
- 데이터의 크기가 달라질수록 메모리 차이가 크게 발생하며, 연산 속도도 느려짐
    
- 데이터의 순서를 유지하기 위해 카테고리라는 데이터형을 사용함
    

11-2.카테고리 데이터의 내부적 관리

- 문자열로 이루어진 카테고리 데이터는 순서대로 저장되며, 이를 통해 연산의 효율성이 높아짐
    
- 카테고리 데이터는 메모리를 덜 차지하고, 속도가 빠르게 연산을 진행할 수 있음
    
- 강의에서는 카테고리 데이터의 저장 방식과 그 특성을 자세히 설명함
    

11-3.실제 카테고리 데이터의 활용 사례

- 브라질 카페에서 제공하는 고기 무한 제공 서비스를 예로 들어 카테고리 데이터의 활용을 설명함
    
- 무한 제공 서비스에서는 다양한 종류의 고기를 원하는 고객의 요구를 충족시키기 위해 카테고리 데이터를 활용함
    
- 강의에서는 이런 실제 사례를 통해 카테고리 데이터의 중요성과 효용성을 재차 강조함
    

12.데이터 정렬과 시각화

12-1.데이터 정렬의 이해

- 데이터 정렬은 순서를 바꾸는 것을 의미함
    
- 순서가 의미 있는 자료형을 다루는 방법에는 여러 가지가 있음
    
- ABC 순서로 정렬하거나, 다른 순서로 정렬할 수 있음
    
- 문자열로 된 값에는 순서를 정할 수 없음
    
- 자료형을 카테고리로 변경하여 순서를 바꾸는 방법도 있음
    

12-2.카테고리 데이터의 순서 변경

- 스트링 자료형을 카테고리 데이터 타입으로 변경할 수 있음
    
- 카테고리 데이터 타입으로 변경하면 메모리 용량이 줄어듦
    
- 카테고리 데이터의 순서를 변경하여 관리하는 방법도 있음
    
- 명목형 변수는 순서 자체가 의미가 없으므로, 순서를 고려하지 않아도 됨
    
- 순서형 변수의 경우, 순서를 바꾸는 과정이 필요함
    

12-3.시각화 도구와 데이터 분석

- 라임플롯을 사용하여 데이터를 시각화하여 트렌드를 파악할 수 있음
    
- 힙스 컬럼에 데이터 프레임을 넣고, 순서를 지정하여 시각화할 수 있음
    
- 연속형 변수와 시간수 데이터를 다루기 위해 아임플롯을 활용할 수 있음
    
- 라임플롯에 기준을 지정하여 데이터의 평균을 보여줄 수 있음
    
- 아임플롯을 통해 데이터의 트렌드를 파악하고, 이상치를 발견할 수 있음
    

13.시각화 모델링

13-1.데이터 시각화

- 연산된 결과를 기반으로 그래프 상의 요소를 활용해 시각화함
    
- 가장 확률적으로 근사하는 값을 표시해 요일별 매출 추이 확인 가능
    
- 다양한 연산 결과를 그래프로 시각화할 수 있음
    
- 에스티메이터 파라미터를 활용해 종합 신뢰 구간을 확인할 수 있음
    
- 95% 신뢰 구간을 통해 가장 확률적으로 값이 맞는 값을 추정할 수 있음
    

13-2.시각화 파라미터

- 다양한 파라미터를 활용해 그래프의 레이아웃과 기능을 조정할 수 있음
    
- 컬럼 기준으로 타임 기준을 지정해 다양한 시각화 방법을 시도할 수 있음
    
- 타임 기준을 지정하면 컬럼의 시각화 여부가 변경됨
    
- 파라미터를 조정해 원하는 시각화 방법을 얻을 수 있음
    
- 예를 들어, 점심과 저녁의 매출 추이를 별도의 그래프로 시각화할 수 있음
    

13-3.시각화 도구

- 넘파이 도구를 활용해 시각화 결과를 간결하고 명료하게 표현할 수 있음
    
- 컬럼 기준으로 각 요일별 매출을 따로 서브셋으로 지정해 시각화할 수 있음
    
- 시각화 도구를 통해 요일별 매출과 점심과 저녁의 매출 추이를 따로 레이아웃할 수 있음
    
- 이는 데이터의 특징을 더 잘 이해하고 시각화하기 위함임
    

14.데이터 분석과 시각화

14-1.데이터 시각화의 중요성

- 데이터 분석에 시각화가 중요한 이유를 설명함
    
- 시각화를 통해 많은 정보를 간결하게 표현할 수 있음
    
- 힙스 데이터에서 스펙터 플롯을 그려 보는 방법을 예로 들어 설명함
    
- 스크립트를 통해 연속형 변수와 이산형 변수를 관계에 따라 조정할 수 있음
    
- 데이터 내 상관관계와 경향을 파악하기 위해 시각화 도구를 사용함
    

14-2.시각화 도구를 이용한 데이터 분석

- 토탈빌과 팁의 관계와 같은 목표에 따라 시각화 도구를 사용함
    
- 시각화 도구를 통해 양의 상관관계와 경향을 파악할 수 있음
    
- 30%를 넘어서면 상관관계가 급변하는 지점을 확인할 수 있음
    
- 이를 통해 상관관계가 없어지는 이유와 향후 방향을 추론할 수 있음
    
- 관계의 세부 사항을 확인하기 위해 2차원 그래프에 색깔을 입힘
    

14-3.시각화 도구의 활용 사례

- 2차원 그래프의 대립상단을 조정해 데이터를 시각화함
    
- 화면에 3개의 종류를 추가해 시각화의 깊이를 더함
    
- 보조 모니터를 활용하여 메모리 사용량을 관리함
    
- 메모리 사용량을 관리하기 위해 화면 설정을 변경함
    
- 여러 종류의 보조 모니터를 연결하여 사용함
    

15.데이터 시각화

15-1.시각화 방식 이해

- 다양한 종류의 데이터 시각화가 있음
    
- 시각화 방식은 데이터의 타입에 따라 달라짐
    
- 카운트 플롯은 명목형 변수의 개수를 세는 방법임
    
- 명목형 변수의 개수를 세고, 이를 시각화한 그래프임
    
- 다양한 명목형 변수를 활용하여 시각화할 수 있음
    

15-2.시각화 방법과 한계

- 명목형 변수의 개수를 세는 방법에는 카운트 플롯과 디스트 플롯이 있음
    
- 카운트 플롯은 연속형 변수의 개수를 세는 방법임
    
- 디스트 플롯은 연속형 변수를 구간별로 나누어 개수를 세는 방법임
    
- 성별, 남성, 여성과 같은 질적 변수는 디스트 플롯을 사용해야 함
    
- 통계 수치가 올바르게 계산되는지 에러 메시지를 확인해야 함
    

15-3.명목형 변수 시각화

- 명목형 변수의 개수를 세는 시각화 방법을 살펴봄
    
- 식당 방문 사람들의 성별을 분류하는 예시임
    
- 성별별로 데이터의 개수를 세어 시각화함
    
- 성별을 X축에, 임상성별을 Y축에 표현함
    
- 명목형 변수의 개수를 세고, 통계 수치를 확인함
    

16.통계분석의 구현과 해석

16-1.히스토그램과 KT 파라미터 이해

- 히스토그램은 연속형 변수의 분포를 시각화한 그래프임
    
- KT 파라미터를 사용해 확률밀도 함수를 계산함
    
- KT 파라미터에는 히스토그램에 사용되는 개수, 히스토그램의 최대/최소값 등이 포함됨
    
- KT 파라미터를 이용해 히스토그램을 보다 정확하게 그릴 수 있음
    
- 히스토그램을 통해 연속형 변수의 분포, 평균값, 중앙값 등을 파악할 수 있음
    

16-2.표본내용과 분석 방법

- 데이터의 구간별 개수를 세어 전체 데이터를 구간으로 나눔
    
- 바구니라는 파라미터를 사용해 전체 데이터를 일정한 크기로 나눔
    
- 바구니의 크기를 10개로 나누어 각 구간에 들어있는 데이터의 개수를 파악함
    
- 해당 구간에 해당하는 값의 분포를 확인하며, 중앙값은 이 구간의 평균값임
    
- 중앙값과 평균값의 차이를 통해 평균보다 사용하는 경향을 확인함
    

16-3.박스 플롯과 주요 통계량

- 박스 플롯은 통계량을 손으로 표시한 플롯임
    
- 평균값을 중심으로 가장 작은 값부터 가장 큰 값을 순서대로 나타냄
    
- 중앙값은 이상치 판정 시 사용하는 값이며, 1.5만큼 곱한 값을 최대값, 1.5만큼 뺀 값을 최소값으로 정함
    
- 이상치 판정 시, 주로 사용하는 방법은 플란임
    
- 박스 플롯을 통해 각 데이터의 분포를 직관적으로 이해하고 분석할 수 있음
    

17.박스 플롯과 이상치

17-1.박스 플롯 그리기

- 데이터의 특성을 시각화할 때 박스 플롯 사용을 권장함
    
- 박스 플롯의 X축에는 성별, 흡연 여부, 요구사항, 요일, 심한 정도 등 변수를 넣을 수 있음
    
- 예를 들어, 남성과 여성의 성별 비율을 확인할 수 있음
    
- x축의 값에 따라 남성과 여성으로 구단을 만들고, 데이터의 분포를 확인할 수 있음
    
- 성별에 따른 값의 분포를 확인하고, 이상치 여부를 판단할 수 있음
    

17-2.이상치 판정

- 통계학자들이 정한 이상치의 기준을 설명함
    
- 이상치는 '보다 벗어난다'는 의미로, 1.5 곱하기 IQR을 넘어가는 값을 의미함
    
- 데이터의 분포에 따라 이상치의 기준이 다를 수 있음
    
- 1.5이 IQR을 구성하는 값이 3분위수보다 높은 경우, 이상치로 판정함
    
- 이상치를 판단하는 기준을 명확히 이해해야 함
    

17-3.이상치 계산과 활용

- 박스 플롯을 통해 이상치를 계산하는 방법을 설명함
    
- 3분위수를 이용한 이상치 판정 기준을 소개함
    
- 데이터의 분포에 따라 이상치의 기준을 세워야 함
    
- Q1과 Q3을 구하고, Q3에서 Q1을 뺀 값들을 이용하여 이상치를 계산함
    
- 이상치를 판정할 때 성별을 고려해야 함
    

18.데이터 분석 기법

18-1.모델링과 예외 상황

- 주가 예측 모델을 만들 때 과거 정보를 기반으로 미래 예측을 추구함
    
- 예외 상황에 대해 모델을 학습시킴으로써, 이 예외 상황이 발생할 때 다른 데이터들에 대한 예측을 보다 정확하게 할 수 있음
    
- 이상치를 포함하여 데이터를 분석하고, 그 이상치를 제거하거나 적용하여 데이터를 정제함
    
- 데이터가 한쪽으로 쏠려있을 경우, 중앙값에서 얼마나 벗어나는지 계산하여 따져볼 수 있음
    
- 통계학자들이 보장하는 이상치라는 개념은 데이터가 정규분포를 벗어나는 지점을 의미함
    

18-2.박스 플롯과 바이올린 플롯

- 통계량에 대한 정보를 한눈에 제공하기 위해, 박스 플롯을 사용함
    
- 연속적인 데이터를 표시하기 위해, 바이올린 플롯을 활용하여 데이터의 5개 통계량을 시각화함
    
- 바이올린 플롯은 성별, 편의, 시간대 등 다양한 요인을 고려하여 데이터의 분포를 확인할 수 있게 함
    
- 기계로 표현할 수 없는 데이터를 표현하기 위해, 성별과 시간대에 따라 색깔을 달리하여 데이터를 시각화함
    
- 성별에 따른 요일별 팁 분포를 박스 플롯으로, 시간대에 따른 요일별 판매량 분포를 바이올린 플롯으로 나타낼 수 있음
    

18-3.지도와 도형

- 플라이트 롱 데이터를 활용하여, 순서가 있는 질적 변수인 '승객 수'를 시각화함

- 연산을 통해 순서를 결정하는 시본 함수를 구현하고, 순서를 바탕으로 데이터를 시각화함
    
- 플라이트 롱 데이터의 각 요일별 판매량 분포를 박스 플롯으로, 시간대별 판매량 분포를 바이올린 플롯으로 나타낼 수 있음

- 평균을 기준으로 국가별 수명을 비교할 수 있지만, 특정 시점의 평균은 의미가 없음
    
- 데이터의 평균을 계산하는 이유는 변하지 않는 값이 아닌, 변화하는 값의 평균임
    
- GDP, 소득 분포 등 다양한 데이터를 확인할 수 있지만, 명목형 변수의 경우 큰 편차가 나타남
    
- 국가별 수명이 연봉의 24%를 차지할 정도로 차이가 날 수 있음
    

5-3.데이터 정제와 보완

- 데이터의 누락이나 오류를 확인하고 수정할 수 있음
    
- '컨트롤' 패키지를 통해 데이터의 이상값을 확인할 수 있음
    
- 평균을 계산할 때, '컨트롤' 패키지의 '질병수' 값으로 계산해야 함
    
- 국제기구의 데이터를 사용할 때, '컨트롤' 패키지의 '수치의 유의미성'을 확인해야 함
    
- '디딤더' 패키지의 '디딤더 리포트'를 통해 데이터를 시각화하고 보완할 수 있음
    

6.의료기술 분석

6-1.데이터 분석 프레임워크

- 데이터 시각화 프레임워크로는 '사용자', '데이터', '변수' 세 가지 요소가 있음
    
- '사용자'는 시각화를 통해 정보를 제공받는 사람을 의미함
    
- '데이터'는 분석하고자 하는 대상 집합을 의미함
    
- '변수'는 분석 대상의 속성으로, 데이터의 특징을 나타냄
    
- 데이터의 특징을 분석해 '변수'로 표현함
    

6-2.데이터 해석과 시각화

- '데이터'에서 '변수'를 찾아 '지수'로 변환함
    
- 변환된 '지수'를 이용해 '데이터'를 시각화함
    
- 시각화 방식은 '막대 그래프', '막점 그래프', '사각 점 그래프', '선 그래프' 등이 있음
    
- 시각화 방식은 '연속형 변수'와 '이산형 변수'에 따라 다르게 표현됨
    
- 예를 들어, '국가별 GDP와 기대수명' 데이터에서 '국가'라는 변수를 '지수'로 변환함
    

6-3.예시 분석

- 1952년부터 2007년까지의 데이터를 선택함
    
- '연속형 변수'인 'GDP'와 '기대수명'을 '지수'로 변환함
    
- '지수'를 이용해 데이터를 시각화함
    
- 시각화 방식으로 '그래프', '서브셋', '점', '줄', '색깔' 등이 사용됨
    
- '연속형 변수'와 '이산형 변수' 간의 관계를 분석해 '상관관계'를 파악함
    

7.데이터 특징

7-1.선형 회귀선 그리기

- 여러 값의 변화 양을 분석해 선형 회귀선을 만듦
    
- 데이터의 특징을 부각하기 위해 X축에 논리적인 수축을 적용함
    
- 변화량을 이용해 곡선의 왜곡을 나타냄
    
- 100불에서 3천불까지 상승할 때 소득 수준이 얼마나 늘어나는지 비교 가능
    
- 변화가 급격히 퍼지는 지점에 선을 그어 강조하기도 함
    

7-2.회귀선의 수정

- 오브스케일을 사용해 데이터의 왜곡을 줄임
    
- 100불에서 300불까지 상승할 때와 100불에서 700불까지 상승할 때의 변화량을 비교 가능
    
- 선형 회귀선을 그리기 위해 근처 점들과의 거리를 최소화함
    
- 근처 점들과의 거리를 고려해 변화량을 반영한 회귀선을 만듦
    
- 변화량과 근처 점들의 가중치를 주어 특징을 부각할 수 있음
    

7-3.데이터 특징 강화

- 데이터의 특징을 강화해 선형 회귀선의 왜곡을 줄임
    
- 오브스케일을 사용해 데이터의 극단적인 값도 표현 가능
    
- 선형 회귀선을 통해 근처 점들과의 거리를 최소화함
    
- 변화량과 근처 점들의 가중치를 주어 데이터의 특징을 강화함
    
- 데이터의 변화 양을 강조하거나 근처 점들의 변화를 고려함
    

8.시계열 데이터 분석

8-1.시계열 데이터 이해

- 데이터 시각화에 시계열 그래프 사용이 효과적임
    
- 연속형과 명목형 변수 간의 상관관계를 시계열 데이터로 한눈에 확인 가능
    
- 다양한 데이터 시각화 방법이 있음
    
- 상황에 따라 데이터 시각화 방법을 선택하는 것이 중요함
    
- 생산된 데이터를 시계열 데이터로 변환하여 시각화 가능
    

8-2.산점도 데이터 활용

- Minder 도구를 활용하여 산점도를 시계열 데이터로 변환할 수 있음
    
- 국가별 기대수명의 변화량을 산점도로 시각화할 수 있음
    
- 색깔을 사용하여 데이터를 세분화하여 분석할 수 있음
    
- 북한과 한국은 같은 색으로 표시되는 경우가 많음
    
- 2007년 기준 전 세계 기대수명을 국가별로 확인할 수 있음
    

8-3.와이파이낸스 API 활용

- 와이파이낸스 패키지의 API를 활용하여 주가 데이터를 확인할 수 있음
    
- 미국의 야후를 통해 와이파이낸스 API를 사용하는 방법을 알아볼 수 있음
    
- API 레퍼런스를 통해 사용 방법을 확인할 수 있음
    
- 간단한 샘플 예제를 통해 사용 방법을 익힐 수 있음
    
- 첫 번째 줄을 중심으로 데이터를 확인하고, 나머지 줄은 생략할 수 있음
    

9.마이크로소프트 및 삼성전자 데이터 분석과 처리

9-1.마이크로소프트 데이터 분석과 처리

- 마이크로소프트의 종목 코드를 통해 데이터를 검색하고, 간단한 메타데이터를 확인함
    
- 주식의 최근 배당금 및 최신 주가 정보 등 다양한 데이터 제공
    
- 마이크로소프트의 주식 정보를 검색하고, 필요한 분할 비율 정보를 제공
    
- 주식 데이터를 불러오는 다양한 방식과 방법에 대해 설명함
    

9-2.삼성전자 데이터 분석 및 처리

- 삼성전자의 미국 주식 패턴을 통해 데이터를 불러옴
    
- 한국 주식인 코스피 및 코스닥 데이터를 불러오고, 10년 분량을 검색함
    
- 삼성전자의 데이터를 업데이트 및 수정하고, 다중 인덱스를 해제하는 작업을 수행함
    
- 데이터의 표본 정보를 확인하고, 데이터를 미리 분류 및 정리하는 작업을 수행함
    

9-3.데이터 처리의 실무적 접근

- 데이터 처리를 위한 실무적 방법론 제시함
    
- 다중 인덱스를 해제하고, 데이터를 분류 및 정리하는 방법에 대해 설명함
    
- 데이터의 순위를 변경하거나 무너뜨리는 방법에 대해 설명함
    
- 데이터를 주기적으로 확인하고, 필요시에 데이터를 업데이트하거나 수정하는 작업을 수행함
    

10.주식 데이터 분석을 위한 그래프 생성 및 정렬 방법

10-1.주식 데이터의 그래프 생성

- 데이터의 특징에 따라 그래프의 형태와 크기를 조절할 수 있음
    
- 복잡한 데이터를 명령어로 작성하면 그래프를 생성할 수 있음
    
- 데이터의 범위를 지정하면 그래프의 범위를 따라 그래프를 생성할 수 있음
    
- 그래프 생성 시, 필요한 부분만 잘라서 사용할 수 있음
    
- 애플 데이터를 사용해 샘플 데이터를 생성하면, 다양한 종류의 데이터를 한눈에 볼 수 있음
    

10-2.주식 데이터의 정렬과 출력

- 주식 데이터의 정렬을 원하는 순서에 따라 데이터를 입력하면, 그래프를 생성할 수 있음
    
- 각 데이터의 시가와 종가를 출력하면서 데이터의 순서를 지정할 수 있음
    
- 데이터를 입력할 때, '인덱스'를 컬럼으로 만들어주고 '변수명'을 지정할 수 있음
    
- '다음 분류사'에 나머지 데이터는 버리고 특정 부분만 잘라서 사용할 수 있음
    
- 정렬을 원하는 데이터에 따라, '최대한 시간 순으로' 데이터를 출력하면 좋음
    

10-3.데이터 정렬의 예제와 참고

- 예제를 통해 주식 데이터의 정렬을 확인할 수 있음
    
- 데이터를 입력할 때, '인덱스'를 컬럼으로 만들어주고 '변수명'을 지정하면, 각 데이터의 인덱스와 해당 변수에 따른 데이터를 출력할 수 있음
    
- 각 데이터의 '시가'와 '종가'만을 출력하며, '변수명'에 '시가', '종가', '량'을 지정하면 됨
    
- '다음 분류사'에 '오픈', '클로즈'를 지정하면, 해당 변수의 값을 따라 그래프를 생성할 수 있음
    
- 이 과정을 통해 데이터의 특성에 따른 최적의 그래프를 생성할 수 있음
    

11.데이터 정렬과 파이썬 소트

11-1.데이터 정렬의 이해

- 데이터 정렬은 순서에 따라 다르게 처리하는 기법임
    
- 날짜 기준으로 정렬하면 작은 날짜부터 큰 날짜 순으로 정렬됨
    
- '카테고리컬 밸류'를 사용하면 정렬이 더욱 명확해짐
    
- 이와 같은 방법으로 정렬을 할 때, 값이 클수록 먼저 배정됨
    

11-2.정렬과 유니크 값

- 정렬 후에는 '유니크 값'이 생성되며, 이를 통해 원하는 카테고리가 확인 가능함
    
- 이 때, 정렬은 일차, 이차 정렬 기준으로 이루어짐
    
- 문자열로 된 값의 경우, 순서를 바꾸어도 카테고리컬 밸류로 변환해야 함
    
- 이를 통해, 정렬의 기준이 더욱 명확해짐
    

11-3.파이썬 소트의 활용

- 파이썬 소트를 활용하면, 퀵소트, 머지소트, 버블소트 등 다양한 정렬 알고리즘을 구현 가능함
    
- 각 알고리즘은 정렬된 데이터를 더 효율적으로 관리할 수 있음
    
- 특히, 퀵소트는 빠른 정렬 알고리즘으로, 효율성과 복잡성이 높음
    
- 이러한 이해를 바탕으로, 복잡한 데이터를 효과적으로 정렬할 수 있음
