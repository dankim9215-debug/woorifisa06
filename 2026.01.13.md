1.판다스와 넘파이

1-1.판다스와 넘파이 소개

- 판다스는 패널 데이터 시스템의 앞글자를 따서 만든 용어로, 특정 그룹을 주기적으로 추적하여 얻는 데이터를 뜻함
    
- 패널 데이터는 시간, 시작값, 가격, 거래량 등의 정보를 포함함
    
- 판다스는 패널 데이터를 일괄적으로 분석하기 위해 넘파이의 날짜, 행, 열, 가격 정보 등을 활용함
    
- 넘파이는 날짜와 행을 순서대로 묶어 저장해 속도가 빠르고, 데이터를 불러와 분석하는 데 최적화되어 있음
    
- 파이썬 기반으로 개발되어 다른 프레임워크와 호환되어 데이터 분석이 용이함
    

1-2.판다스 설치와 의존성

- 판다스는 파이썬 명령어로 설치되며, 넘파이와 의존되어 동작함
    
- 판다스는 넘파이의 여러 함수들을 불러내는 자료형으로, 넘파이에 설치되어 동작함
    
- 판다스는 데이터를 조작할 때, 먼저 넘파이의 함수들을 불러내는 것이 중요함
    
- 판다스의 엔디어레이는 데이터의 행과 열 정보를 포함하는 고유한 자료형임
    
- 판다스는 넘파이와 함께 설치되어 동작하며, 넘파이의 함수들을 불러내는 등의 동작을 함
    

1-3.데이터 조작과 시각화

- 판다스는 데이터를 일괄적으로 분석하기 위해 필요한 통계 연산 메서드를 제공함
    
- 판다스는 데이터를 불러와 분석하는 데 최적화되어 있으며, 많은 양의 패턴을 가진 정형화된 데이터를 분석하는 데 유용함
    
- 판다스는 파이썬을 기반으로 개발되어 호환성이 뛰어나며, 엑셀 파일을 불러와 분석하는 데에도 적합함
    
- 파이썬 명령어로 판다스를 설치하면, 파이썬 환경에 맞춰 판다스를 설치할 수 있음
    
- PIP 명령어를 사용하여 판다스를 설치하면, 판다스가 제어되는 여러 패키지들이 함께 설치됨
    

2.판다스 데이터 프레임

2-1.데이터 프레임의 특징

- 열벡터로 이루어진 시리즈의 열을 하나의 인덱스로 나란히 붙인 데이터 프레임으로 정의함
    
- 인덱스와 값으로 구성된 열의 배열을 데이터 프레임이라고 부름
    
- 데이터 프레임은 2차원 행렬 형태로, 인덱스를 통해 열의 위치를 관리함
    
- 여러 개의 열을 인덱스로 나란히 붙인 형태로, 특정 열에 값을 저장할 수 있음
    
- 데이터 프레임은 파이썬의 내장 집합 자료형인 '디프셔널'과 유사함
    

2-2.데이터 프레임의 변환

- '디프셔널' 자료형은 키와 값으로 구성되며, '데이터 프레임'은 2차원 행렬 형태로 데이터를 저장함
    
- 데이터 프레임은 '컬럼', '인덱스', '밸류'로 구성된 열의 배열로 정의됨
    
- 인덱스는 0, 1, 2 등의 값을 갖는 범위를 관리함
    
- 데이터 프레임의 값은 인덱스와 연결되어 있으며, 배열의 각 열에 데이터를 저장할 수 있음
    
- 데이터 프레임은 표와 같이 열의 순서와 인덱스를 가짐
    

2-3.데이터 프레임의 타입과 장단점

- 판다스 내 데이터 프레임 클래스는 '디프셔널' 자료형의 변환으로 생성됨
    
- 데이터 프레임은 컬럼, 인덱스, 밸류로 구성된 열의 배열로 관리됨
    
- 데이터 프레임의 인덱스는 0, 1, 2 등의 범위를 포함하며, 행별로 인덱스가 연결되어 있음
    
- 데이터 프레임의 값은 컬럼과 인덱스로 제어할 수 있으며, 통계 연산 등을 통해 대표값을 찾을 수 있음
    
- 데이터 프레임은 표와 달리 열의 순서와 인덱스를 가짐
    

3.데이터 프레임

3-1.데이터 프레임 이해

- 데이터 프레임은 시리즈 여러 개를 묶은 2차원 객체임
    
- 데이터 프레임의 컬럼과 인덱스는 열, 행 순서로 접근함
    
- 열과 행 순서를 조합해서 특정 열에 접근함
    
- 특정 열의 값을 변경할 때는 열과 행 순서를 함께 변경해야 함
    
- 데이터 프레임의 접근은 컬럼과 인덱스의 변경으로도 가능함
    

3-2.컬럼과 인덱스 변경

- 컬럼과 인덱스를 변경하면 특정 위치의 값을 조회할 수 있음
    
- 컬럼과 인덱스를 변경할 때는 개수가 맞아야 동작함
    
- 컬럼과 인덱스의 개수가 맞지 않으면 데이터 프레임이 무너질 수 있음
    
- 직접 접근해 특정 값을 변경할 때는 컬럼과 인덱스의 개수에 맞춰야 함
    
- 컬럼과 인덱스를 변경할 때는 내부적으로 구조화되어 있음을 기억해야 함
    

3-3.원본 데이터 형식

- 원본 데이터의 형식을 알고 있다면 컬럼과 인덱스를 직접 변경하는 것이 더 용이함
    
- 인덱스를 변경하면 해당 열의 인덱스 순서가 유지되도록 함
    
- 컬럼을 변경하면 해당 열의 모든 위치에 변화가 생김
    
- 원본 데이터 형식을 알고 있다면 컬럼과 인덱스를 변경하는 것이 더 쉬움
    
- 데이터 프레임의 함수를 사용해 컬럼과 인덱스를 변경할 수 있음
    

4.데이터 프레임

4-1.데이터 프레임 이해

- 데이터 프레임은 시리즈들로 구성되어 있음
    
- 시리즈 하나하나를 열 별로 붙여서 사용하는 구조는 불가능함
    
- 하나의 데이터 프레임에는 시리즈의 이름, 나이, 점수 등의 정보가 포함됨
    
- 데이터 프레임에 새로운 컬럼을 추가하거나 중복되는 컬럼을 삭제할 수 있음
    
- 새로운 컬럼을 추가할 때는 데이터 프레임의 개수를 맞춰야 함
    

4-2.컬럼 삽입과 삭제

- 데이터 프레임에 새로운 컬럼을 삽입할 때는 개수를 맞춰야 함
    
- 컬럼을 삭제할 때는 해당 열이 존재하는지 확인해야 함
    
- 데이터 프레임의 컬럼을 삭제하려면 해당 열이 있는지 확인해야 함
    
- 컬럼을 삽입할 때는 '디프레임 지역'을 삭제하려는 명령어를 사용할 수 있음
    
- 컬럼을 삭제하면 해당 열이 삭제된 것으로 간주되며, 삭제되지 않은 경우에도 개수를 일괄적으로 맞춰야 함
    

4-3.데이터 프레임의 활용

- 데이터 프레임에 기준을 추가하거나 변경할 수 있음
    
- 나이가 25세 이상인 사람을 검사하고, 25세 이하를 검사하는 컬럼을 만들 수 있음
    
- 데이터 프레임의 컬럼을 사용하여 현실의 데이터를 조건에 따라 분리하거나 검사할 수 있음
    
- 예를 들어, '나이가 25세 이상인 사람'을 검사하는 컬럼을 만들 수 있음
    
- 컬럼을 사용하여 데이터를 조건에 따라 분리하고, 필요에 따라 수정할 수 있음

- 패널 정보 관리 위한 자료형이며, 내부적으로 숫자로 변환하여 사용함
    
- 열과 행 순서로 데이터 접근하며, 행과 열에 특정 명령어를 부여할 수 있음

- 값을 추가할 때는 시리즈 단위로 직접 값을 삽입하고, 개수가 맞으면 성공, 아니면 '값이 오류'를 반환함
    

6-2.데이터 삽입과 수정

- 시리즈 단위로 데이터를 추가하거나 수정할 때는 열행 순서로 직접 값을 삽입하거나, 위치 명령어를 사용함
    
- 판다스에서 값을 삽입할 때는 열과 행에 특정 명령어를 부여하며, 개수가 맞으면 성공, 아니면 '값이 오류'를 반환함
    
- 데이터 타입을 확인하여 원본 데이터와 다른 자료형의 데이터를 넣었을 때 문제가 발생할 수 있음
    
- 예를 들어, 숫자로 변환되지 않은 실수(스트링)를 넣었을 때, 자료형에 맞지 않아 자동 변환되어 다른 자료형으로 변경될 수 있음
    

6-3.데이터 확인과 활용

- 판다스에서 데이터를 확인할 때는 '행', '열', '당뇨점수' 등을 불러주어 값을 확인할 수 있음
    
- 열과 행에 특정 명령어를 부여하여 데이터의 각 열과 행에 접근하며, 특정 열과 행에만 값을 삽입하거나 수정할 수 있음
    
- 데이터의 '성공', '실패' 여부를 판별하여 결과를 출력할 수 있음
    
- 판다스는 데이터의 특정 열과 행에 접근하여 값을 확인하고, 이를 바탕으로 다양한 데이터 처리 및 활용이 가능함
    

7.판다스 데이터 처리

7-1.판다스 데이터 프레임 조작

- 판다스는 데이터를 분석하거나 다루는 데 사용되는 기능임
    
- 판다스의 기본 명령어들을 살펴보기 전에 데이터를 로드하고 변환하는 과정을 이해해야 함
    
- 판다스로 데이터를 변환할 때, 각 데이터 프레임의 인덱스, 데이터 컬럼, 데이터 값, 자료형 등을 확인할 수 있음
    
- 데이터 변환 시, 인덱스와 컬럼을 제어하는 명령어들을 사용함
    
- 판다스로 데이터를 조작하기 위해서는 분석 과정에서 사용하는 다섯 가지 기능을 알아야 함
    

7-2.데이터 프레임 설명과 통계

- 데이터 프레임은 인덱스, 컬럼, 데이터 값, 자료형을 포함한 정보를 제공함
    
- 인덱스는 데이터의 시작 위치를, 컬럼은 데이터의 각 열(행)을 나타냄
    
- 데이터 값은 데이터의 실제 값을, 자료형은 데이터의 유형(직업, 성별, 수명 등)을 나타냄
    
- 자료형에는 문자열, 숫자, 논리값, 다른 값을 포함할 수 있음
    
- 데이터 프레임의 대표값은 전체 데이터에서 가장 값이 자주 나온 값, 가장 작은 값, 가장 큰 값 등을 의미함
    

7-3.데이터 처리 기능

- 데이터 프레임에 대한 명령어로는 인포, 헤드, 테일, 포함, 평균, 표준편차, 중앙값 등이 있음
    
- 인포 명령어는 데이터 프레임의 인덱스를 가져와 전체 데이터를 대표값으로 반환함
    
- 헤드와 테일 명령어는 데이터의 시작 위치나 끝 위치를 대표값으로 사용함
    
- 포함 명령어는 데이터에 특정 열(행)을 추가하거나 변경하는 것을 의미함
    
- 평균과 표준편차 명령어는 데이터의 평균과 표준편차를 계산하여 대표값으로 반환함
    

8.데이터 전처리

8-1.데이터 이해와 전처리 방법

- 데이터에서 유니크한 값 몇 개인지 확인 가능함
    
- 데이터의 빈도수로 모니터링하여 모델 훈련에 활용함
    
- 특정 성별 데이터에 치우치면 모델이 특정 성별 정보에 치우칠 수 있음
    
- 데이터 전처리 시, 빈 값이 섞이거나 불가능한 경우 등을 확인해야 함
    
- 데이터의 대표값을 확인하여 개발 시 예외 상황에 대비해야 함
    

8-2.데이터 프레임 조작

- 판다스의 넘파이 계승 명령어로 데이터 프레임을 복사하여 유지함
    
- 데이터 프레임을 합치기 위해 명령어로 조작함
    
- 중복되는 컬럼을 기준으로 새로운 데이터 프레임을 생성함
    
- 서픽스 키워드를 사용하여 중복된 이름을 잠시로 미리 정해 놓음

9.데이터 프레임의 다양한 연산과 연결

9-1.데이터 프레임의 연산과 컬러스 이름 붙이기

- 데이터 프레임의 연산은 덧셈, 뺄셈, 곱셈 등으로 이루어짐
    
- 컬러스의 이름은 중복되는 키를 기준으로 붙여져 결과를 출력
    
- 이때, 중복되는 키가 포함된 순서대로 컬러스의 이름을 붙임
    
- 데이터 프레임의 중복되는 컬럼을 서픽스로 추가하여 결과 출력
    
- 특정 컬럼에 중복되는 키를 기준으로 다른 컬럼에 대해서는 스스로 서픽스를 붙임
    

9-2.통캡을 이용한 데이터 프레임 연결

- 통캡은 두 개의 데이터를 연결하는 방법
    
- Df + 통캡을 사용하여 두 개의 데이터를 연결함
    
- 통캡은 튜플 단위로 데이터를 연결하므로 결과 결과 프레임은 순서대로 전달됨
    
- 특정 컬럼에 없는 값을 새로운 열로 추가하여 데이터를 변환
    
- 다중 값의 조건에 따라 데이터를 변환하거나 새롭게 설정함
    

9-3.조인을 통한 데이터 프레임 합침

- 조인은 여러 데이터를 합치는 연산으로, 교집합을 합칠 때는 이너 조인, 서로 겹치는 데이터를 합칠 때는 오른쪽 기준으로 왼쪽을 합칠 때는 레프트 조인을 사용
    
- 조인 명령어의 기본값은 아웃도어 조인
    
- 중복되는 값만 가지고 결과를 반환하므로 중복되는 컬럼에 대해 조인을 사용
    
- 조인을 사용하여 데이터 프레임을 합칠 때는 이너 조인을 사용하여 각 컬럼에 대해 따로 조인을 사용함

12.데이터 프레임

12-1.데이터 프레임

- 데이터 프레임은 한 개 이상의 데이터 항목으로 구성됨
    
- 각 항목은 특정한 값으로 지정되어 있음
    
- 데이터 프레임을 불러오기 위해선 특정 열의 인덱스로 불러오거나, 인덱스로 데이터를 가져와야 함
    
- 인덱스로 데이터를 가져올 때는 '로케이션' 키워드를 사용함
    
- '로케이션' 키워드를 사용하면, 각 열의 인덱스에 해당하는 데이터만을 가져옴
    

12-2.정렬

- 데이터 프레임을 정렬할 때는 '서브셋'이라는 개념을 사용함
    
- 정렬을 할 때는 특정 열을 기준으로 정렬하는 올라가다 (서드파티) 명령어를 사용함
    
- 올라가다 명령어를 사용할 때는 '오르다'라는 뜻의 '서드파티' 명령어로 사용함
    
- 작은 값부터 큰 값 순서로 정렬하거나, 그 반대로 정렬할 수 있음
    
- '평트 밸류스' 명령어를 사용하면, 원본 데이터 프레임을 유지하면서 정렬을 함
    

12-3.정렬 우선순위

- 데이터를 정렬할 때는 '서드파티' 명령어를 사용해 정렬 우선순위를 지정할 수 있음
    
- '서드파티' 명령어를 사용할 때는 '폴트 밸류스'를 지정하면, 각 시리즈의 인덱스에 따라 순서대로 정렬됨
    
- '스코어'를 사용해 정렬을 하면, 데이터가 많을 때는 오름차순으로 정렬함
    
- 정렬 우선순위를 지정하면, 작업을 간단하게 할 수 있음
    
- '서드파티' 명령어를 통해 정렬 우선순위를 지정할 수 있음

13-2.브로드캐스팅과 결과

- 여러 개의 개별 값을 하나씩 대입하여 원하는 결과로 변화시킴
    
- 브로드캐스팅은 조건에 따라 데이터의 일부를 제거하거나 추가함
    
- 전처리 기능에서 사용한 기능들은 데이터 전처리 과정을 순서대로 진행함
    

13-3.예제

- 전처리 기능을 통해 문자열을 대체하거나 변환하여 사용함
    
- 전처리 기능은 데이터의 개수, 자료형, 배열 구조 등을 확인함
    

14.데이터 분석과 차원 탐색

14-1.데이터의 특성 분석과 기초 개념 이해

- 데이터 분석에서 가장 중요한 것은 자료형의 이해와 값의 확인임
    
- 데이터의 종류에 따라 값의 처리 방법이 달라짐
    
- 숫자로 된 자료형에 대해 중복된 값이 있는지 확인하는 것이 중요함
    
- 중복된 값이 있을 경우 이를 제거하거나 넣어주는 것이 데이터 처리의 핵심임
    
- 원본 그대로 값을 넣어 분석을 진행하며, 이를 null 값으로 간주할 수 있음
    

14-2.데이터 프레임의 파악과 분석

- 데이터 프레임을 파악하기 위해 '포괄함'이라는 명령어를 사용함
    
- 이 명령어를 통해 데이터 프레임 내부의 데이터 분포를 확인할 수 있음
    
- 각 데이터에 대한 평균값, 최소값, 최대값 등의 정보를 확인할 수 있음
    
- 또한, 데이터의 분포를 시각화하여 보여줄 수 있음
    
- 이 명령어를 사용하여 특정 개체가 아닌 전체 데이터의 경향성을 볼 수 있음
    

14-3.서브셋의 확인과 활용

- '서브셋'이라는 명령어를 통해 일부 데이터를 탐색하고 확인할 수 있음
    
- '서브셋'은 시리즈 단위 또는 데이터 프레임 단위로 사용될 수 있음
    
- 이 명령어를 통해 특정 개체가 아닌 전체 데이터의 중복된 값이 있는지 확인할 수 있음
    
- 중복된 값이 있는 경우 이를 제거하거나 넣어주는 것이 중요함
    
- 중복된 값이 있는지 확인하는 것은 데이터 처리의 핵심이며, 이를 통해 다음 단계의 작업을 진행할 수 있음
    

15.데이터프레임 활용

15-1.데이터프레임 단위

- 태국 음식을 먹었는데, 냄새가 좋고 맛있는 음식임
    
- 저장된 음식을 먹었고, 태국 갈비탕을 먹었음
    
- 닭이 날개를 가진 닭성을 먹었음
    
- 월요일에 가면 됐는데, 간판이 잘 보였음
    
- 토마토라는 이름의 식당이 지하에 있었음
    

15-2.데이터 추출

- 판다스의 데이터 프레임과 시리즈 단위를 활용하여 원하는 데이터를 추출할 수 있음
    
- 특정 기준에 맞는 데이터를 조건에 따라 끄집어낼 수 있음
    
- 나이가 22살보다 많은 사람들의 이름을 출력하려고 할 때, 컬럼 기준으로 필요로 하는 데이터를 조건에 따라 끄집어낼 수 있음
    
- 데이터프레임을 활용하여 직관적으로 결과를 확인할 수 있음
    
- 대괄호를 사용하여 원하는 열을 추가하거나, 행 순서대로 값을 출력할 수 있음
    

15-3.값 추출

- 값을 추려낼 때, 데이터 프레임으로 출력되는지 확인해야 함
    
- 시리즈를 받고, 데이터 프레임을 줄 때는 따로 인덱싱을 해야 함
    
- 시리즈를 인덱싱하여 데이터 프레임으로 만들 때, 패키지에 따라 결과를 달리 받아야 함
    
- 값이 열 기준으로 나올 때, 로케이션을 찍어낼 수 있음
    
- 행 순서대로 값을 출력할 때, 결과가 행 순서대로 출력되는지 확인해야 함
    

16.불린 인덱싱

16-1.불린 인덱싱의 개념과 장점

- 인덱스의 값이 불린 상태로 저장되면 불린 인덱싱을 통해 원하는 위치의 값을 직접 조회할 수 있음
    
- 판다스에서는 내부적으로 비교 연산자를 나란히 사용하는 것이 불가능함
    
- 불린 인덱싱을 통해 값의 범위를 명확히 하여 조건에 맞는 값을 정확하게 추려낼 수 있음
    
- 판다스의 내장 함수에서 불린 인덱싱을 통해 값을 추려내는 방법을 이해해야 함
    
- 파이썬에서는 불린 인덱싱을 통해 값을 직접 확인하는 방법을 사용함
    

16-2.불린 인덱싱의 세부 방법

- 22세 이상, 25세 미만의 사람을 찾는 예시에서 불린 인덱싱을 활용하여 데이터를 조회함
    
- '불린 인덱싱' 메서드를 사용하여 행 또는 열 단위로 원하는 조건을 만들 수 있음
    
- '최고 나이보다 작은 사람' 조건을 불린 인덱싱을 통해 확인할 수 있음
    
- 비교 연산자를 나란히 사용하는 것은 불가능하며, '==', '==지 않음'을 사용해야 함
    
- 대괄호 안에서 '==', '==지 않음'을 의미하는 'not'와 '&&'를 사용해야 함
    

16-3.불린 인덱싱의 특징과 한계

- 불린 인덱싱을 할 때는 행과 열 순서로 값을 가져오면 직관적이고 일기 쉽게 확인 가능함
    
- 불린 인덱싱은 주로 '보통' 방식을 사용하며, '포지션'으로도 접근 가능함
    
- 불린 인덱싱을 통해 값을 직접 확인할 때는 결과가 트루, 폴스로 나옴
    
- 불린 인덱싱은 비교 연산자를 나란히 사용하는 것이 불가능하므로 주의해야 함

17-2.조건에 따른 열 생성

- 조건에 따라 열을 생성할 때, '없음'의 기본값을 설정하거나 특정한 값으로 열의 위치를 변경할 수 있음을 설명함
    
- 데이터 프레임의 구조와 조건에 따라 열을 생성하거나 변경하는 방법을 설명함
    

17-3.블린 인덱싱의 활용

- '고객의 나이가 24세 이상이거나, C반인 경우'라는 조건을 만족하는 고객 정보를 출력하는 예시를 들음

- CSV 파일은 엑셀에서 불러오기 어려우며, 데이터를 세로로 길게 열기 때문에 불러오기 어려움

- 불러온 데이터의 형식이 판다스 형식에 맞는지 확인해야 함
    
- 판다스 형식은 컬럼별로 데이터를 저장하며, 이를 확인하기 위해 '열' 명령어 사용함

- 이 두 연산을 활용하면, 분석된 결과를 기반으로 가설을 검증하거나 새로운 이론을 세울 수 있음
    

5-3.그룹 바이 명령어의 활용

- '그룹 바이' 명령어를 활용하면, 특정 컬럼을 기준으로 그룹핑 후 개수를 세는 등 다양한 분석이 가능함
    
- 이를 통해 개별 데이터 항목에 대한 통계치를 산출하거나, 전체 데이터의 일관된 경향성을 파악할 수 있음
    
- 특히, 여러 가지 통계치를 산출하거나, 특정 기준에 따른 데이터의 특성을 파악하는 데 유용함
    

6.시리즈 그룹핑

6-1.그룹핑 기초

- 시리즈 그룹핑은 많은 대규모 데이터를 처리할 때 사용함
    
- 반별 혹은 장기별로 묶을 수 있고, 카운트라는 명령어로 그룹핑할 수 있음
    
- 반 기준으로 몇 개인지, 담당자 몇 개인지 등을 확인할 수 있음
    
- 연산 적용 시, 데이터 타입이나 이름 등의 정보를 함께 확인할 수 있음
    
- 연산 결과에 의도치 않은 결과가 나올 수 있으니 주의해야 함
    

6-2.그룹핑 시 유의점

- 데이터 프레임으로 묶을 경우, 자료형 정보가 함께 보존됨
    
- 분산을 계산할 때는 문자열이 아닌 실제 값(밸류)를 사용해야 함
    
- 분산 연산은 연산 불가능한 컬럼에 적용 시, 오류를 발생시킬 수 있음
    
- 연산 결과에 오류가 발생할 수 있으니 주의해야 함
    
- 연산에 사용되는 집계함수의 방식에 따라 결과가 달라질 수 있음
    

6-3.그룹핑 연산 예

- 테스트 점수의 평균을 계산하기 위해 '나이' 컬럼에 그룹핑할 수 있음
    
- '나이'에 1을 붙여 '나이_1', '나이_2' 등으로 나누어 묶을 수 있음
    
- '나이'가 1부터 5까지 2개씩 떨어져 있을 때, 각각에 1을 붙여 묶을 수 있음
    
- '나이' 컬럼에 1을 붙인 후, '나이'에 1을 붙인 것과 '나이'에 2를 붙인 것 등을 합쳐 '나이_1'과 '나이_2'로 묶을 수 있음
    
- '나이'에 1을 붙인 후, '나이'에 1을 붙인 것과 '나이'에 2를 붙인 것을 합쳐 '나이_1'과 '나이_2'로 묶을 수 있음
    

7.데이터프레임과 그룹바이

7-1.데이터프레임과 그룹바이의 이해

- 데이터프레임은 특정한 형식의 데이터의 집합이라고 설명함
    
- 각 데이터는 각각의 값으로 데이터프레임에 포함되어 있음
    
- 그룹바이는 데이터프레임을 일정한 그룹으로 묶는 방법임
    
- 그룹바이를 통해 여러 개의 서로 다른 값들을 하나의 그룹으로 묶을 수 있음
    
- 반복적인 연산을 위해 그룹바이를 사용하는 경우가 많음
    

7-2.그룹바이를 통한 데이터 접근

- 데이터프레임의 특정 위치에 접근하는 방법을 이해함
    
- 그룹바이 명령어를 이용해 특정 위치의 값을 직접 접근하거나 순서를 변경할 수 있음
    
- 다양한 인덱스를 생성하여 특정 위치에 접근하거나 특정 순서를 유지할 수 있음
    
- 인덱스는 특정 위치에 접근하는 데 사용되며, 행렬 순서로 데이터를 접근할 때도 사용됨
    

7-3.그룹바이를 이용한 데이터 재구조화

- 그룹바이를 이용해 데이터를 재구조화하면, 원본 데이터는 그대로 두고 그룹바이를 통해 새로운 데이터 프레임을 생성할 수 있음
    
- 데이터의 인덱스를 변경하거나 바꾸면 원래의 데이터는 변하지 않음
    
- 원본 데이터를 보존하면서 새로운 그룹바이를 생성할 수 있음
    
- 그룹바이를 통해 여러 개의 서로 다른 값들을 하나의 그룹으로 묶을 수 있음
    

8.데이트타임 자료형

8-1.데이트타임 자료형 이해

- 문자열로 이루어진 값도 유니코드로 변환 가능함
    
- 문자열을 문자열로 변환해도 값 비교는 가능함
    
- 문자열을 유니코드로 변환할 때는 0315와 같이 먼저 오는 건 불가능함
    
- 변환 시, 등록일자를 2022년 3월 15일 이후의 사람만 선택하도록 인덱싱을 사용할 수 있음
    
- 문자열과 같은 옵션도 가지고 있어 인터프리터가 문자열을 유니코드로 변환해줌
    

8-2.날짜 수정

- 등록일자를 1년 후로 변경하고자 함
    
- 변경은 문자열로 이루어진 값의 순서대로 바꿔줌
    
- 정규식을 사용한 문자열을 찾는 것도 가능함
    
- 문자열이 정규식 패턴에 있는 경우, 데이트타임으로 변경해주는 함수를 사용할 수 있음
    
- 이 함수를 통해, '2022년 3월 11일'을 '2026년 1월 11일'로 변경 가능함
    

8-3.연도 추가

- 1년을 추가하고자 함
    
- 연도를 추가하면, 월과 일을 바꿔버리는 것과 같음
    
- 직접 계산하지 않아도 데이트타임 모듈을 통해 간단히 연도를 추가하거나 계산할 수 있음
    

9.데이터 변환과 시간 연산 모듈 이해

9-1.데이터 변환과 모듈 이해

- 데이터 변환 모듈인 '데이터 타임'을 파이썬 공식 문서에서 찾을 수 있음
    
- 이 모듈은 문자와 숫자 사이의 연산을 지원하며, 연산 결과를 문자열로 변환함
    
- 등록 일자를 '데이터 타임'으로 변환했을 때, 변환된 자료형에 맞게 동작하는지 확인 가능
    
- 변환 시 문자열로의 값 비교는 가능하지만, 숫자로의 변환은 불가능함
    

9-2.'데이터 타임' 모듈 사용법

- '데이터 타임' 모듈 내 '타임 델타' 메서드를 사용하여 데이터 변환
    
- '타임 델타'는 초, 마이크로세컨드, 릴리세컨드, 미니타임 등의 단위로 시간의 변화량을 계산
    
- 등록 일자에 '타임 델타'를 적용하여 원하는 연도, 월, 일을 추가할 수 있음
    
- 연산 결과는 다시 문자열로 변환되므로, 원하는 날짜에 대한 변환이 가능함
    

9-3.데이터 변환 모듈의 필요성

- 필요한 시간 계산을 위해 '데이터 타임' 모듈이 필요하며, 이 경우 별도의 자료형을 사용해야 함
    
- 변환 시 원본 데이터를 변경하지 않고, 변환된 자료형에 맞게 데이터를 저장하면 됨
    
- 조회, 분류 등의 기능은 변환된 자료형에 따라 다르게 작동하므로, 필요한 경우에만 사용해야 함
    
- 이 강의의 목적은 데이터 변환 모듈의 활용과 활용을 위한 이해를 돕는 것임
    

10.데이터베이스 정리와 함수 활용

10-1.데이터베이스 정리의 필요성

- 데이터베이스에서 단위, 번호 컬럼의 삭제가 필요함
    
- 인덱스 번호가 1부터 시작하므로 이를 이용해 인덱스를 1증가로 설정하는 방안을 제안함
    
- 중복되는 값은 남겨두지 않고, 정리해 놓는 것이 관리에 용이함
    
- 삭제 대신 '기존 데이터'를 '존재하지 않는 데이터'로 설정하는 것이 가능함
    

10-2.데이터 수정 및 저장

- 데이터를 저장할 때, 원본 데이터를 유지하면서 필요한 형식으로 재구조화함
    
- 원본 데이터에서 '고유한 값'을 제외한 나머지 값을 녹여 새로운 데이터 프레임을 만듦
    
- 저장시 메모리에 날아가는 것이 아니라 실제 파일에 저장되며, 파일의 헤더 내용도 확인이 가능함
    
- 재구조화시 판다스 함수와 넘파이 함수에 따라 데이터 수정이 달라짐
    

10-3.판다스와 넘파이 함수 활용

- 판다스와 넘파이 함수를 이용해 데이터 수정 및 저장, 활용이 가능함
    
- 판다스와 넘파이의 차이점은 판다스는 복잡한 데이터를 간결하게 처리, 넘파이는 데이터 수정이 비교적 간단함
    
- 판다스 함수는 판다스만의 공식 문서에서 이용하며 넘파이는 넘파이 함수의 공식 문서에서 이용함
    
- 판다스와 넘파이를 적절히 사용하면 데이터베이스 관리 및 활용이 더욱 용이해짐
    

11.데이터 분석 및 그룹별 처리 방법

11-1.데이터 변환 및 그룹바이 이해

- 데이터 변환 과정에서 변수는 고유값을 유지하며, 나머지 변수들은 속성에 대한 명시를 통해 접근 가능하게 만듦
    
- 그룹바이는 특정 컬럼 기준으로 특별한 값을 추려서 반환하는 함수임
    
- 그룹바이의 목적은 특정 값이 어떤 조건에 의해 어떤 결과를 가져오는지를 파악하는 것임
    

11-2.그룹바이의 활용과 재구조화

- 그룹바이는 예를 들어 매장의 가격과 판매량 등을 파악할 때 유용하게 사용됨
    
- 가격, 판매량 등 원래의 데이터를 바탕으로 새로운 데이터 프레임을 만들어낼 수 있음
    
- 데이터 프레임의 재구조화 과정에서 중간 과정에서 '에필' 함수를 사용함
    

11-3.피벗 테이블의 활용

- 피벗 테이블은 그룹바이와 비슷한 방식으로 데이터를 재구조화함
    
- 데이터를 재구조화할 때는 원본 데이터를 유지하려는 시도가 중요함
    
- 피벗 테이블은 주로 상품의 가격과 판매량 등을 파악할 때 활용됨
    

12.데이터 처리 및 분석

12-1.데이터 입력 및 변환

- 데이터 입력 및 수정 방법 설명함
    
- 데이터 수정 시 피벗, 티벗 등을 활용하여 기존 데이터 변환 가능함
    
- 필요시 다른 함수나 메소드를 활용하여 복잡한 연산 없이 데이터를 처리할 수 있음
    
- 원본 데이터를 그대로 유지하되, 해석하기 쉽게 변환하는 것이 핵심임
    
- 예시로 원하는 방식으로 데이터를 재구조화하고 사용자에게 돌려주는 방법 설명함
    

12-2.데이터 재구조화

- 원본 데이터를 보존하면서 복잡한 연산을 피하기 위해 파생 변수를 만드는 방법 설명함
    
- 데이터의 수량별 단가를 계산하기 위해 파생 변수를 이용하는 방법 설명함
    
- 모여도 좋을 때는 데이터 엔지니어의 관점에서 대시보드 출력을 고려함
    
- 테이블의 경우 데이터를 재구조화하여 보기 쉽게 만들어주는 방법 설명함
    
- 데이터의 형태에 따라 정형, 반정형, 비정형 데이터로 분류할 수 있음
    

12-3.데이터 타입 이해

- IOT에서 반정형 데이터가 문제가 될 수 있음을 설명함
    
- 반정형 데이터는 규칙에 따라 입력받기 때문에 문제가 발생함
    
- 정형 데이터는 표와 같이 입력받아 수치나 언어로 표현할 수 있음
    
- 정성 데이터는 언어나 문자 등 느낌상 정성적으로 만든 것 같은 데이터임
    
- 데이터의 타입에 따라 처리 방법이 달라짐을 기억해야 함
    

13.데이터베이스 설계와 기능

13-1.데이터베이스의 분류와 특성

- 데이터베이스를 수치형과 범주형으로 분류함
    
- 수치형은 계산 가능한 형태, 연속형은 계산 불가능한 형태임
    
- 범주형은 순서나 등급 등에 따라 가치를 매길 수 있는 형태를 가짐
    
- 각 분류에 따라 데이터베이스의 사용 목적에 따라 차이가 있음
    
- 암묵지와 형식지 간 상호작용이 있음
    

13-2.데이터베이스의 활용과 가치

- 데이터베이스는 정보를 축적하고 전달하는 도구임
    
- 정보를 이용하고 관리하는 데 있어 가치가 있음
    
- 정보의 축적은 기계 가독성, 검색 가능성, 원격 조작성 등의 요소를 포함함
    
- 정보 이용은 검색 가능성, 정보 관리, 정보 기술 발전 측면에서 중요함
    
- 공용 데이터의 관리와 제공이 필요함
    

13-3.데이터베이스의 설계 절차

- 요구조건 분석, 개념적 설계, 논리적 설계, 물리적 설계의 과정을 거침
    
- 데이터베이스 설계는 주로 요구조건 분석과 논리적 설계를 통해 이루어짐
    
- 데이터베이스 설계 시 고려해야 할 사항들에 대해 이해함
    
- 데이터베이스의 설계와 관리는 시스템의 성과에 직결되는 중요한 요소임
    

14.기업 데이터베이스

14-1.데이터 설계와 물리적 설계

- 개념적 설계에서 원하는 그림을 그리는 것과 같다고 생각하면 됨
    
- 물리적 설계에서 개념적 설계를 모델링해 컴퓨터가 이해할 수 있는 형태로 바꾸고, 중복을 정규화하거나 어떤 식별자를 할지 배우는 과정이 논리적 설계에 해당함
    
- 논리적 설계에서 데이터의 정의, 쿼리 작성, 테이프 작성 등도 포함함
    
- 물리적 설계 이후, 설계한 내용을 코딩하고 논리적 설계를 수행함
    
- 논리적 설계의 단계에서 필요한 작업을 암기해야 함
    

14-2.데이터 스키마와 SQL

- 데이터 스키마는 데이터를 설명하고, 데이터 사전은 메타 데이터와 데이터 구조를 포함함
    
- DML은 조작하고 선택하는 용어이고, DCL은 접근 권한을 설정하는 용어임
    
- SQL은 데이터베이스에서 접근하는 표준 언어로, DML과 DCL의 차이를 알아야 함
    
- DML 명령어와 DCL 명령어를 외우면 향후 SQL을 이해하는 데 도움이 됨
    

14-3.데이터 웨어하우스와 오픈소스 빅데이터

- 데이터 웨어하우스는 기업 내 다양한 데이터를 한 곳에 저장하는 곳임
    
- ODS(디지털 테스트 베이스)에서 기존 데이터베이스를 임시로 저장한 후, DW(웨어하우스)로 통합해 저장함
    
- DW 이후, 소규모 데이터베이스로 간다는 점을 이해해야 함
    
- 데이터 웨어하우스와 반대로, 데이터 레이크는 비정형 데이터를 연계해 대용량 처리하는 오픈소스 빅데이터 솔루션임
    
- 데이터 레이크의 본질을 이해해야 함
    

15.빅데이터의 개념과 주요 특징

15-1.빅데이터의 개념과 분류

- 빅데이터는 규모가 방대하고 처리 속도가 빠른 데이터를 의미함
    
- 웨어하우스 데이터와 데이터 워크플로우 등 다양한 형태로 존재함
    
- 데이터 레이크는 웨어하우스에 저장되지만, 데이터 마켓에서는 ETL을 통해 관리됨
    
- 데이터 처리는 원칙적으로 비정형 데이터를 활용하며, 처리 과정에는 추출과 정제가 포함됨
    
- 데이터 워크플로우는 주어진 입력에 대해 처리된 결과를 저장하며, 관리에는 웨어하우스 데이터와 데이터 워크플로우가 포함됨
    

15-2.빅데이터의 활용과 발전

- 빅데이터의 활용은 다양한 분야에서 이루어짐
    
- 빅데이터는 가치 창출, 편의성 향상, 등록까지 가능함
    
- 빅데이터는 복잡성과 양으로 인해 분야별로 적절한 처리 및 관리 방법이 필요함
    
- 클라우드 컴퓨팅 기술 발달에 따라 빅데이터 처리 비용 절감이 가능해짐
    
- 빅데이터는 고객의 요구에 따라 다양한 형태로 제공되며, 이를 위한 새로운 기술 및 관리 프로세스 필요성을 부각함
    

15-3.빅데이터 가치와 영향

- 빅데이터의 가치는 원칙적으로 데이터가 사용자의 요구에 부합할 때 창출됨
    
- 빅데이터의 가치는 시각화, 가변성, 신뢰성 등을 포함함
    
- 빅데이터의 영향은 사용자의 요구에 따라 다양하며, 사용자의 라이프스타일, 환경 등에 따라 달라짐
    
- 빅데이터 처리는 사전 처리와 사후 처리의 두 가지 방식이 있으며, 이 중 사후 처리가 주로 이루어짐
    
- 빅데이터의 사용은 사용자의 요구에 따라 변할 수 있으며, 이에 따른 관리와 평가가 필요함
