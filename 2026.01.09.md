1.파이썬 기초 이해

1-1.파이썬의 기본 개념

- 파이썬은 인트, 불, 스트링, 리스트, 투플, 레인지 등의 클래스를 기반으로 개발됨
    
- 파이썬에서 '파이썬'이라는 주소를 통해 다양한 인스턴스 생성 가능
    
- 클래스를 사용해 데이터를 일괄 처리하기 위해 클래스를 만들 수 있음
    
- 클래스는 설계도와 같으며, 인스턴스를 통해 데이터를 관리함
    
- 파이썬에서 클래스는 변수와 함수를 포함한 객체임
    

1-2.클래스와 인스턴스

- 클래스는 대문자로 시작해 띄어쓰기를 하며, 대체로 대문자로 시작함
    
- 클래스에 속성과 함수를 추가해 인스턴스 변수와 인스턴스 함수를 만들 수 있음
    
- 인스턴스는 클래스를 통해 생성되며, 각 인스턴스는 고유의 변수 값을 가짐
    
- 파이썬에서 클래스와 인스턴스를 통한 데이터 관리의 예로 자동차 정보 입력과 중고차 판매 데이터를 들 수 있음
    
- 파이썬에서 클래스는 메모리에 영향을 주지 않고, 인스턴스마다 데이터를 별도로 관리함
    

1-3.파이썬 코드의 일관성과 유연성

- 파이썬에서 인스턴스를 생성할 때, 클래스와 인스턴스의 변수와 함수를 함께 사용함
    
- 파이썬 코드는 일관성을 유지하며, 코드의 재사용성을 높임
    
- 파이썬에서 클래스를 통해 데이터를 관리할 때, 인스턴스는 데이터를 일괄적으로 처리함
    
- 파이썬에서 데이터 처리 관련 프레임워크를 활용해 인스턴스마다 데이터를 일괄 처리함
    
- 파이썬에서 클래스를 만들고, 변수와 함수를 추가해 인스턴스를 생성할 수 있음
    

2.클래스 변수의 활용과 관리

2-1.클래스 변수의 개념과 용도

- 클래스 변수는 공유 변수로서 클래스의 인스턴스에 대한 정보를 저장함
    
- 클래스의 인스턴스가 생성될 때마다 생성자의 정보를 관리하며, 이를 위해 클래스 변수를 사용함
    
- 클래스 변수는 인스턴스의 생성, 관리가 모두 가능한 공유 변수로 간주됨
    
- 클래스 변수를 통해 모든 인스턴스에 대한 정보를 한 곳에 모을 수 있음
    

2-2.클래스 변수의 예시와 활용

- 연지중고차에서 차의 번호를 관리하기 위해 '토털럼' 클래스를 생성하고 '카' 인스턴스에 해당 번호를 부여함
    
- '모닝' 인스턴스가 생성될 때마다 '토털럼'의 값에 1을 추가함
    
- '테슬라' 인스턴스도 생성될 때마다 '모닝'의 값에 2를 추가함
    
- 이를 통해 '모닝'의 인스턴스가 '카'에 판매된 차량의 번호를 기억하게 됨
    

2-3.클래스 변수의 특징과 중요성

- 클래스 변수는 인스턴스의 생성과 관리 모두를 가능하게 하는 공유 변수임
    
- 클래스 변수를 통해 각 인스턴스에 대한 정보를 하나의 공간에 모을 수 있음
    
- 이를 통해 핵심 정보를 명료하게 유지하면서도 효율적인 데이터 관리를 가능하게 함
    
- 특히, 연지중고차와 같은 회사에서는 차량 판매 등의 실무에서 유용하게 활용됨
    

3.객체지향 프로그래밍과 캡슐화

- 설계도를 통해 각 인스턴스의 속성과 행동을 명확하게 이해 가능

- 캡슐화는 객체의 속성과 메소드를 숨기는 과정
    
- 이 과정을 통해 객체의 안정성과 완결성, 유지 보수성이 높아짐
    
- 설계자는 캡슐화를 통해 인스턴스의 안정성과 재사용성을 보장
    
- 또한, 코드의 안정성과 재사용성을 높여 유지 보수의 용이성을 향상시킴
    

3-3.객체 지향 프로그래밍의 장점

- 객체 지향 프로그래밍은 캡슐화를 통해 데이터와 연산을 숨길 수 있음
    
- 이 과정을 통해 설계자는 인스턴스의 상세 정보를 보호하고 안정성을 확보
    
- 이는 시스템의 안정성과 재사용성을 높여주는 장점 제공
    
- 이는 부수적인 효과로서 설계자에게 추가 혜택과 보안을 제공

- 클래스 변수는 클래스의 속성을 저장하며, 클래스 생성 시 값을 변경할 수 있음
    
- 클래스의 특수한 함수인 '단위'는 클래스 생성을 위한 생성자 함수임
    
- 클래스 함수는 클래스의 관리나 작용을 위한 기능들을 표현함

- 클래스 메소드는 인스턴스 메소드와 구분하여, 클래스 함수를 이용하여 클래스의 동작을 수정할 수 있음
    
- 클래스 메소드 안에 데코레이터(어노테이션)가 있을 경우, 클래스 메소드의 동작이 먼저 이루어짐
    

4-3.클래스 메소드의 응용과 복잡한 문제 해결

- 클래스 메소드는 복잡한 문제를 해결하기 위해 사용되며, 각각의 클래스에서 고유한 동작을 표현함
    
- 강의에서 다룬 문제들(인스턴스 체크)에서 클래스 메소드를 활용하면, 문제를 더 효율적으로 해결할 수 있음
    
- 이 과정에서 클래스 메소드가 제대로 작동하는지, 클래스의 속성(변수)이 올바르게 이해되어 있는지 확인하는 과정이 중요함
    

5.파이썬 언어 기반 클래스와 함수 이해

5-1.파이썬 언어의 기본적인 이해

- 파이썬은 C언어와 유사하게 만들어져 있어 공통적인 이해를 돕고 있음
    
- 파이썬 언어의 기본적인 구조를 이해하는 것이 중요하며, 객체와 클래스의 개념을 잘 파악해야 함
    
- 클래스와 객체를 사용하여 데이터와 연산자를 함께 묶음
    
- 클래스 내부에 있는 변수는 각각의 데이터를 담당하며, 이들 변수들의 동작을 통해 데이터 처리가 이루어짐
    
- 파이썬 언어의 기본 구조를 이해하면 C언어나 C++에서도 더 빠르게 익힐 수 있음
    

5-2.파이썬에서의 함수와 메소드 이해

- 파이썬에서의 함수는 특정한 동작을 수행하는 코드의 집합을 의미함
    
- 문자열을 받아 특정한 패턴을 확인하고, 해당 패턴이 맞을 시 '참'을, 아니면 '거짓'을 반환하는 함수 등이 있음
    
- 파이썬에서의 메소드는 클래스 내부에서의 동작을 제어하는 데 사용됨
    
- 메소드를 통해 클래스의 변수를 조작하고, 이는 클래스의 동작을 제어하는 데에 필수적임
    
- 파이썬에서의 함수와 메소드는 각각 다른 작업을 수행하므로 적절하게 사용되어야 함
    

5-3.파이썬에서의 인스턴스 함수와 스태틱 메소드 이해

- 파이썬에서 클래스 함수와 인스턴스 함수의 차이를 이해하는 것이 중요함
    
- 인스턴스 함수는 클래스의 인스턴스를 참조하여 사용하는 함수이며, 클래스 안에 두어도 사용 가능함
    
- 스태틱 메소드는 클래스와 인스턴스에 모두 결부되어 클래스의 상태를 나타내는 함수임
    
- 스태틱 메소드는 클래스의 변수와 인스턴스 변수에 결부되어 있어, 논리적으로는 클래스 안에 넣어두는 것이 적절함
    
- 그러나 실제로 클래스를 논리적으로 설계하는 것이 아닌, 클래스를 데이터와 연산자로 묶는 경우, 클래스 외부에 두어도 문제 없음
    

6.기반 커스터마이징을 위한 클래스 설계

6-1.기반 커스터마이징의 개념

- 기반 커스터마이징은 클래스 설계에서 중요한 부분임
    
- 클래스의 모든 변수와 메소드는 그 클래스를 사용하는 인스턴스의 행동을 제어하는 데 사용됨
    
- 인스턴스는 각각의 값과 조건에 따라 다른 동작을 수행하도록 설계됨
    
- 클래스나 인스턴스는 별개의 실행을 통해 데이터를 제어할 수 있어야 함
    

6-2.클래스 메소드와 인스턴스 변수

- 클래스 메소드는 클래스의 주소를 통해 값을 전달하고, 인스턴스 변수를 통해 실제 값의 접근이 가능하게 함
    
- 인스턴스 변수는 메모리의 인스턴스에 연결되어, 실제 인스턴스의 값을 가리킴
    
- 인스턴스 메모리 주소가 없을 때는 메모리 인스턴스의 메모리 주소를 전달하는 것에 주안점을 둠
    
- 클래스 메소드는 별도의 인스턴스 생성을 통해 동작이 제어되도록 설계됨
    

6-3.인스턴스 변수의 설정과 활용

- 클래스 메소드를 통해 인스턴스 변수를 생성하고, 이를 통해 실제 값을 전달할 수 있음
    
- 이 인스턴스 변수는 원하는 조건에 따라 다른 동작을 수행하도록 설계됨
    
- 각 인스턴스는 별개의 실행을 통해 데이터를 제어하고, 특정 동작을 수행하도록 설정할 수 있음
    
- 이런 설정을 통해, 각 인스턴스는 개별적으로 동작하도록 강하게 제어할 수 있음
    

7.인스턴스 설계

- 인스턴스의 변수명을 지을 때 참인지 거짓인지 판단하는 것과 유사하게 '이즈?'로 리턴하는 함수 이름을 생각해볼 것
    
- 변수명에 논리적으로 포함된 내용은 정적 함수를 정의함

7-2.인스턴스 변수 및 함수 설정

- '카타입'과 '국산차 여부', '희망 색깔' 등을 파라미터로 입력받음
    
- '체크'라는 인스턴스 메서드를 추가해 각 인스턴스에 따라 다른 동작을 하도록 설정함
    
- 셀프 키워드를 인스턴스마다 추가해 인스턴스의 메모리 주소를 전달함
    

7-3.인스턴스 함수 정의 및 실행

- '체크'라는 인스턴스 메서드를 추가해 각 인스턴스에 따라 다르게 동작하도록 설정함
    
- '모닝'이라는 인스턴스 메서드를 통해 정기 검사 여부를 판단함
    
- 인스턴스의 국산차 여부, 희망 색깔 등을 파라미터로 입력받음
    
- 셀프 키워드를 인스턴스 메모리 주소에 삽입해 인스턴스 함수를 완성함
    
- '체크'와 '모닝' 인스턴스 메서드를 통해 각 인스턴스에 따라 다른 동작을 하도록 설정함
    

8.객체지향 프로그래밍과 디폴트 파라미터

- 상속은 부모 클래스의 모든 변수와 메소드를 포함하며, 이 변수와 메소드는 초기화 또는 재생성 시점에 의해 결정됨
    
- 부모 클래스에서 상속받은 변수와 메소드는 초기화하거나 재생성될 때마다 값을 재설정하거나 변경할 수 있음
    
- 상속 메소드는 코드 재사용을 가능하게 하며, 이는 유지 보수의 용이성을 높여줌

13.상속과 셀프

13-1.상속과 셀프 이해

- 상속을 통해 부모 클래스의 기능을 물려받을 수 있지만, 자식 클래스의 기능을 새로 작성할 수도 있음
    
- 상속을 표현할 때는 자식이 부모에게 가는 방식으로 화살표로 나타냄
    
- 부모에는 있는 것들은 자식에게 고스란히 물려주지만, 자식이 갖고 있는 별도의 것들도 있음
    
- A가 B를 참조한다는 방식으로 상속을 표현할 수 있음
    
- 프로그래밍 언어에서는 화살표 방식을 통해 인스턴스의 객체를 확인하고 동작시킴
    

13-2.인스턴스 변수와 셀프

- 인스턴스 변수를 통해 각 객체의 메모리 주소를 확인할 수 있음
    
- 인스턴스 변수에 셀프라는 파라미터를 넣으면, 인스턴스의 메모리 주소에서 셀프를 참조하도록 함
    
- 셀프는 메모리 주소로 설계도 안에 있는 인스턴스 메소드를 동작시키기 위한 키워드임
    
- 셀프는 메모리 주소 절대 변하지 않고, 인스턴스를 통한 변수 참조로 작용함
    
- 파이썬에서는 객체를 메모리 주소로 위치시키고, 같아 보이는 객체는 같은 주소에 위치한 객체임
    

13-3.파이썬의 객체 주소와 메모리 공간

- 파이썬에서 객체는 메모리 주소로 위치하며, 같은 객체는 같은 메모리 주소에 위치함
    
- 셀프 변수는 메모리 주소 절대 변하지 않고, 인스턴스를 통한 변수 참조로 작용함
    
- 파이썬에서 메모리 공간을 아끼기 위해 메모리 주소 절대 변하지 않는 인스턴스 변수를 사용함
    
- 설계도 안의 인스턴스 메소드를 실행할 때, 인스턴스 변수를 통해 메모리 주소를 전달함
    
- 파이썬에서는 객체를 메모리 주소로 위치시키고, 같아 보이는 객체는 같은 주소에 위치한 객체임
    

14.값 비교와 기능

14-1.값 비교 메소드

- 값 비교할 때 '더블언더바 메소드' 사용함
    
- 값 비교 연산자가 '==', '==지 않다' 등의 의미를 가짐
    
- 값을 비교할 때 '==' 사용이 중요함
    
- 파이썬 인터프리터는 '=='를 통해 값을 비교함
    
- 값 비교 연산자는 실제로는 '==', '==지 않다' 등의 의미로 사용됨
    

14-2.더블언더바 메소드

- '더블언더바 메소드'는 값을 비교할 때 사용하는 함수임
    
- 더블언더바 메소드는 '==', '==지 않다' 등의 의미를 가짐
    
- '=='는 실제 값의 비교를, '==지 않다'는 값의 불일치를 의미함
    
- 모닝과 트럭의 배기량을 비교할 때, '=='를 사용함
    
- 모닝과 트럭의 배기량을 비교하여 결과를 리턴함
    

14-3.값 비교와 출력

- 모닝과 트럭의 배기량을 비교하여 결과를 출력함
    
- 모닝과 트럭의 배기량을 비교하는 '더블언더바 메소드'를 작성함
    
- 이 메소드는 '==', '==지 않다' 등의 의미로 값을 비교함
    
- 값 비교 연산자는 파이썬의 '=='와 유사함
    
- 더블언더바 메소드를 이용하여 값 비교와 출력을 코드로 구현함
    

15.클래스와 메소드 이해

15-1.클래스와 메소드의 개념

- 클래스와 객체를 활용한 프로그래밍의 기본 단위인 클래스와 메소드를 이해함
    
- 클래스와 메소드는 각각 데이터와 연산을 저장하고, 실행하는 도구임
    
- 클래스는 데이터와 연산자로 구성되며, 메소드는 연산자를 통해 데이터를 처리함
    
- 클래스는 여러 개의 메소드로 이루어져 있고, 메소드는 클래스 내에 존재함
    
- 클래스와 메소드의 개념을 이해하면 프로그래밍 언어를 이해하는데 도움이 됨
    

15-2.메소드의 종류와 특징

- 메소드는 연산자의 종류에 따라 다양한 유형으로 나뉨
    
- 기본 메소드에는 더블언더바 메서드, 던더 메서드, 인터섹션 메서드 등이 있음
    
- 더블언더바 메서드는 두 개 이상의 값을 받음, 던더 메서드는 한 개의 값을 받음
    
- 던더 메서드는 입력값의 개수에 따라 그에 따른 연산이 달라짐
    
- 인터섹션 메서드는 입력값과 출력값을 서로 비교함
    

15-3.메소드의 활용

- 메소드는 프로그래밍에서 필수적인 도구로, 데이터를 처리하고 분석하는 데 활용됨
    
- 클래스와 메소드를 활용하면 의도하는 대로 값을 조정하거나 분석할 수 있음
    
- 클래스와 메소드의 개념을 이해하면 프로그래밍의 기초를 배우는데 도움이 됨
    
- 특히, 객체 지향 프로그래밍에서는 클래스와 메소드의 이해가 필수적임
    
- 따라서, 메소드를 정확하게 이해하고 활용하는 능력은 프로그래밍 실력에 큰 도움이 됨
    

16.객체지향 프로그래밍과 캡슐화 이해

16-1.객체지향 프로그래밍의 장점과 특징

- 객체지향 프로그래밍은 공통된 특징들을 클래스로 만들 수 있음
    
- 불필요한 세부사항들을 따로 작성하는 대신, 가장 본질적이고 공통적인 부분만을 추출해 클래스로 만들 수 있음
    
- 클래스를 기존의 것 재활용해 새로운 클래스를 작성하는 문법률, 클래스 간 공유하는 속성과 기능을 반복적으로 정의할 필요 없이, 한번만 쓰고 간편하게 재수용이 가능함
    
- 필요한 경우에 몇 개 바꿔야 되는 부분은 부모와 같다면, super 명령어를 가지고서 그 안에 있는 이 함수 가져다 쓰는 것
    

16-2.다형성과 캡슐화의 이해

- 다형성은 객체의 속성이나 기능이 맥락에 따라 다르게 역할을 수행함
    
- 캡슐화는 서로 연관 있는 속성과 기능들을 하나의 캡슐로 만들어 데이터를 외부로부터 보호함

16-3.은닉성과 트릭

- 개발자들은 은닉성을 구현하기 위해 트릭을 사용함
    
- 클래스 안에 은닉성 선언을 해놓으면, 실제로는 클래스 안에서 숨겨져 있지만, 실제로는 클래스 안에 숨겨져 있지 않음
    
- 은닉성 구현에 대한 개별적인 방법은 파이썬에서는 다 주어져 있지 않지만, 언더바를 앞에다가 적어주는 것으로 가능함

5.파이썬의 클래스와 함수 관리

5-1.클래스와 함수의 기본 이해

- 파이썬에서 클래스와 함수라는 개념을 이해하고, 이를 활용하여 데이터를 관리함
    
- 함수는 숨겨져 있어야 하는 속성들을 처리하고, 특정 값을 가져다 쓰도록 설정함
    
- 숨기기하고 싶은 속성의 값을 그대로 유지하면서, 필요한 경우에만 해당 속성을 변수처럼 사용하도록 설정함
    
- 이 때, 속성의 값은 직접 조정할 수 있지만, 변수처럼 사용할 수 있게 될 수 있음
    

5-2.이중으로 숨기는 방법

- 하나의 변수를 숨기고 싶은 속성의 값을 이용하여, 이중으로 숨기는 방법을 설명함
    
- 이 방법은 원래의 변수를 숨기는 것이 아니라, 변수의 값을 조정하여 숨기고 싶은 속성을 변수처럼 설정함
    
- 숨기고 싶은 속성의 값을 세트로 설정하면, 해당 변수는 속성의 값으로 사용되면서 그 값을 직접 조정할 수 없음
    
- 하지만, 속성의 값을 조정하여 원하는 조건에만 값을 대입하면, 숨겨진 속성의 값은 그대로 유지되면서도, 해당 변수는 다른 속성의 값으로 사용됨
    

5-3.속성의 기본 이해와 활용

- 속성은 숨겨져 있어야 하는 속성을 변수처럼 설정하고, 이 변수를 통해 조정하여 숨겨져야 하는 속성을 숨기지 않음
    
- 속성의 값은 직접 조정할 수 있지만, 변수처럼 사용할 수 있게 될 수 있음
    
- 예를 들어, "비밀번호"라는 속성을 숨기고 싶을 때, 해당 속성을 변수처럼 설정하면, 숨겨져 있는 속성의 값은 그대로 유지되면서도, 해당 변수는 다른 속성의 값으로 사용될 수 있음
    
- 이 방법은 데이터의 보안성을 높이는 데 효과적이며, 특히 시스템의 보안을 강화하는 데 도움이 됨
    

6.변수 보안

6-1.속성 보안 이해

- 속성에 직접 접근하는 건 절대 금지됨
    
- 변수를 조회/설정할 수 있는지, 수정할 수 있는지에 따라 접근 제한함
    
- 접근 제한을 위해 변수에 보안을 강화하는 방식을 씀
    
- 예를 들어, '김현지'라는 이름과 비밀번호를 설정할 수 있게 함
    
- '비밀번호'에 속성의 '이변형'을 부여해 접근을 제한함
    

6-2.속성 보안 구현

- '비밀번호'를 '새 패스워드'로 변경하도록 함수를 만듦
    
- '새 패스워드'에 '변수명'을 부여해 속성 보안을 강화함
    
- '변수명'에 접근 시 '변수명'을 부여해 수정을 못 하게 함
    
- '변수명'에 프로퍼티를 추가해 변수처럼 부르지만, 보안을 강화함
    
- 이렇게 하면, 해커가 변수를 수정하더라도 프로퍼티를 변경하면 안 됨
    

6-3.보안의 장점

- 속성에 '이변형'을 부여해 수정을 못 하게 함
    
- '새 패스워드'의 값에 따라서 '변수명'을 수정해도 안 됨
    
- 이는 '비밀번호'가 수정되면 다른 사람에게 전달되더라도 수정 불가하게 함
    
- 예를 들어, '비밀번호'를 대입할 때 조건을 넣어 수정을 못 하게 함
    
- 즉, '변수명'에 접근하는 것을 제한해 보안을 강화함
    

7.객체 지향 프로그래밍과 보안

7-1.객체 지향 프로그래밍의 이해

- 객체 지향 프로그래밍은 실제 세계를 모사한 프로그래밍 모델임

- 객체 제한 프로그래밍을 통해 데이터의 외부 접근을 제어하고 보안성을 높임
    
- 예를 들어, 파이썬의 속성 이름을 숨겨서 보안성을 향상시킬 수 있음
    
- 객체 지향 프로그래밍은 조건문을 통해 프로그램의 동작을 제어하고 이론적인 접근을 방지함

- 프로그래밍 언어마다 스피킹 메소드의 방법이 다름
    
- 프로그래밍 언어에 따라 스피킹 메소드를 활용한 재사용성은 다름
    
- 스피킹 메소드를 통해 코드의 재사용성과 유연성을 높일 수 있음
    

8-2.은닉성과 상속

- 은닉성은 프로그래밍 언어에서 코드의 기능을 숨기는 것을 의미함
    
- 은닉성은 코드의 재사용성과 안정성을 높이기 위해 필요함

- 상속을 통해 재사용성과 유연성을 높일 수 있고, 안정성을 강화할 수 있음
    
- 파이썬은 다중상속을 허용하여 코드의 재사용성과 안정성을 높임
    

8-3.다중상속과 오버라이딩

- 상속을 통해 기능을 한 곳에서 다른 곳에서 다시 한번 사용할 수 있음
    
- 오버라이딩은 프로그래밍 언어에서 한 개 이상의 값을 가지는 것을 의미함
    
- 오버라이딩을 통해 다양한 값을 가지도록 코드를 작성할 수 있음
    
- 파이썬은 다중상속을 허용하여 코드의 재사용성과 안정성을 높임
    

9.파이썬의 다중상속과 맹글링 이해

9-1.다중상속과 그 필요성

- 다중상속은 클래스 간 속성과 기능을 공유하기 위함임
    
- 파이썬에서 다중상속은 허용되지만, 다중상속 받을 때 중복되는 요소는 제거해야 함
    
- 다중상속을 받을 때, 클래스 네임에 중복되는 요소가 있으면 이를 잘 관리해야 함
    
- 중복되는 요소가 있을 경우, 한 개의 클래스에서 다른 클래스의 속성을 가져와 상속받을 수 있음
    

9-2.다중상속과 맹글링

- 다중상속을 받을 때, 값을 묶어주는 맹글링이라는 기법을 사용하여 재사용 가능하게 함
    
- 맹글링을 통해 값의 손실을 최소화하면서, 다중상속을 효과적으로 사용할 수 있음
    
- 맹글링된 값은 조심스럽게 다뤄져야 하며, 파이썬에서는 보안을 위해 맹글링을 의도적으로 사용하기도 함
    
- 맹글링은 다중상속의 의도치 않은 결과를 최소화하면서, 다중상속을 적절히 활용하도록 도와줌
    

9-3.다중상속과 맹글링의 활용

- 다중상속과 맹글링을 적절하게 사용하면, 코드를 줄이고, 효율적인 프로그래밍이 가능해짐
    
- 파이썬에서 다중상속은 허용되지만, 다중상속 받을 때 맹글링을 통해 가치를 재사용할 수 있음
    
- 맹글링을 통해 불필요한 속성에 대한 손실을 최소화하면서, 다중상속을 효과적으로 활용할 수 있음
    
- 파이썬에서는 맹글링을 통해 다중상속을 조절하고, 원활한 코드 실행을 도모함
