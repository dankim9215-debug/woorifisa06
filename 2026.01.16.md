- 연속형 변수를 사용할 때는 바플라, 연속형 변수를 사용할 때는 카운트플롯을 사용함

6.데이터 분석을 통한 시각화

6-1.데이터 분석의 중요성과 방법

- 데이터 분석을 통해 연속적인 분포를 확인할 수 있음
    
- 픽스의 성별 데이터를 분석하여 그 값을 추출하고, 이를 시각화 패키지로 출력함
    
- 파라미터를 조정하여 결과를 보다 정확하게 추출할 수 있음
    
- 시각화 원하는 목표를 설정하고, 이를 적절한 플랫폼을 통해 시각화 패키지로 생성
    
- 데이터에 대한 도메인 지식을 바탕으로 시각화 패키지 생성 가능
    

6-2.시각화 플롯의 선택과 보정

- 시각화 패키지에서 제공하는 파라미터를 활용하여 플롯의 보정이 가능
    
- 파라미터를 바꿔가며 원하는 목표에 부합하는 결과를 확인
    
- 결과의 적합성 확인을 위해 직접 파라미터를 수정하며 시각화 원하는 결과를 보여줄 수 있음
    
- 시각화 원하는 목표를 충족시키지 못하는 경우, 필요한 파라미터를 수정하거나 보정
    
- 데이터 분석 결과를 바탕으로 시각화 패키지 생성 및 수정
    

6-3.시각화 결과의 해석과 시각화

- 시각화 결과를 해석하여 원하는 목표를 충족시킨 경우, 시각화 패키지의 결과로 사용
    
- 시각화의 목표에 부합하지 않는 결과가 나타나면, 필요한 부분을 수정
    
- 시각화 결과를 통해 데이터의 특성을 파악하고, 이를 바탕으로 분석을 진행
    
- 시각화 패키지 생성 및 수정을 통해 데이터에 대한 통찰력을 얻을 수 있음
    
- 시각화 결과를 바탕으로 새로운 데이터를 생성하거나, 기존 데이터를 수정 가능

12.지도작성

12-1.지도작성 프로세스 이해

- 입력 데이터의 유형에 따라 지도의 형식과 개략적인 구조가 달라짐

12-2.지도작성의 세부 과정

- 입력 데이터의 유형에 따라 로케이션, 색깔, 견해 등을 지도에 표시하는 과정이 다름
    
- 로케이션을 지도에 표시할 때, 색깔은 범주별로 명목형 데이터를 활용함
    
- 색깔은 데이터의 유형에 따라 '색깔' '숫자' '연속형 변수' 등으로 변경됨
    
- 견해는 '견해'를 통해 지도에 의견을 표시하는 것으로, 추가 정보를 포함할 수 있음
    
- 다양한 색깔과 범주를 사용하여 로케이션을 지도에 표시할 수 있음
    

12-3.지도작성 시 주의점

- 데이터의 유형에 따라 지도의 형식과 개략적인 구조를 예상하고 작성해야 함
    
- 로케이션, 색깔, 견해 등을 지도에 표시할 때, 데이터의 유형에 맞게 표시해야 함
    
- '견해'를 표시할 때는 '견해'를 통해 지도에 의견을 표시할 수 있는 다양한 방식을 활용할 수 있음

- 로케이션, 색깔, 견해 등의 정보를 지도에 표시할 때, 단순화하고 명확하게 표현하는 것이 중요함

- 파이썬을 설치할 때 기본 환경을 선택하지 않고, 프로젝트별로 폴더를 생성함
    

14-2.가상환경을 통한 프로젝트 관리

- 가상환경은 파이썬을 설치할 때 원본 파이썬을 참조하여 별도의 패키지로 관리함
    
- 가상환경의 이름을 SP로 지정하고, 폴더를 생성함
    
- 가상환경 안에서 파이썬을 실행할 때, 가상환경의 이름을 적어주면 프로젝트가 실행될 때까지 명령어로 프로젝트를 제어할 수 있음
    
- 가상환경을 만들고, 필요한 패키지들을 분리하여 프로젝트를 관리함
    
- 프로젝트별로 폴더를 생성하여 파이썬 패키지들을 설치하고, 가상환경을 통해 프로젝트를 관리함
    

14-3.가상환경을 통한 프로젝트 관리

- 가상환경은 파이썬을 설치할 때 필요로 하는 패키지들을 분리하여 관리함

16.스트림릿 사용법

16-1.스트림릿의 개념과 주요 기능

- 스트림릿은 표준 출력 방식이며, 화면에 출력된 결과를 제어할 수 있음
    
- 스트림릿은 시트, 스테이지, 스트림릿 세트 등을 사용하여 정보를 입력받고 출력함
    
- 스트림릿은 텍스트와 동적 내용을 모두 포함하여 출력함
    
- 표준 출력 방식이 아니라, 여러 형식에 대해 변환하여 출력할 수 있음
    
- 코드의 복잡성을 줄이고, 복잡한 데이터를 간결하게 표현할 수 있음
    

16-2.스트림릿의 출력 형식

- 스트림릿은 문자열, 텍스트, 동적 데이터 등을 출력함
    
- 메모리에 저장된 문자열을 출력할 경우, 메모리의 문자열이 화면에 뜨도록 설정함
    
- 스트림릿의 출력 형식은 '라이트', '디스플레이', '마크다운', '캐션', '레이텍스' 문법을 따름
    
- '라이트'로 표현된 부분은 화면에 표시되고, '디스플레이'로 표현된 부분은 화면에 표시되도록 설정함
    
- '마크다운'은 복잡한 수식을 텍스트로 표현하도록 함
    

16-3.스트림릿의 동작과 조작

- 스트림릿은 코드를 작성하고 실행함으로써 결과를 출력함
    
- 스트림릿은 메모리에 저장된 문자열을 화면에 표시하고, '컨트롤 S'로 끄면서 바로 반영됨
    
- 스트림릿은 저장을 통해 결과를 메모리에 남기고, '컨트롤 S'로 끄면서 바로 반영됨
    
- 스트림릿의 결과는 메모리에 저장된 문자열로, 메모리에 저장되지 않으면 화면에 표시되지 않음

6-1.파이낸스 데이터 리더 설치

- 바이너리 파일 형식인 엑셀 파일을 받아와 바이트 단위로 데이터를 쪼개서 보내주는 모듈을 가져옴
    
- 스트림릿, 다스 임포트, 코리아나이즈 매틀럴 라이브 설치 등 추가 설치 필요함
    
- 파이낸스 데이터 리더 설치, KRX 종목 코드를 통해 종목 정보와 시가총액 등 다양한 데이터를 불러올 수 있음
    
- 가상환경에 경로를 설정해줌으로써, 다른 프로젝트에서 사용할 수 있게 함
    

6-2.파이낸스 데이터 리더 사용법

- 파이낸스 데이터 리더 설치 후, '설정' 버튼을 눌러 '서버에 저장'함
    
- 가상환경에 '서버'로 설정한 후, '디폴트 경로'로 지정해주면 터미널을 열어도 '서버' 경로로 프로젝트를 생성할 수 있음
    
- '서버' 경로로 설정하면, 다른 프로젝트에서 '디폴트 경로'로 지정해주면 항상 '서버' 경로로만 프로젝트를 생성하게 됨
    
- 파이낸스 데이터 리더 설치 후, '서버' 경로로 가상환경에 들어가 '디폴트 경로'로 지정해주면 '서버'로 불러옴
    
- '서버'로 들어가면 '디폴트 경로'로 들어오는 설정이 적용되어 프로젝트 생성에 영향을 미침
    

6-3.파이낸스 데이터 리더 코드 작성

- 파이낸스 데이터 리더로 엑셀 파일의 바이너리 형식의 데이터를 받아와 시각화하는 코드 작성 필요함
    
- 코리아나이즈 매틀럴 라이브 설치 후, '서버'로 들어가면 '서버' 경로로 지정해주면 항상 '서버' 경로로만 프로젝트를 생성하게 됨
    
- 파이낸스 데이터 리더 설치 후, '서버' 경로로 들어가면 '디폴트 경로'로 지정되어 프로젝트 생성에 영향을 미침
    
- 바이너리 파일을 받은 후, 판다스의 데이터 프레임으로 변환하는 코드를 작성해야 함

- 입력한 종목 코드를 검색하여, 해당 종목 코드의 데이터 프레임을 결과로 출력함
    
- 이 과정에서 강의에서 제공하는 코드에 포함된 전처리 과정이 자세히 설명되어 있음
    

7-3.그래프 그리기 및 엑셀 파일 다운로드

- 입력받은 데이터에 대해 분모 코드를 찾아 결과로 출력함
    
- 분모 코드를 통해 종목 코드와 시작일, 끝날지를 0월 1일로 포맷팅함
    
- 제목은 "종가 추이"로 설정함
    
- 다운로드 버튼을 누르면 엑셀로 결과 파일을 다운로드받을 수 있음
    
- 엑셀로 다운로드받은 후, 바이너리 파일 형식의 데이터 프레임으로 변환함
    
- 강의에서 제공하는 코드에 포함된 전처리 과정은 엑셀의 그래프 그리기 메서드를 사용하여 결과를 출력하는 과정과 유사함
    

8.파이썬 코드를 활용한 날짜 처리 및 입력

8-1.파이썬 코드를 활용한 날짜 입력과 조회

- 파이썬 코드를 활용해 날짜를 입력하고 조회하는 방법을 설명함
    
- 특정 날짜를 조회하기 위해 사용되는 함수들을 설명함
    
- 날짜 입력에 있어 선택기능 사용 방법에 대해 설명함
    
- 입력된 날짜를 활용해 결과를 출력하는 방법에 대해 설명함
    

8-2.파이썬 코드를 활용한 날짜 수정 및 삭제

- 파이썬 코드를 활용해 날짜를 수정하고 삭제하는 방법에 대해 설명함
    
- 날짜 수정 시, '그리기' 버튼을 사용하는 방법에 대해 설명함
    
- 날짜 삭제 시, 직접적으로 삭제하는 방법에 대해 설명함
    
- 날짜 수정과 삭제 시, 출력되는 결과에 대해 설명함
    

8-3.파이썬 코드에 관한 실습 및 피드백

- 강사의 실습을 통해 파이썬 코드에 대해 이해하고, 문제 해결 능력을 향상시킴
    
- 학습 동료들과 코드에 대한 피드백을 통해 코드 작성 능력을 향상시킴
    
- 학습 동료들로부터 받은 피드백을 통해 학습 진행 상황을 파악함
    
- 강사가 학습자에게 제공한 가이드를 따라 코드를 작성함
    

9.사이드바 배포

9-1.사이드바 모듈로의 전환

- 데이트 타임을 변수로 변경하여 데이터를 재뉴얼함
    
- 파라미터를 변경하여 매뉴얼라이브를 설정함
    
- "최신 데이트를 가져오라"를 선택하여 셀렉티드 데이트를 생성함
    
- 코드의 "라이브러리" 메뉴에서 "라이브러리 폴더"로 들어가서 파일을 업로드함
    
- 최종적으로 서비스를 마레오카에 업로드하는 것이 중요함
    

9-2.사이드바 모듈과 화면

- 사이드바에서 직접 값을 입력하는 방법을 설명함
    
- "성분"을 입력하고 "제목"을 변경하여 값들을 입력함
    
- 화면 사이드바에 값을 입력하고 화면을 사용하도록 설정함
    
- "사이드바에 값 입력하고 화면 사용"을 선택하여 실행함
    
- 입력한 값을 바탕으로 메인 화면에서 모든 작업을 진행함
    

9-3.배포 방법과 점수 산출

- 배포 방법을 설명하고, 파일을 업로드하여 서비스를 시작함
    
- 1년에 10억 원 정도 걸리는 비용이 발생함
    
- 점수를 산출하고, 학생들에게 점수를 분배함
    
- 다음 주에 가려면 학생들이 휴대폰으로 수업을 들을 수 있도록 함
    
- "코드의 재뉴얼"을 선택하여 변수를 변경하여 재뉴얼함
