1.파이썬 기반 웹 애플리케이션 구현 및 서비스

1-1.파이썬 기반 웹 애플리케이션 구현

- 파이썬 기반 웹 애플리케이션에서 입력 받은 결과를 출력하는 간단한 애플리케이션을 구현할 예정임
    
- 사용자로부터 입력을 받고, 그에 따라 서로 다른 결과를 출력하는 웹 애플리케이션을 만들 수 있음
    
- 애플리케이션에서 생성된 데이터를 데이터베이스에 저장 가능함
    
- 서버가 꺼지더라도, 저장된 데이터를 서버에서 불러와 사용자에게 제공할 수 있음
    

1-2.파이썬 기반 서버 배포 및 관리

- 파이썬 기반의 서버에서 빌려주는 작은 용량의 램과 하드디스크를 이용하여 웹 개발을 진행함
    
- 파이썬 패키지 안에서 스트림릿이라는 간단한 패키지 설치를 통해 웹 개발 환경을 구축함
    
- 파이썬 코드를 구현하기 전에, 서비스에 대한 개념적인 이해를 위해 가상환경을 먼저 생성함
    
- 가상환경을 생성하기 위해 파이썬 패키지에 'DCM'과 'Benv' 명령어를 추가하고, 'B'로 가상환경 이름을 입력함
    

1-3.파이썬 기반 웹 애플리케이션의 실행과 관리

- 파이썬 패키지에서 'S스템', '모듈', '코드', '디렉토리', '에티켓' 등의 명령어를 이용하여 웹 개발을 진행함
    
- '시스템' 명령어를 통해 웹 개발 환경을 생성하고, '모듈' 명령어를 통해 필요한 패키지를 불러옴
    
- '코드' 명령어를 통해 직접 작성한 프로그램을 실행하고, '디렉토리' 명령어를 통해 결과 파일을 저장함
    
- 서버에 서버 별로 배포하여 사용자에게 제공함
    

2.파이썬 가상환경 배포

2-1.파이썬 가상환경 구축

- 파이썬 코드로 웹사이트 생성, 이어서 스트림릿 패키지 활용해 볼 것임
    
- 스트림릿 제공자로부터 가상환경 구축을 위한 명령어를 받아들임
    
- 명령어를 스트림릿 실행 메서드에 따라 다른 방식으로 실행함
    
- 스트림릿을 통해 커스터마이징한 웹사이트를 외부에 배포할 수 있음
    
- 명령어 실행 시, 스트림릿의 버튼을 눌러 명령어를 입력하고 실행함
    

2-2.웹사이트 동작 메커니즘

- 웹사이트 동작은 '보기' 버튼을 눌러 열기로 시작, 명령어로 제어 가능함
    
- 메인 페이지에서 '이름', '이미지', '파트1', '파트2' 등 요소들을 첨부하여 만들 수 있음
    
- 이들 요소들을 스트림릿의 메서드에 따라 작성하거나 조합하여 사용함
    
- 사용자로부터 입력을 받기 위해, 입력 버튼을 통해 입력 값을 대입할 수 있음
    
- 입력 데이터는 메모리에 방을 대입하여 저장하고, 사진 찍기 전과 후의 변화를 보여줄 수 있음
    

2-3.가상환경에서의 웹사이트 배포

- 스트림릿의 메서드를 활용해 웹사이트를 빠르게 배포하는 과정이 중요함
    
- 파이썬 인터프리터가 스트림릿 패키지를 읽어 웹장 프라이브 액션을 생성함
    
- 스트림릿의 '포트' 개념을 활용해 외부 사용자의 서버 접근을 제한할 수 있음
    
- 서버의 램과 CPU는 허락한 수만큼만 실행하도록 제어하여 네트워크 보안을 강화함
    
- 여러 개의 서버를 분리해 띄워 서버 간 겹침을 방지하고, 한정된 서버로만 실행되도록 함
    

3.스트림릿 서비스

3-1.스트림릿의 소개와 작동 원리

- 스트림릿은 파이썬 패키지 중 하나임
    
- 파이썬의 입력 값과 결과 값을 연결해주는 디렉토리 구조를 지님
    
- 사용자로부터 입력받은 데이터를 기반으로 시각화 및 분석을 지원함
    
- 사용자가 특정 시간대의 데이터를 받아볼 수 있음
    
- 스트림릿은 파이낸스 데이터 리더와 웹플러 라이브러리를 사용함
    

3-2.스트림릿의 입력 값과 출력 값

- 스트림릿은 입력받은 값을 변수로 조작하고, 조건에 따라 출력 값을 결정함
    
- 특정 날짜의 데이터를 검색하고, 필요한 경우 엑셀 파일로 다운로드 받을 수 있음
    
- 입력받은 데이터를 바탕으로 시각화 및 분석 기능을 제공함
    
- 사용자가 입력한 회사명과 날짜를 기준으로 데이터를 조회하고, 이를 스타트, 엔드 데이터로 반환함
    
- 조회된 데이터 중 값이 없으면 제목에 입력받은 회사명과 날짜에 대한 결과값을 출력함
    

3-3.스트림릿의 활용과 배포

- 스트림릿은 사용자 편의를 위해 입력받은 종목 코드를 활용하여 해당 종목의 모든 데이터를 조회함
    
- 파이낸스 데이터 리더 패키지를 활용하여 종목 코드로 검색할 수 있음
    
- 사용자가 입력한 두 개의 날짜를 기준으로 데이터를 조회하고, 조건에 따라 그래프를 그릴 수 있음
    
- 사용자로부터 입력받은 회사명과 날짜를 바탕으로 데이터 프레임을 생성하고, 이를 스타트, 엔드 데이터로 반환함
    
- 서비스 테넌스와 웹플러 라이브러리를 설치하여 사용함
    

4.패키지 배포

4-1.패키지 설치와 저장

- 기존 서비스에 설치된 패키지들을 그대로 저장함
    
- 배포를 위해 필요한 파일들을 실제 서비스 컴퓨터로 옮김
    
- 패키지 설치 시 '기초부'로 명령어 사용을 권장함
    
- 'PIP 쇼'와 'PIP 리스트' 명령어를 사용해 패키지 정보를 확인함
    
- '얼리다' 명령어로 패키지를 기계가 알아보기 쉽게 변경 가능함
    

4-2.패키지 배포 과정

- 파이썬 패키지 배포 시 '리파이얼먼트' 파일명 사용을 권장함
    
- 요구사항을 포함한 파일명을 사용해 패키지가 올바르게 설치됐는지 확인함
    
- 기존에 설치된 패키지들을 '에코' 형식으로 결과를 나타냄
    
- 요구사항을 포함한 파일을 기존 패키지들과 함께 배포함
    
- 불필요한 패키지들을 제거하고, 사용하지 않는 파일들을 수집함
    

4-3.패키지 배포 자동화

- '에코' 형식의 결과 파일을 '기본자료'로 제공함
    
- 서버에 필요한 파일들을 직접 확인하고 배포 자동화를 통해 효율적으로 배포 가능함
    
- 'Jupyter' 환경을 통해 부가적인 패키지들을 설치하고, 불필요한 패키지를 제거함
    
- 1MB 미만의 패키지는 비효율적이므로 불필요한 패키지를 제거함
    
- 패키지 용량이 클 경우, 메모리에 올리기보다 서버로 올리는 것이 효율적임

5-1.레파지토리 관리에 필요한 명령어

5-3.레파지토리에 대한 파일 생성과 수정

- 상장법에 대한 목록 작성
    
- 이 목록에 따라 파일들을 파일명으로 구분하여 저장
    
- 기존에 생성된 파일들을 다시 모두 삭제하고 재생성

7.깃경로 관리

7-1.저장과 추적 관리

- 저장이 안 되어 '캐시드' 명령어로 저장했다가 'OK'를 받으면 저장이 완료된 것임
    
- 기존 저장된 파일들을 모두 지우고 새롭게 저장할 파일 목록을 확인함
    
- 기존에 따라가던 것들을 지우고 새로운 추정 리스트를 만듦
    
- 추정 리스트를 저장했다가 '이정' 명령어로 다시 저장하면 목록이 변경된 것임
    
- 기존에 따라가던 것들을 지우고 새롭게 따라가는 추정 리스트를 확인할 수 있음
    

7-2.기존 파일 제거와 추적

- '기존 목록에서 제거하기' 명령어로 기존에 따라가던 파일들을 지움
    
- 기존에 '리커올먼트' 파일이 있으면 '필요하지 않은 파일들' 명령어로 제거함
    
- '박정표'로 표시한 파일은 다시 올리면서 확인할 수 있음
    
- 기존 파일들을 제거했다가 '이정' 명령어로 기존 목록을 저장했다가 'OK'를 받으면 기존 파일들이 제거된 것임
    
- 기존 파일들을 확인할 수 있으니 꼭 주의해야 함
    

7-3.파일 배포를 위한 준비

- '파이썬 패키지' 명령어로 현재 상태를 저장함
    
- '필요하지 않은 파일들' 명령어로 불필요한 파일들을 간단히 작성함
    
- '위키리드' 사이트를 통해 '분류' 명령어로 파일들을 분류함
    
- '로우' 명령어로 작성한 파일은 원격 서버에 저장할 준비가 완료된 것임
    
- '배포 준비 완료' 명령어를 통해 배포를 위한 준비가 완료된 것을 알림

9.웹프레임워크와 파이썬 활용

9-1.웹프레임워크와 파이썬의 기초

- 웹프레임워크들이 표준으로 사용하는 숨기기들에 대해 설명함
    
- 파이썬의 방식으로 약속 파일을 불러오는 것이 아니라, 표준적으로 쓰는 방식을 이용함
    
- 파이썬에서 에 VMV 리뷰를 사용하여 업데이트하고 사용함
    
- 파이썬 다리에 설치하여 활용하는 방법을 설명함
    

9-2.파이썬을 이용한 웹프레임워크 업데이트

- 파이썬 다리에서 PIP 설치 및 활용하는 방법을 설명함
    
- 서비스 업데이트 시 파이썬 환경에서 동작하는 것을 확인함
    
- 기존에 파이썬 다리에 설치된 패키지가 변경될 경우, 새로운 업데이트를 진행함
    
- 패키지 업데이트 후, 로컬 변수로 다시 값을 읽어와서 동작을 확인함
    

9-3.웹프레임워크 업데이트와 배포

- 업데이트가 완료된 경우, 다시 액션 파이썬 환경에서 동작을 확인함
    
- 변수로 값이 변경되면 새로운 업데이트를 진행함
    
- 테스트가 완료된 경우, 서비스에 배포함
    
- 기존의 웹사이트가 단순해짐을 통해 보다 자세하게 세부 내용을 확인함
    

10.파이썬 코드로 통합 스트리밍 서비스 생성하기

10-1.파이썬 코드로 서비스 생성을 위한 초기 준비

- 데이터를 가지고 파이썬 코드로 스트리밍 서비스를 생성하는 방법을 설명함
    
- 기존 데이터로 새로운 폴더를 생성하고, 불필요한 패키지를 제거함
    
- 필요한 기능을 추가하고, 현재 상태를 파일로 저장함
    
- 스트리밍 서비스를 만드는 과정이 단순하나, 중요한 단계는 아님
    
- 복잡한 과정이 아니라 메모장에 순서대로 진행하면 됨
    

10-2.파이썬 코드로 스트리밍 서비스 만들기

- 스트리밍 서비스를 만들기 위해 코드를 작성하고, 필요 패키지를 불러옴
    
- 여기서부터 완성된 코드를 실행하고, 결과를 확인함
    
- 기존의 코드로 구현되지 않는 부분은 다른 기능을 추가하거나 제거함
    
- 프로젝트를 완성한 후, 결과를 확인하고 필요한 수정을 함
    
- 이렇게 수정된 코드를 다시 실행하고, 결과를 확인함
    

10-3.장애 해결과 마무리

- 강의에서는 장애가 생겼을 때 어떻게 해결하는지에 대해 설명함
    
- 기존에 작성된 코드로는 문제를 해결하지 못하는 경우, 다른 방법을 추가하거나 수정함
    
- 장애의 원인을 찾아내고, 그에 맞는 해결책을 찾는 것이 중요함
    
- 마지막으로, 수정된 코드를 테스트하고, 다른 개선점을 찾아내는 작업을 함
    
- 이 과정을 통해 프로젝트를 완성하고, 서비스를 출시함
    

11.프로그래밍 언어 및 코드 수정 방법

11-1.프로그래밍 언어의 기본 이해

- 프로그래밍 언어는 컴퓨터 언어로, 자바, 파이브, 파이썬 등이 있음
    
- 각 언어마다 특징과 장단점이 있어, 상황에 맞게 적절한 언어를 선택해야 함
    
- 자바는 디자인, 엔진 테스트, 반응형 프로그래밍에 주로 사용됨
    
- 파이썬은 객체 지향, 유연성, 반복성 등을 특징으로 하며, 프로그래머들이 선호하는 언어
    

11-2.코드 수정의 필요성과 방법

- 코드는 프로그램의 동작을 나타내는 코드로, 실수로 인한 수정이 많음
    
- 코드의 문제가 발생했을 때, AI에게 이의 제기하거나 조작하려 하지 말고, 빈칸을 새로 만들거나 분리해야 함
    
- 새로운 레포트를 만드는 것도 하나의 방법으로, 이에 대해 원하는 내용을 입력하면 수정됨
    
- 메모리에 올린 레포트를 새로 생성하려면, 연동을 해서 기존 레포트와 새롭게 만든 레포트를 모두 저장해야 함
    

11-3.코드 수정 시 고려 사항

- 프로그래밍 언어의 특성상, 코드의 수정은 연결과 연동을 통해 이루어져야 함
    
- 각 코드의 특성에 따라, 수정 과정에서는 사용자에 의해 개선될 수 있음
    
- 새로운 레포트의 생성은 기본적으로 원하는 방향으로 이루어져야 하며, 이는 AI에게 명령을 내리는 것이 아닌, 코드의 본질적인 특성에 따라 이루어짐
    
- 따라서 코드 수정을 할 때는, 원하는 방향에 맞춰서만 진행해야 함

3.파이썬 코드로 그림을 그리는 방법

3-1.코드로 그림을 그리는 초반 단계

- 파이썬 코드로 그림을 그리기 위해 필요한 도구로 플래틀리 설치와 파이썬 공식 문서 참조함
    
- 플래틀리로 그림을 그리는 과정에서 초기 환경 설정이 중요하며, 만약 설정이 잘못되면 코드 실행에 오류가 생김
    
- 플래틀리 공식 문서에서 맵핑과 다른 데이터 프레임 설정을 확인해야 함
    
- 선택적으로 사용하는 기능들을 플래틀리 코드에 입력하면 그림의 형성에 도움이 됨
    
- 그리는 과정에서 필요한 메소드들을 추가로 설치하거나 변경할 수 있음
    

3-2.그림의 세부사항과 디테일 조정

- 플래틀리 코드를 통해 그림의 세부 사항과 디테일을 조정할 수 있음
    
- 특히, "캔들" 자체에 대한 내용은 플래틀리 공식 문서를 참조하면 됨
    
- 조정된 그림의 구체적인 사항은 "스트림릿" 공식 문서에서 확인 가능
    
- 구체적인 조정 사항은 사용자 모드와 호환성에 따라 다르게 적용될 수 있음
    
- 그림의 화이트 위는 특정 지점에 표시를 하기 위해 필요함
    

3-3.그림의 변환과 배포

- 조정된 그림은 다시 플래틀리 코드를 통해 그림이 형성됨
    
- 그 후, 그림은 실제 환경에서 동작되도록 서버에 업로드해야 함
    
- 서버에 업로드된 그림은 웹사이트에서 보여짐
    
- 만약 기존 환경에서 변경사항이 반영되지 않았다면, 코드 수정이 필요함
    
- 변경사항이 반영되지 않았다면, 초기 환경에서 그대로 실행하는 것으로 이해하면 됨
    

4.스트림릿을 이용한 웹 개발

4-1.스트림릿과 그 중요성 이해

- 스트림릿은 웹 개발에서 사용하는 프레임워크임
    
- 사용자로부터 입력을 받되, 불필요한 정보는 제거하여 사용자의 요청에 부합하도록 관리함
    
- 서비스에서는 이러한 과정이 불필요한 정보를 제거하여 서비스의 편리성을 높임
    
- 웹 서버에서는 램에 코드를 올려 서비스가 계속 작동되도록 데이터베이스를 관리함
    

4-2.데이터베이스의 역할과 중요성

- 데이터베이스는 데이터를 저장하고 관리하는 장소임
    
- 데이터의 유동성을 유지하기 위해 가이드가 필요하나, 이를 지키지 않더라도 유동성은 어느 정도 유지됨
    
- 데이터는 복수형이며, 이는 다양한 값을 포함하고 있음
    
- 데이터의 관리에는 정형 데이터와 반정형 데이터가 있음
    

4-3.데이터의 수집, 저장, 관리

- 데이터의 수집은 사용자로부터 입력을 받는 과정임
    
- 사용자 정보는 스트림릿을 통해 데이터베이스에 저장됨
    
- 서비스에서는 이러한 데이터를 계속 업데이트하고, 이 과정에서 데이터의 유일성을 유지함
    
- 데이터베이스는 서버에 데이터를 영구적으로 저장하도록 설계됨
    

5.빅데이터와 데이터베이스

5-1.데이터의 구조와 종류

- 데이터는 입력과 출력으로 이루어져 있는 스템리 애플리케이션으로 구성됨
    
- 데이터베이스 연결과 AI 적용을 통해 데이터를 관리하고 활용함
    
- 데이터의 종류는 정형, 비정형, 정형 데이터가 대부분이며, 이 중 텍스트와 이미지 데이터가 대부분을 차지함
    
- 비정형 데이터는 규칙이 없고 크기나 해상도에 따라 정해지는 형태를 가짐
    
- 비정형 데이터에는 텍스트, 이미지, 음성, 영상 등이 있음
    

5-2.데이터베이스의 개념과 역할

- 데이터베이스는 데이터를 모아놓고 관리하는 시스템으로, DBMS(데이터베이스 매니지먼트 시스템)라고도 불림
    
- 데이터베이스는 객체 지향 프로그래밍 언어를 사용하여 데이터 행을 저장하고 관리함
    
- 데이터 행은 키를 이용하여 관리되며, 이는 데이터베이스의 기초적인 구성 요소임
    
- 데이터베이스의 생성, 유통, 소비는 이전보다 빠르게 이루어짐
    
- 빅데이터는 데이터를 관리하기 위한 시스템과 데이터에 대한 관점을 포괄하는 개념임
    

5-3.데이터 생성과 관리

- 데이터는 입력을 통해 생성되며, 데이터베이스를 통해 관리됨
    
- 데이터베이스의 프로그래밍은 객체 지향 프로그래밍으로 이루어지며, 이는 데이터베이스의 효율성과 안정성을 높임
    
- 데이터 생성에는 다양한 요소가 포함되며, 이는 데이터의 입력과 처리에 영향을 미침
    
- 데이터의 입력은 데이터베이스의 입력 행에 해당하며, 이는 데이터 생성에 중요한 역할을 함
    
- 데이터 입력은 데이터의 생성과 유지, 삭제에 영향을 미치며, 이는 데이터베이스 관리 시스템(DBMS)의 주요 기능임

9.워크벤치를 통한 SQL 쿼리 작성 및 실행

9-1.워크벤치 설치 및 동작 원리 이해

- 워크벤치는 SQL을 잘 알지 않는 사람들에게 만들어진 프로그램임
    
- 워크벤치를 이용해 DB에 접속하고 쿼리를 작성하면 동작함
    
- 워크벤치에서 필요한 편한 쿼리를 가져와서 실행함
    
- 워크벤치는 SQL의 역할을 하며, 사용자가 입력한 데이터를 이용하여 SQL 쿼리를 작성함
    

9-2.데이터베이스 및 테이블 관리

- 워크벤치를 통해 데이터베이스를 생성, 관리하고, 필요 시 수정하거나 삭제할 수 있음
    
- 데이터 베이스의 관리와 테이블의 생성, 수정, 삭제에 대한 정보를 DDL 언어를 사용함
    
- 테이블의 속성에 따라 필요한 데이터를 정의하고 관리함
    
- 데이터를 정의하는 것부터 데이터를 실제로 삽입하거나 수정하거나 삭제하는 것까지 모두 DDL 언어로 처리함
    

9-3.SQL 쿼리 작성 및 동작 원리 이해

- SQL은 표준분법으로, 다른 데이터베이스 매니지먼트 시스템에서도 거의 비슷한 쿼리문을 사용함
    
- SQL 쿼리는 데이터를 조작하는데 주로 사용되며, 주로 4가지 작업(데이터 정의, 생성, 수정, 삭제)을 포함함
    
- 데이터 정의 작업은 데이터를 어떻게 만들 것인지 정의하는 작업이며, 주로 DDL 언어로 수행함
    
- 데이터를 삽입하거나 수정하거나 삭제하거나 필요에 따라 조회하는 명령어들이 있음

- 트랜잭션은 데이터를 한꺼번에 처리하기 위해 사용되며, 실패 시 원상태로 복구할 수 있음
    

10-3.데이터 조작의 세부 방법과 주석 사용

11.MSQL 활용

11-1.MSQL의 특징과 용어 이해

- MSQL은 대소문자 대조 안 함, 운영체제 따라 달라짐
    
- MSQL의 데이터 저장과 활용을 위해 변수명과 명령어 구분 필요함
    
- 데이터베이스 스키마, 테이블, 행을 표현하는 데 쓰임
    
- 명령어는 대문자로, 변수명과 컬럼명은 소문자로 씀
    
- 명령어는 대소문자 사용 안 하지만, 변수명은 대소문자 가리지 않음
    

11-2.MSQL의 기초 용어와 명령어

- MSQL의 데이터 저장과 활용을 위한 기본 용어와 명령어 이해 필요함
    
- 쿼리, 행, 열, 단락, 본문, 결과 열 등의 용어를 알면 원하는 결과 얻을 수 있음
    
- '컨트롤 렌터' 명령어로 현재 MSQL 접속 위치와 작업자 확인 가능
    
- MSQL의 데이터 저장과 활용을 위해 '스크립트'와 '서밋'을 이용하는 방법 알아둠
    
- MSQL의 명령어는 대소문자, 변수명은 소문자로 쓰는 게 예쁘고 번역 가능함
    

11-3.MSQL 명령어와 단축키

- MSQL 명령어는 '에', '그', '또', '그리고', '그다음' 등의 단축키로 표현 가능
    
- MSQL 명령어의 단축키는 한 개 이상의 글자로 시작하고, 다른 글자로 끝남
    
- MSQL에서 '에'는 불릿 포인트를 의미, '그'는 지금 현재의 위치를 의미
    
- '또'는 조건에 따라 다른 값을 갖도록 설정, '그다음'은 이어서 다음 단축키를 의미
    
- MSQL 명령어에서 '에' 뒤에 오는 단어는 한 개 이상, '그' 뒤에는 다른 개의 값을 갖음
    

12.MSQL 데이터베이스 관리와 체계적 메모리 구조

12-1.MSQL 데이터베이스의 생성과 주석 처리

- MSQL 데이터베이스 생성 명령어는 '데이터베이스 생성'이며, 해당 줄에 주석을 놓고 위치를 옮김
    
- 컨트롤 레터를 누르면 주석이 몇 개든 현재 생겨나게 됨
    
- 한 줄 삭제 명령어는 '컨트롤 D'로, 줄 삭제 시 '잠시만요'로 함
    
- 복제, 즉 다른 위치에 새로운 데이터를 저장하려면 '컨트롤+다른 위치'를 이용
    

12-2.MSQL 데이터베이스의 체계적 메모리 구조

- MSQL 데이터베이스는 여러 줄로 주석이 가능하며, 주석은 탭으로 처리
    
- 테이블, 뷰 등 복잡한 객체들은 메모리에서 전체적으로 관리하고 사용
    
- 이 객체들은 '스키마' 안에 저장되며, 이 스키마를 만드는 명령어는 '데이터베이스 생성'
    
- 이렇게 생성된 스키마를 바탕으로 테이블을 생성, 삭제
    

12-3.MSQL 데이터베이스의 조작과 에러 메시지

- MSQL 데이터베이스의 테이블 생성, 삭제 시 에러 메시지가 발생하며 확인 필요
    
- 예를 들어, 데이터베이스를 없애려 할 때 '데이터베이스 삭제' 명령어 실행 후, '제대로 삭제되었는지' 확인 필요
    
- 해당 메시지는 '존재하는 데이터가 없음' 등과 같은 내용으로 나타남
    
- 이러한 에러 메시지를 통해 데이터베이스의 문제점을 미리 파악하고 개선할 수 있음
    

13.데이터베이스 테이블 만들기

13-1.데이터베이스 스키마 정의

- 데이터베이스 스키마를 크리에이트 명령어로 생성함
    
- 테이블을 만들 때도 크리에이트 명령어로 정의함
    
- 테이블은 하나 이상의 컬럼을 포함해야 함
    
- 데이터베이스는 메모리 공간을 차지하기 때문에 작은 공간으로 빠르게 처리해야 함
    
- 데이터베이스와 관련된 소프트웨어는 발전하고 있음
    

13-2.테이블 정의

- '학생' 테이블을 생성하려면 컬럼명, 자료형, 제약 조건을 명확히 정의해야 함
    
- 각 자료형은 정해진 자릿수만큼 공간을 차지하도록 함
    
- 문자열은 주로 바, 숫자도 소수점과 부동소수점이 포함된 값이 허용됨
    
- 시간과 연도를 포함한 숫자를 받아낼 수 있도록 제약을 둠
    
- 테이블은 행을 추가하거나 변경할 수 있으므로 조작이 필요할 수 있음
    

13-3.테이블 생성 과정

- 테이블을 생성하기 전에 컬럼명과 자료형을 명확히 정의해야 함
    
- '학생' 테이블의 컬럼명은 '학생', 자료형은 '숫자', '이미지'로 정의됨
    
- 테이블 생성 시 제약 조건을 설정하여 실수를 최소화할 수 있음
    
- 동작 시간을 확인하여 효율적인 코딩을 위한 조건을 설정할 수 있음
    
- 코딩을 단순화하고 복잡한 코딩을 최적화하기 위한 다양한 방법이 있음
    

14.테이블 생성과 제약

14-1.테이블 생성과 기능

- 데이터베이스 테이블 생성 기능을 사용해 학생 정보를 표현함
    
- 테이블 내 컬럼으로 이름과 5글자 이내의 데이터를 넣는 방식으로 생성함
    
- 테이블을 생성한 후, 해당 테이블을 '드랍'하면 데이터가 사라지는 것을 확인함
    
- 테이블 내 컬럼의 제약 조건을 추가하여 데이터의 정합성을 지키는 방법을 설명함
    
- 테이블을 생성할 때, '에러' 메시지를 받는 경우를 확인하여 데이터의 정합성을 점검함
    

14-2.다양한 자료형과 제약

- 테이블에 다양한 자료형을 생성하여 사용하는 방법을 소개함
    
- 자료형에는 숫자나 문자, 논리 등 다양한 종류가 있음
    
- 예를 들어, '이넘이' 타입은 '멀'과 '우먼' 두 가지 값을 받을 수 있음
    
- 타이니인트 자료형은 작은 숫자를 받아서 메모리에 작게 차지하면서도 데이터를 저장할 수 있음
    
- 테이블 내 컬럼의 제약 조건을 설정하여 데이터의 정합성을 보장할 수 있음
    

14-3.고유값과 중복 방지

- 테이블의 '인터페이스'에 고유값을 설정하여 중복을 방지함
    
- 고유값을 설정하면, SQL 명령어로도 테이블의 값을 가져올 수 없음
    
- 중복 없는 데이터를 조회할 때, 고유값을 기준으로 조회하도록 설정함
    
- M펑션과 같은 자동화 툴을 사용하여 고유값을 포함한 데이터를 더욱 쉽게 조회할 수 있음
    
- 이 노력을 통해 테이블의 데이터 정합성을 보장할 수 있음
    

15.테이블 정의

15-1.테이블 생성 및 조건 설정

- 데이터베이스에 테이블 생성, 테이블 조작을 위한 프로그래밍 언어 숙지함
    
- 테이블 조작 시 조건에 따라 다른 명령어 사용함
    
- 프라이머리 키는 중복을 막고 식별하는 값으로 설정함
    
- 테이블의 자료형과 순서를 설정하고, 원하는 값을 삽입하기 위해 컬럼명 순서와 자료형을 지정함
    
- 작은, 큰 컬럼을 포함한 전체 테이블을 만들고, 각 테이블 내부에 구체적인 값을 삽입하기 위해 행 단위로 값 넣기 위한 명령어 설명함
    

15-2.값 삽입 및 수정

- 테이블에 특정 값을 삽입하기 위해 컬럼명과 자료형을 지정함
    
- 중복을 막고 식별하는 프라이머리 키를 설정함
    
- 조건에 따라 '삽입' 명령어를 사용하여 특정 테이블에 값을 삽입함
    
- 컬럼명을 생략하고 값을 일괄 삽입하여 오류를 최소화함
    
- 직접 값을 넣지 않고, 컬럼명 순서에 따라 간단히 값을 삽입하는 방법 설명함
    

15-3.테이블 조작 명령어

- '삽입', '삭제', '변수명', '셋값', '추기', '엑셀', '변수명 찾기' 명령어 설명함
    
- '삽입' 명령어를 사용하여 특정 조건에 따라 데이터를 삽입함
    
- '삭제' 명령어를 사용하여 테이블을 완전히 삭제함
    
- '변수명 찾기' 명령어를 사용하여 테이블 내의 특정 테이블을 찾음
    
- '셋값' 명령어를 사용하여 특정 조건에 따라 테이블의 값을 조정함
    

16.데이터베이스의 삽입과 조작

16-1.데이터베이스에서의 값 삽입과 조작

- 데이터베이스에서 삽입을 위해서는 특정 명령어를 활용해야 함
    
- '셀렉트' 명령어를 사용해 값들을 조회하고, 특정 순서대로 삽입
    
- 조작을 위해 '커서'라는 객체를 사용, 커서의 위치를 표현하기 위해 '널널' 사용
    
- '오토 인크리먼트' 기능을 활용해 특정 순서대로 값을 삽입
    

16-2.컬럼 별 널 값 삽입과 조건

- 특정 컬럼에 '널' 값을 삽입하면, 해당 컬럼의 다음 값에 '널'이 삽입된 것으로 판단
    
- 컬럼에 '널' 값이 있고, 이 값이 대소문자로 바뀔 수 있음
    
- 대소문자 변경이 불가능하도록 컬럼에 제약 조건을 설정
    
- 예를 들어, 성별 컬럼에 '대소문자 변수'라는 제약 조건을 설정해 대소문자 변환이 불가능하도록 함
    

16-3.튜플 삽입과 조회

- 특정 순서대로 여러 개의 튜플을 삽입할 때는 '컨트롤B' 명령어를 활용
    
- '컨트롤B' 명령어를 사용하면 튜플의 순서대로 삽입되며, 첫 번째 튜플부터 마지막 튜플까지 순서대로 삽입됨
    
- '데이터 튜플'에서는 순서대로 삽입되도록 설정
    
- 튜플을 한꺼번에 삽입할 때는 순서대로 삽입되며, 앞서 넣은 튜플부터 마지막으로 넣음
    
- 핵심은 데이터의 유형과 제약 조건을 잘 확인하며 삽입과 조작을 하는 것
