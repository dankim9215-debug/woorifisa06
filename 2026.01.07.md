1.파이썬 기초 문법

1-1.함수의 종류와 정의

- 함수는 자료를 입력받지 않고 시키는 동작, 입력받아 동작하고 끝나는 동작, 메모리에 달린 결과 값을 가져오는 동작, 이렇게 네 가지 종류가 있음
    
- 함수에 넣는 파라미터는 '밸류 형식'으로 받거나 '직접 형식'으로 받을 수 있음
    
- 함수 헤더에서 '디폴트 파라미터'를 미리 설정하여 사용자가 넣지 않으면 그 값을 디폴트 값으로 선택할 수 있음
    
- 함수 안의 파라미터는 '밸류 형식'으로만 받을 수 있고, '직접 형식'으로 받을 때는 강제할 수 없음
    
- 함수를 작성할 때, 파라미터 개수에 따라 값을 넣어야 하는지, 아니면 디폴트 파라미터를 사용해야 하는지 고려해야 함
    

1-2.함수의 메모리 주소

- 함수를 작성할 때, 함수가 실행되기 전과 동작 중인 스택 메모리의 메모리 주소를 확인해야 함
    
- 함수 안의 명령어들이 실제로 실행되는 스택 메모리 구조는 메모리 공간의 복사와 같음
    
- 함수에 값을 넣을 때, 함수 안의 별도의 공간을 만들어 실행하지 않고 직접 값을 저장하는 방식이 사용됨
    
- 메모리 주소를 공유하여 같은 메모리 주소를 가진 값을 가질 수 있음
    

1-3.좋은 함수와 불필요한 함수

- 함수를 작성할 때, 코드가 짧고 동작이 덜할수록 좋음
    
- 불필요한 메모리 공간을 만들지 않고, 입력받은 값을 변환하여 바로 리턴하는 함수가 좋은 함수임
    
- 파이썬 인터프리터는 자료형을 강제할 수 없으므로, 펑니언스를 리스트만 넣어 사용하길 원함
    
- 하지만 다른 사람이 사용하게 되면, 자료형을 강제할 수 없음
    

2.파이썬 함수와 전역변수, 지역변수 이해

2-1.파이썬 함수와 전역변수 이해

- 파이썬에서 함수는 특정한 기능을 수행하는 코드의 집합
    
- 전역변수는 스택 메모리에서 별도의 공간을 생성해 실행되며, 함수 종료 후 사라짐
    
- 전역변수와 지역변수는 메모리 공간에 별도로 저장되며, 지역변수는 전역변수 저장 공간에 겹치지 않게 관리해야 함
    
- 전역변수와 지역변수의 차이를 알고 적절히 사용하는 것이 중요
    

2-2.파이썬에서의 전역변수와 지역변수의 활용

- 파이썬에서 전역변수와 지역변수의 차이를 알고 적절히 사용하는 것이 중요
    
- 전역변수는 함수 실행 전에만 생성되며, 지역변수는 함수 실행 후에 생성되며 이후 변경 불가
    
- 복잡한 시스템을 만드는 경우, 함수 안에서 지역변수를 이용하되, 전역변수를 최소화
    
- 전역변수와 지역변수의 적절한 사용을 통해 코드의 가독성과 안정성을 높일 수 있음
    

2-3.파이썬에서의 함수 실행과 변수의 관리

- 파이썬에서 함수를 실행하면, 해당 함수가 실행되는 동안에만 전역변수에서 변수가 생성되며, 이후 전역변수에서 변하지 않음
    
- 함수를 실행한 후에는 해당 변수는 더 이상 전역변수에서 변하지 않음
    
- 각각의 변수는 함수에 따라 다른 방식으로 사용되며, 변수명이 겹칠 경우 다른 결과를 가져옴
    
- 따라서, 전역변수와 지역변수를 적절히 사용하고, 변수명의 일관성을 유지하는 것이 중요
    

3.재귀함수

3-1.재귀함수의 정의와 활용

- 재귀함수는 동작을 반복적으로 수행하기 위해 정의한 함수 안에 다시 자기 호출하는 방식임
    
- 이 과정을 통해 특정 상황에 대해 다양한 결과를 만들 수 있음
    
- 앱 다운로드에서 앱이 무한 반복을 막기 위해 재귀함수를 추가함
    
- 재귀함수는 반복문과 유사하지만, 반복되는 동작에 대해 개별적으로 제어할 수 있음
    
- 복잡한 동작을 간단한 코드로 실행할 수 있도록 재귀함수를 활용하면 유용함
    

3-2.재귀함수의 구현과 동작 원리

- 대비함수와 재귀함수를 만들 수 있음
    
- 재귀함수는 특정 위치에서 특정 작업을 수행할 때, 특정 조건을 만족하지 않으면 다른 동작을 수행하도록 구성할 수 있음
    
- 재귀함수는 특정 상황에 따라 다른 결과를 만들도록 의도적으로 구현할 수 있음
    
- 재귀함수의 결과는 파라미터를 통해 다른 곳에 전달되며, 파라미터의 변경사항은 재귀함수 안에 전달됨
    
- 재귀함수는 문제 해결에 있어 중요한 기술 중 하나임
    

3-3.재귀함수의 실습과 효과

- 재귀함수는 코드의 간결성을 높이고, 의도치 않은 결과를 방지하는 데 효과적임
    
- 강의자는 앱 다운로드에서 재귀함수를 사용해 무한 반복을 막음
    
- 재귀함수를 통해 특정 동작을 반복적으로 수행하는 과정을 제어할 수 있음
    
- 재귀함수는 복잡한 데이터 처리에서 특히 효과적임
    
- 재귀함수를 통해 다양한 결과를 만족하도록 조정할 수 있음
    

4.재귀함수와 람다의 활용

4-1.재귀함수의 이해

- 재귀함수는 특정 순서나 규칙에 따라 동작하는 함수임
    
- 재귀함수는 함수의 결과를 직접적으로 결과로 내놓지 않고, 다음 함수로 이동하게 함
    
- 재귀함수는 스택 메모리의 구조에 따라 종료 순서가 달라짐
    
- 재귀함수의 입력 순서와 반대로 종료 순서가 이루어짐
    
- 재귀함수는 결과적으로 여러 차원의 데이터를 처리할 수 있음
    

4-2.재귀함수의 활용

- 재귀함수는 간단한 동작을 반복하는 상황에서 활용됨
    
- 예를 들어, 1차원 리스트의 평탄화 작업을 재귀함수로 처리할 수 있음
    
- 재귀함수를 이용하면 데이터를 처리할 때 간결하고 효율적으로 동작 가능
    
- 재귀함수를 이용한 데이터 처리는 람다 키워드를 사용하여 간단한 동작을 람다식으로 표현할 수 있음
    

4-3.람다 함수의 이해

- 람다 함수는 메모리에서 바로 버릴 수 있는 함수임
    
- 람다 함수는 간단한 동작을 수행하고, 그 결과를 다른 변수로 넘기는 역할을 함
    
- 람다 함수를 선언하면 변수명으로 람다를 묶어서 사용할 수 있음
    
- 주로 1차원 데이터를 일괄 처리할 때, 람다 함수를 사용하여 솔티드 명령어로 정렬할 수 있음
    

5.데코레이터와 객체 지향 프로그래밍

5-1.데코레이터의 역할과 사용법

- 데코레이터는 함수의 결과를 조절하거나 명령을 순서대로 처리하는 도구임
    
- 입력값이 없는 함수나 반환 결과가 없는 함수에도 적용 가능
    
- 데코레이터는 코드의 재사용성을 극대화하는 역할을 함
    
- 데코레이터는 함수 안에 미리 작성된 데코레이터를 함수에 부여함
    
- 데코레이터는 코드의 특정 부분을 제어하는 데 사용됨
    

5-2.타이머 데코레이터의 구현

- 타이머 데코레이터는 시간을 재는 기능을 제공하는 데코레이터임
    
- 데코레이터는 입력받은 함수를 실행한 후 결과를 반환받음
    
- 입력값이 없는 함수에도 타이머 데코레이터가 적용 가능
    
- 입력값이 있는 함수에서도 타이머 데코레이터가 제대로 동작함
    
- 입력값이 있고, 출력이 있고, 반환 결과가 있는 함수에서도 제대로 동작함
    

5-3.객체 지향 프로그래밍의 한계와 문제점

- 객체 지향 프로그래밍은 개발자가 생성한 객체를 활용하는 방식임
    
- 객체 지향 프로그래밍은 처음에 획기적인 방법론이었으나, 시간이 지나면서 여러 문제가 발생함
    
- 불필요한 메서드들이 메모리에 올라가 부작용을 일으킴
    
- 객체 지향 프로그래밍은 결국 프로그래밍 철학의 한계에 도달함
    
- 객체 지향 프로그래밍의 한계는 객체 지향 프로그래밍이 만든 메모리의 불필요한 속성 때문임
    

6.함수형 프로그래밍과 메모리 효율성

6-1.함수형 프로그래밍의 특징과 원리

- 함수형 프로그래밍은 메모리 효율성을 위해 일관된 처리를 요구하는 함수들을 개발
    
- 입력에 대해 똑같은 방법으로 결과를 출력하며, 동작 자체가 이미 메모리 안에 들어가고 나감
    
- 함수형 프로그래밍의 주요 특징은 입력에 똑같은 방법으로 결과를 출력
    
- 함수형 프로그래밍을 사용한 종류의 방법론들은 효율적인 메모리 사용 가능
    

6-2.함수형 프로그래밍의 예시와 메모리 효율성

- 리스트 컴플레이션을 이용하여 일괄적으로 처리 가능
    
- 람다식을 통해 일괄처리를 정의하고, 원하는 대로 메모리에 저장
    
- 메모리 주소를 간직한 객체를 지연평가 또는 레이지 평가라고 함
    
- 함수형 프로그래밍을 이용하면, 동작 자체가 메모리 안에서 일어나고 결과는 메모리에서 풀려헤쳐짐
    

6-3.임의로 함수를 변환 시 실수에 대한 대응

- 데이터 변환 시, 함수형 프로그래밍을 이용하면 분류 에러를 방지
    
- 함수형 프로그래밍을 사용하면, 변경 후 의도치 않은 데이터 변환이 일어나지 않음
    
- 따라서 임의로 함수를 변환하는 경우, 변환에 실수 없이 함수형 프로그래밍을 이용
    
- 이는 메모리 효율성을 보장하며, 원하는 대로 데이터를 처리 가능
    

7.데이터 처리를 위한 함수형 프로그래밍

7-1.함수형 프로그래밍과 그 특징

- 함수형 프로그래밍이란 특정한 함수를 만들기 위해 필요한 코드를 분리, 묶음
    
- 이 과정에서 대상이 어떤 데이터인지, 어떤 종류의 데이터인지 확인 가능
    
- 해당 데이터의 손실이나 복잡성을 최소화 하기 위해 개수가 적은 원소를 중심으로 대상 집합을 선정
    
- 함수처럼 정의역과 치역 사이의 관계를 1대1로 매핑하는 방식 사용
    

7-2.함수형 프로그래밍의 활용과 문제점

- 데이터를 일괄 묶어 처리할 수 있는 함수형 프로그래밍의 활용
    
- 데이터의 복잡성을 줄이고 효율적으로 처리할 수 있게 해줌
    
- 하지만, 데이터의 손실이 발생하거나, 불필요한 메모리 공간이 증가하는 문제가 있음
    
- 이런 손실을 최소화하려면, 입력 데이터의 순수성과 처리속도가 중요
    

7-3.데이터 처리를 위한 함수형 프로그래밍의 주의점

- 함수형 프로그래밍 사용 시, 데이터의 손실이 일어나지 않도록 주의 필요
    
- 입력 데이터의 순수성과 처리속도가 효율적인 데이터 처리를 위해 중요한 요소
    
- 특히, 로그가 몇억 건으로 늘어나는 데이터에서는 이런 손실이 큰 문제가 될 수 있음
    
- 따라서, 데이터의 누락이 일어나지 않도록 코드 작성 시 신중해야 함
    

8.프로그래밍 언어를 활용한 집합 및 세트 표현

8-1.프로그래밍 언어를 활용한 집합 및 세트 표현

- 데이터의 특정 범위를 넘어서는 값들을 불필요하게 제거하여 효율성을 높임
    
- 배열에서 원소를 하나씩 추출하여 별도의 메모리 공간에 저장
    
- 범위를 넘어서는 값들을 제거하기 위해 인덱스를 사용
    

8-2.프로그래밍 언어를 활용한 세트 표현

- 리스트 컴플레인션을 활용하여 순서대로 세트를 묶음
    
- 일정한 키 값을 갖는 튜플로 묶어 세트를 표현
    
- 서브젝트와 스코어의 값을 갖는 딕셔너리로 변환하여 세트 표현
    
- 리스트 컴플레인션을 통해 각 튜플을 딕셔너리로 변환
    

8-3.프로그래밍 언어를 활용한 반복 동작 처리

- 팩맨과 유사하게 제거하려는 객체의 각 요소를 함수로 처리
    
- 반복 가능한 객체의 각 요소를 누적하여 반환
    
- 불필요하게 제거하려는 요소를 딕셔너리로 변환하여 사용
    
- 파이썬 내부에서 제공하는 '컴프레인먼션' 모듈을 활용하여 간편하게 설정 및 활용
    

9.데이터 처리 기법과 그래프의 활용

9-1.데이터 처리 기법과 그래프 개념 이해

- 데이터 처리 기법 중 하나인 reduce 함수에 대해 설명함
    
- reduce 함수는 배열의 값을 하나씩 더해 최종 결과를 얻는 기법임
    
- 특정 반복을 동작해 결과를 계산하며, 결과는 메모리 내에 남게 됨
    
- 데이터 처리를 위한 팩맨(=더하기) 같은 기능을 가지고 있음
    
- 그래프를 이해하는 것이 중요하며, 이는 데이터 처리 기법을 이해하는 데 도움이 됨
    

9-2.구현된 함수의 활용

- 어떤 계열의 값을 처리하는 함수를 구현함
    
- 어떤 요소가 변하면, 해당 요소의 전체값이 변하게 됨
    
- 값의 변환은 여러 단계를 거치며, 이 중 하나는 결과값의 변화임
    
- 해당 반복문을 이용해 다른 작업을 반복하게 됨
    
- 이런 과정을 통해 결과를 얻을 수 있음
    

9-3.함수를 이용한 데이터 처리의 효율성

- 함수를 이용한 데이터 처리는 불필요한 변수 생성을 줄여줌
    
- 반복되는 동작을 하나의 명령어로 줄여줌
    
- 함수를 이용하면, 동작을 반복하는 과정이 단순화됨
    
- 이를 통해 더 많은 패턴을 가진 데이터 처리가 가능해짐
    
- 동작을 반복하는 과정에서 발생하는 계산 과정을 자세히 이해하면 좋음

13.파이썬 파일 작성과 읽기

13-1.파일 작성과 초기화

- 하드디스크에 파일을 저장하지 않고 메모리에 임시로 파일을 생성함
    
- 메모리에 '파이썬' 폼으로 파일을 쓰고 다른 사람이 같은 이름으로 쓰지 못하게 함
    
- '입력' 명령어로 메모리의 '파이썬' 파일을 열고 '해'로 쓰기 위한 임시 메모리 버퍼를 만듦
    
- '해' 명령어로 메모리 버퍼를 닫으면, 파일의 내용이 하드디스크에 쓰임
    
- 파일을 닫고 난 후에도 메모리에 남은 파일을 다시 열어볼 수 있음
    

13-2.파일의 읽기

- '리드' 명령어로 메모리 버퍼의 내용을 한 줄씩 읽어오면, 더는 읽어올 수 없음
    
- '리드 라인' 명령어로 메모리 버퍼의 전체 내용을 한 줄씩 읽어오면 빈 리스트가 나옴
    
- '리드 라인스' 명령어로 메모리 버퍼의 내용을 한 줄씩 불러오면, 처음에는 한 줄씩 불러오다가 더는 불러올 수 없음
    
- '리드라인' 명령어로 메모리 버퍼의 전체 내용을 한 줄씩 불러오면, 처음에는 빈 리스트가 나옴
    
- 파일을 열 때와 달리, '리드라인' 명령어로 메모리 버퍼의 전체 내용을 한 줄씩 불러오면, 읽기한 내용이 모두 빈 리스트가 되는 것임
    

13-3.파일의 읽기와 확인

- '리드라인' 명령어로 메모리 버퍼의 전체 내용을 한 줄씩 불러오면, 처음에는 빈 리스트가 나옴
    
- 다시 한 줄씩 불러오면, 처음에는 한 줄씩 불러오다가 점점 줄 수가 줄어들 때까지 불러옴
    
- 파일을 열 때와 달리, '리드라인' 명령어로 메모리 버퍼의 전체 내용을 한 줄씩 불러오면, 처음에는 빈 리스트가 나옴
    
- 파일이 열렸는지 확인하려면, '쓰기 가능한지' 여부를 확인해야 함
    
- 파일을 닫은 후, '리드라인' 명령어로 메모리 버퍼의 전체 내용을 한 줄씩 불러오면, 파일이 열렸는지 확인할 수 있음
    

14.파일 작업

14-1.파일을 열고 읽기

- 파일을 열 때 '리드모드', '라이트모드', '열기' 명령어를 사용할 수 있음
    
- '리드모드'는 한 줄씩 불러오는 모드이고, '라이트모드'는 파일을 닫고 열 수 있는 모드임
    
- 파일을 열 때 '열기' 명령어를 사용하면 메모리에 파일 내용이 존재하지 않으므로, 파일을 열면 내용이 추가되지 않음
    
- 파일을 닫을 때 '닫기' 명령어를 사용하면 파일에 남은 내용이 메모리에 남아있음
    
- 파일을 열 때는 위드문을 사용하면, 들여쓰기 안의 동작을 일괄적으로 실행할 수 있음
    

14-2.파일 작업의 편리함

- 위드문을 사용하면, 들여쓰기 안의 동작을 일괄적으로 실행하여 편리하게 파일을 열고 쓸 수 있음
    
- 위드문을 이용한 파일 작업은 들여쓰기 안의 동작을 조합하여 전체 파일을 열고 수정하는 것과 같음
    
- 위드문으로 파일을 열고 수정한 후, 파일을 닫을 때는 위드문의 '미드' 명령어를 사용하면 됨
    
- 파일을 닫을 때, 위드문의 '닫기' 명령어를 사용하면, 파일에 남은 내용이 메모리에 남아있음
    
- 위드문을 사용하면, 들여쓰기 안의 동작을 일괄적으로 실행하여 작업의 편리성을 높일 수 있음

- 스트리밍의 성능과 효율이 파이썬과 파이브가 제공하는 기능에 어떻게 영향을 미치는지 설명함
    
- 스트리밍의 효과를 극대화하는 방법과 그에 따른 잠재력에 대해 설명함
    

17.메모리의 버퍼링과 데이터 처리

17-1.메모리의 버퍼링과 버퍼의 역할

- 메모리의 버퍼링이란 메모리에 작업이 끝나기 전까지 일부 공간을 할당해 주는 것을 의미함
    
- 이 공간을 버퍼라고 하며, 이로 인해 데이터를 일괄 처리할 수 있음
    
- 버퍼링이 이루어지지 않으면 안정성이 확보되고, 불필요한 오버헤드를 줄일 수 있음
    
- 메모리에 남겨둔 데이터로 처리하는 방식이 효율적이고, 병렬 처리보다 빠름
    

17-2.데이터 처리와 확장자

- 데이터 처리 시 확장자 사용을 권장하며, 이는 데이터 표 형태를 돕기 위함임
    
- 데이터를 표 형태로 처리할 때, 각 열에 데이터를 포맷별로 넣어줌
    
- CSV(자기주기식 보고문) 파일 형식이 권장되며, 이는 데이터를 편리하게 읽고 쓸 수 있게 함
    
- CSV 파일은 메모리에 남겨 두지 않고 한꺼번에 처리하므로, 메모리에 부담을 줄여줌
    

17-3.실제 데이터 처리 예제

- 데이터 처리를 위해 실제 CSV 파일을 작성하고 활용해봄
    
- 데이터는 '컬레'라는 확장자로 표시되며, 이를 활용해 데이터를 표 형식으로 쓰고 열어 봄
    
- 데이터의 각 열에 어떤 값이 들어가는지 확인하며, 데이터 표 형식을 파악함
    
- 데이터를 분할해주는 '엔드' 파라미터 사용과 '자기주기식 보고문' 형식의 데이터 처리를 진행함
    

18.데이터 처리와 메모리 활용

18-1.데이터 처리 메소드 활용

- 데이터 처리는 메모리 활용을 통해 이루어짐
    
- 메모리에서 데이터를 닫을 때 저장 형태가 바뀜
    
- 기존의 데이터 형태로 다시 열어두어 상황에 맞게 활용 가능
    
- 데이터 처리에는 자주 쓰는 메서드들이 있어 효율적인 활용이 가능
    
- 데이터의 패턴에 따라 각기 다른 메서드를 활용하여 처리
    

18-2.데이터 불러오기와 라인 사용

- 데이터를 불러오기 위해 메모리에서 파일로 가져오는 방식 사용
    
- 메모리에서 파일로의 데이터 전송은 단방향으로 이루어짐
    
- 불러온 데이터를 순서대로 한 줄씩 출력하는 라인 사용을 통해 효율적으로 처리 가능
    
- 특정 횟수로 반복 동작을 하기 위해 반복문 사용을 고려해야 함
    

18-3.메모리 효율과 반복문

- 메모리 효율성 위해 특정 횟수로 데이터를 불러오지 않고 중간에 브레이크를 걸어주어 반복을 제어
    
- 라인은 하나씩 불러오기 때문에 리스트로 사용해 효율성을 높임
    
- 한 줄씩 불러오면 메모리 효율이 떨어질 수 있으므로, 불러오는 횟수에 따라 반복문의 동작 조정 필요
    
- 반복문은 강제 종료를 통해 원하는 패턴이 완전히 생성되도록 설정 가능
    

19.문자열 내장 메소드

19-1.문자열 생성 및 내장 메소드 소개

- 문자열은 앞뒤로 따옴표로 감싸 놓은 문자들의 순서 있는 집합임
    
- 문자열 작업을 위해 내장 함수와 메소드를 사용함
    
- 메소드로는 슬래시 기준으로 문자를 끊거나, 문자열의 특정 문자를 리스트로 돌려주는 등의 명령어가 있음
    
- 분기, 조건문 등 여러 메소드를 이용해 문자열을 다양한 방식으로 분해, 합칠 수 있음
    
- 문자열을 가지고 특정 조건에 맞는 값을 찾아 치환하는 방법도 있음
    

19-2.분해, 합칠 메소드

- 슬래시 기준으로 문자를 끊어줄 수 있음
    
- '마이너스 1'로 시작하는 문자열을 '마이너스'로 치환할 수 있음
    
- 문자열의 모든 문자를 끊어주고, 조건에 따라 다른 값을 치환하는 방법도 있음
    
- 분해 메소드의 '디폴트' 파라미터는 일괄 적용 가능함
    
- 스트립 명령어는 문자열에서 노이즈를 제거함
    

19-3.스트립 메소드 특징

- 스트립은 왼쪽 기준으로 벗겨내고, 오른쪽 기준으로 벗겨내는 등 다양한 방법으로 적용 가능함
    
- 'L3' 명령어는 왼쪽 기준으로, 'L1' 명령어는 오른쪽 기준으로 비슷한 메소드를 사용함
    
- 스트립은 문자열의 노이즈를 제거하면서 중간에 있는 문자열은 제거하지 않음
    
- 불필요한 공백을 제거하여 원본 문자열을 그대로 유지하면서도 필요한 값을 치환할 수 있음

1.문자열 처리

1-1.문자열 내 데이터 처리

- 문자열에 데이터를 입력할 때, 대부분 문자열로 처리함
    
- 조건에 따라 데이터를 특정 방식으로 처리할 수 있음
    
- '더하기' 함수를 통해 문자열을 합쳐주는 방식을 소개함
    
- '끼리함수'를 통해 문자열을 특정 기준으로 분할하거나 결합하여 새로운 문자열을 만들 수 있음
    
- '디키셔너리' 자료형은 공백을 중심으로 순서가 있는 목록형 자료임
    

1-2.문자열 변환

- 문자열을 특정 기준에 따라 변환하는 방법을 설명함
    
- '들어놓기' 함수를 통해 문자열을 리스트 형식으로 변환할 수 있음
    
- '맥프트' 함수를 통해 문자열을 특정 기준에 따라 변환한 후, 제너레이터로 값을 끄집어내는 방식을 소개함
    
- '비트 밸류'를 통해 문자열 내 순서를 확인할 수 있음
    
- 문자열을 특정 기준에 따라 변환하는 메소드는 '스트링' 메소드와 '맥프트' 메소드로, 각각 다른 방식으로 활용됨
    

1-3.문자열 데이터 처리

- '해피' 노래의 가사에 등장하는 '해피'의 횟수를 찾는 예시를 소개함
    
- 노래 가사에 등장하는 '해피'와 '마지막에 등장하는 단어'를 찾는 과정을 보여줌
    
- 검색을 통해 가사에 등장하는 '해피'와 '마지막에 등장하는 단어'를 찾는 예시를 설명함
    
- 데이터를 '해피'와 '마지막에 등장하는 단어'를 순서대로 찾아주는 방식으로 변환할 수 있음
    
- 가사에 등장하는 '해피'의 횟수와 '마지막에 등장하는 단어'를 순서대로 찾아주는 방식을 구현함
    

2.스트링 데이터 분석을 위한 노가다 알고리즘

2-1.노가다 알고리즘과 스트링의 기능

- 노가다 알고리즘은 문자열 내 특정 단어의 반복을 분석함
    
- 해피라는 단어가 몇 번 나오는지, 그리고 가장 많이 나오는 단어를 리스트로 만드는 알고리즘을 작성함
    
- 스트링의 특정 기능을 이용해 중복되는 단어를 찾고, 이를 리스트로 만드는 방법을 학습함
    
- 고유값을 찾기 위해 스트링의 명령어 '셋'을 활용함
    

2-2.컬렉션 모듈과 검색 엔진

- 컬렉션 모듈은 문자열의 중복을 제거하고, 특정 문자열의 빈도를 계산하는데 사용됨
    
- 검색 엔진에서는 컬렉션 모듈을 사용해 문자열 내에서 특정 단어의 반복과 빈도를 파악함
    
- 소문자와 대문자 처리에 따라 결과가 달라질 수 있으므로, 이를 감안해 검색 엔진에 요청해야 함
    
- 연쇄 기법 또는 체이닝 기법을 사용해 검색 엔진의 결과를 처리함
    

2-3.문자열의 복잡한 처리

- 스트링에서 문자열을 복잡하게 분석하고, 중복을 제거하는 과정을 학습함
    
- 텍스트를 소문자로 바꾸고, 카운트라는 명령어를 사용해 문자열 내에서 특정 단어의 반복을 찾는 과정을 이해함
    
- 카운트 결과를 기반으로 해피와 같은 단어가 얼마나 많이 나오는지, 그리고 어떤 단어를 가장 많이 나오는지 정리함
    
- 이 과정은 스트링의 소속 작업으로, 향후 검색 엔진 개발에서 중요한 역할을 함
    

3.문자열 처리

3-1.문자열 변환과 전처리

- 문자열의 원본과 결과 값을 비교하여, 원본을 바꾸면 안 됨을 강조함
    
- 문자열을 수정할 때, 원본을 변경하면 중복되는 단어들이 남아 에러가 발생할 수 있음
    
- 함수를 사용할 때, 결과 값을 올바르게 반환할 수 있도록 주의해야 함
    
- 함수의 반환 값과 매개변수의 값이 일치하지 않으면 에러가 남
    
- 리스트와 관련된 함수들이 결과 값을 수정할 수 있으므로, 결과 값을 확인하는 것이 중요함
    

3-2.특수문자 처리

- 검색, 조회, 삽입, 수정 등 데이터 처리 시, 특수문자 처리가 필요함을 설명함
    
- 데이터에 특수문자(공백, 위, 아래, 아웃, 어퍼스트로피)가 포함되면 이를 처리해야 함
    
- 알파벳과 특수문자가 섞인 문자열에서 '알파'가 포함되면 '참'을, '이스'가 포함되면 '거짓'을 반환함
    
- 특수문자를 찾기 위해 스트링의 메모리에서 검색하고, 특수문자가 포함된 문자열을 검색함
    
- 거짓가 아닌 진짜 문자열을 출력하기 위해, '참'이 포함된 문자열을 출력하려면 '이스'가 포함되지 않아야 함
    

3-3.특수문자 처리 결과

- 특수문자를 찾은 후, 알파벳과 공백이 포함된 문자열에서 '알파'가 포함된 문자열만 남김
    
- '이스'가 포함된 문자열에서 '이스'가 포함되지 않으면 '참'을 반환함
    
- 특수문자를 제거한 후, 문자열을 소문자화하고, 공백을 넣어 처리를 완성함
    
- 제거한 특수문자에 대해 '캐'를 붙여주고, '리플레이스'로 변경하여 처리함
    
- 최종적으로, 조건에 따라 문자열을 한 칸 띄우거나, 위쪽을 넣어주어 처리를 완성함
    

4.스트링 데이터 분석

4-1.스트링 데이터 전처리

- 데이터에서 중복 제거, 칸을 잘라주고, 불필요한 요소 제거함
    
- 칸을 나누어 처리하며, 나누는 과정에서 오리지널 스트링을 잃지 않도록 백업을 함
    
- 얕은 복사를 기본으로 하며, 원본이 바뀌면 메모리 주소만의 참조는 유지되도록 함
    
- 파이썬에서는 기본적으로 얕은 복사를 하지 않으나, 리스트의 경우 깊은 복사를 해야 하므로 주의해야 함
    
- 스트링을 변경하면 원본 스트링에도 문제가 생길 수 있으므로 주의해야 함
    

4-2.단어 분석을 위한 준비

- '단어별 빈도수'를 계산하기 위한 알고리즘을 만들 필요가 있음
    
- 단어와 단어의 빈도를 기록하는 딕셔너리(사전 키와 값)를 생성함
    
- 딕셔너리의 키를 단어, 값은 단어의 빈도수로 설정함
    
- 딕셔너리의 각 요소를 빈도수로 만들 수 있도록 조건문을 만들고, 해당 요소를 추가하거나 수정함
    
- 솔트드라는 명령어를 이용해 딕셔너리의 키와 값에 따라 우선순위를 정할 수 있음
    

4-3.단어 순서 정렬

- 딕셔너리의 요소들을 순서대로 정렬하여 결과를 반환함
    
- 작은 것부터 큰 것 순서대로 정렬하거나, 특정 방식으로 정렬할 수 있음
    
- 예를 들어, '단어'를 '해피'의 10번째 값까지만 뽑아서 반환하도록 할 수 있음
    
- 다른 개발자들의 코드를 참고하여 단어의 순서를 정렬한 결과를 반환함
    
- 코드 작성 시, 데이터를 순서대로 처리하고, 중복을 제거하며, 필요한 부분만 뽑아서 반환하도록 주의해야 함
    

5.문자열 활용

5-1.문자열 내장 함수

- 문자열을 데이터 처리에 활용한 정수로 변환하는 방법이 있음
    
- 텍스트 데이터의 전처리 과정을 통해 같은 값이 나오도록 함
    
- 데이터의 특징을 파악하는 것이 중요함
    
- 스트링, 리스트, 집합 자료형 등의 함수를 활용하여 데이터 처리함
    
- 내장 함수를 통해 데이터의 특성을 파악하고 활용함
    

5-2.문자열 찾기

- '찾음' 명령어를 활용하여 문자열을 찾을 수 있음
